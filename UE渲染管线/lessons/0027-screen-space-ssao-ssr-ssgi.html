Part 5 · 延迟着色 · Lesson 0027

屏幕空间：SSAO / SSR / SSGI

三种以 GBuffer 深度和法线为输入、仅用屏幕内信息近似计算遮蔽/反射/间接光的技术——快且局限共存。

时长 ~22 分钟 前置 0021（GBuffer 布局）、0017（HZB 层级 Z 缓冲） 产出 能解释三种屏幕空间技术的输入、执行阶段与局限，以及它们在帧内的顺序

延迟着色的直接光照（0025）和阴影（0026）做完后，场景里仍缺少两类间接效果：角落与缝隙里的环境光遮蔽（接缝显得不自然）和反射与间接光弹射（镜面与漫反射间接光）。UE 4.26 在不动用光线追踪的前提下，用三个屏幕空间 Pass 来近似这些效果：SSAO、SSR、SSGI。它们统一的代价是：只能用当前帧屏幕内可见的信息，屏幕边缘和被遮挡的几何对它们而言是不存在的。

2SSAO：屏幕空间环境光遮蔽

代码：PostProcessAmbientOcclusion.cpp，着色器：PostProcessAmbientOcclusion.usf。

SSAO 的物理直觉：一个表面点的半球遮挡率越高（周围有更多几何挡住天空/间接光），它得到的环境光越少。SSAO 用屏幕空间的深度重建来采样这个点周围的"邻居深度"，统计有多少邻居高于自己，从而估算遮挡率。

具体步骤：

  1. 从 GBuffer 深度重建每个屏幕像素的视图空间位置；
  2. 从 GBuffer A/B 读取世界法线（作为半球朝向）；
  3. 在法线方向的半球内随机采样 N 个点（用旋转噪声图 tiling 以减少规律感），把采样点重投影到屏幕空间，与深度缓冲比较；
  4. 深度缓冲比采样点更浅（邻居比自己更近相机）→ 该方向被遮挡；
  5. 遮挡比率作为 AO 系数输出，存入 AO Mask；
  6. 用高斯/双边模糊平滑噪声。
// PostProcessAmbientOcclusion.usf（核心逻辑示意）
float Occlusion = 0.0;
for (int i = 0; i < NumSamples; i++)
{
    float3 SampleDir = TBN_Rotate(HemisphereSamples[i], N, RandomRot);
    float3 SamplePos = ViewPos + SampleDir * SampleRadius;
    float2 SampleUV  = ProjectToScreen(SamplePos);
    float  SampleDepth = SceneDepth.SampleLevel(SampleUV, 0).r;
    // 如果深度缓冲里比采样点更近 → 被遮
    Occlusion += (SampleDepth < SamplePos.z - Bias) ? 1.0 : 0.0;
}
OutAO = 1.0 - (Occlusion / NumSamples);

半球采样+深度比较是 SSAO 的本质。采样数和半径决定质量与性能的权衡。

SSAO 作用在间接光，不是直接光

SSAO 的 AO 系数被乘到间接（环境/天光）分量上，而不是直接阴影——因为直接光的阴影已由 0026 的 ShadowDepth + Projection 处理。把 SSAO 应用到直接光会让有阴影的区域变两次暗，造成过暗 artifact。代码里的 AmbientOcclusionIntensity 和 r.AmbientOcclusionLevels 都只控制间接光路径。

3SSR：屏幕空间反射

代码：ScreenSpaceReflections.cpp，着色器：ScreenSpaceReflections.usf。

SSR 给镜面/光滑表面提供屏幕内的反射信息。方法是屏幕空间光线步进（Ray March）：

  1. 读 GBuffer 法线和粗糙度，反向计算反射光线方向（视图空间）；
  2. 沿反射方向在屏幕上步进，每步把当前 3D 位置重投影到屏幕 UV，查 HZB（层级 Z 缓冲，0017）判断是否与场景几何相交——用 HZB 的 mip 级别控制步进粒度，远处用粗粒度快速跳步；
  3. 找到相交点后，采样 SceneColor 得到反射颜色；
  4. 粗糙度越高，反射越模糊（采样 SceneColor 的模糊 mip）；
  5. 根据步进路径的置信度输出 alpha，边缘渐变，供反射合成 Pass（0028）混合。
HZB 在 SSR 里的作用（呼应 0017）

SSR 的光线步进如果逐像素检测，代价极高。HZB 把深度缓冲做成 mip 金字塔，粗 mip 存一个区域内的最小深度（最近物体）。步进时先用粗粒度 HZB 大跨步前进，当 mip-0 层检测到潜在交叉时才细步确认——这就是 Hierarchical Ray March，让步进次数从 O(屏幕宽高) 降到 O(log)。

// SSR HZB 层级步进（伪代码）
int MipLevel = MaxMip;
while (MipLevel >= 0 && StepCount < MaxSteps)
{
    float2 UV = ProjectToScreen(RayPos);
    float  HZBDepth = HZBTexture.SampleLevel(UV, MipLevel).r;
    if (RayPos.z > HZBDepth)  // 射线在场景后面 → 相交
        MipLevel--;            // 细化步进
    else
        RayPos += RayDir * StepSize[MipLevel]; // 前进
    StepCount++;
}

4SSGI：屏幕空间全局光照

SSGI（Screen Space Global Illumination）在 UE 4.26 中通过 r.SSGI.Enable 1 开启（实验性功能）。思路与 SSR 相近，但目标是漫反射间接光（一次弹射）而非镜面反射：

  • 在法线半球内发射多条光线，同样用 HZB 层级步进；
  • 命中后采样 SceneColor（包含直接光照信息）作为该方向的入射辐亮度；
  • 按 Lambert 余弦加权积分，近似一次弹射漫反射全局光；
  • 时域降噪（复用 TAA 历史帧信息）平滑低采样率带来的噪声。

SSGI 与 SSAO 的区别：SSAO 只算遮蔽率（标量），SSGI 算带颜色的入射辐照（向量）——SSGI 包含了 SSAO 的遮蔽信息，两者同时开启时 SSAO 可关闭。

5三者对照：输入 / 输出 / 局限

技术主要输入输出核心局限
SSAO GBuffer 深度、世界法线 AO 系数（灰度，乘间接光） 采样半径内才能检测遮挡；法线不够精确时伪影明显
SSR GBuffer 法线/粗糙度、HZB、SceneColor 镜面反射色 + 置信 alpha 屏幕外物体无法反射；步进交叉深度正面时有缺失；高粗糙度质量差
SSGI GBuffer 法线、HZB、SceneColor（含直接光） 漫反射间接光（彩色辐照） 只有一次弹射；屏幕外 / 被遮挡表面不贡献；噪声需时域降噪
统一局限：屏幕空间的天花板

三种技术共享一个根本限制：只有屏幕内可见的几何才能参与计算。当视角转动、物体离开视野时，SSAO 角落遮蔽消失，SSR 反射闪烁，SSGI 间接光跳变——这些 artifact 不是 bug，是方法论的边界。UE5 的 Lumen 系统用场景级别的 SDF + 探针缓存来绕过屏幕限制，但那超出 4.26 范围（超纲）。

检索练习

SSR 为什么要用 HZB（层级 Z 缓冲）而不是直接用原始深度缓冲做光线步进？

从帧内顺序出发，依次说出 SSAO / SSGI / SSR 的输入、输出与核心局限（不看表格）。

帧内顺序：直接光照（0025）→ SSAO → SSGI → SSR → 反射合成（0028）。

  • SSAO：输入 GBuffer 深度+法线，半球采样统计遮挡率，输出 AO 系数乘间接光；局限：采样半径外无法检测，屏幕外几何不参与。
  • SSGI：输入 GBuffer 法线 + HZB + SceneColor，发射漫反射光线命中后读 SceneColor 积分，输出一次弹射漫反射间接光（彩色）；局限：单次弹射，屏幕外无贡献，需时域降噪。
  • SSR：输入 GBuffer 法线/粗糙度 + HZB + SceneColor，镜面反射方向层级步进，输出反射颜色+置信 alpha；局限：屏幕外无反射，步进缺失高粗糙度质量差。

带走这几条

  • SSAO（PostProcessAmbientOcclusion.cpp）：深度+法线半球采样，输出 AO 系数，仅作用于间接光分量。
  • SSR（ScreenSpaceReflections.cpp）：HZB 层级步进 + SceneColor 采样，输出带置信度的镜面反射，进入 0028 反射合成。
  • SSGI（r.SSGI.Enable，4.26 实验性）：漫反射方向 HZB 步进 + SceneColor，一次弹射间接光，包含 SSAO 遮蔽信息。
  • 三者共享的天花板：仅屏幕内可见信息，屏外几何不参与，边缘/遮挡处有 artifact。
  • UE5 Lumen 替代了这三者，4.26 不存在（超纲）。
