View 的诞生：从 PlayerController 到 FViewInfo

渲染一帧必须知道"从哪个角度、用哪个视锥看世界"——这个信息就是 View。这节课从 PlayerController 开始，拆清四个 View 相关类的职责边界：谁轻、谁重、谁跨帧持久。

时长 ~20 分钟 前置 0009（FScene/Proxy 两份场景） 产出 能说出 FSceneView、FSceneViewFamily、FViewInfo、FSceneViewState 各自存什么、属于哪层、为何存在

渲染器每帧必须回答两个问题：场景里有什么（答案在 FScene），以及从哪里看（答案在 View）。View 体系乍看有四个名字相近的类，但只要搞清"谁是给游戏/引擎模块用的跨模块接口"、"谁是渲染器内部重量级工作对象"、"谁存跨帧持久状态"，就完全不混了。

1从 PlayerController 到 FSceneView

游戏线程里，一帧的 View 参数由 ULocalPlayer::CalcSceneView（LocalPlayer.cpp）构造。它用 FSceneViewInitOptions 收集这一帧的相机参数：

// ULocalPlayer::CalcSceneView 节选（逻辑简化）
FSceneViewInitOptions ViewInitOptions;
ViewInitOptions.ViewMatrix = ...;             // 视图矩阵（相机 World → View）
ViewInitOptions.ProjectionMatrix = ...;       // 透视矩阵（View → Clip）
ViewInitOptions.SetViewRectangle(ViewRect);   // 视口像素范围
ViewInitOptions.ViewOrigin = CamLoc;
// 后处理设置、LOD 偏差、曝光设置等也在此设入

FSceneView* View = new FSceneView(ViewInitOptions);
ViewFamily.Views.Add(View);

FSceneView（SceneView.h）是各引擎模块共用的 View 接口——编辑器、在线子系统、调试工具都可以构造或读取它，不依赖 Renderer 私有头。它的核心内容：FViewMatrices（含视图/投影/视图投影矩阵和它们的逆矩阵）、FIntRect ViewRect（屏幕像素矩形）、后处理设置、曝光值、LOD 偏差、FSceneViewState*（见第 4 节）。

FViewMatrices 里有什么

FViewMatrices（SceneView.h）是一个自包含的矩阵集，存了当前帧的：ViewMatrix、ProjectionMatrix、ViewProjectionMatrix、它们各自的逆矩阵（InvViewMatrix 等）以及用于 HZB / TAA 的上一帧矩阵（PrevViewMatrix 等）。每帧构造一次，渲染器全程读它。

2FSceneViewFamily：一帧的全部 View

一帧里可能有多个 View：分屏（多个 ULocalPlayer）、编辑器分割窗口、VR 左右眼。FSceneViewFamily（SceneView.h）把同一帧的全部 FSceneView 收进一个数组，并携带帧级别的共享信息：渲染时间（CurrentRealTime）、调试渲染模式（EngineShowFlags）、是否允许后处理等。

FSceneViewFamily 在游戏线程构造，随后被传给 FRendererModule::BeginRenderingViewFamily，启动渲染线程对这一帧的渲染（见 0013）。

3FViewInfo：渲染器内部的重量级 View

渲染线程拿到 FSceneViewFamily 后，场景渲染器（FDeferredShadingSceneRenderer）把每个 FSceneView 上转为 FViewInfo（SceneRendering.h）——它继承 FSceneView，并在上面附加渲染器私有的中间数据：

数据字段类型（示意）用途
PrimitiveVisibilityMapTBitArray可见性 pass 后，标记哪些图元可见
RelevantLightsTArray<FLightSceneProxy*>影响这个 View 的灯光列表
DynamicMeshElementsTArray<FMeshBatchAndRelevance>GetDynamicMeshElements 收集结果
ShadingModelMaskInView位掩码View 内出现了哪些着色模型，用于按需启用 Pass
bHZBOcclusionTestsbool是否用 HZB 做遮挡测试

FViewInfo 只在渲染器内部（Renderer/Private/）使用；外部模块用 FSceneView。这条边界很清晰：跨模块接口轻量，渲染器内部工作对象重量。

关键洞见 · FViewInfo 是渲染器的"草稿纸"

FSceneView 是输入——它描述"从哪看"；FViewInfo 是工作缓冲——它在 InitViews、可见性、剔除、MeshDraw 收集的过程中不断被填充，是渲染器一帧内部状态的单点来源。每帧新建，帧结束即析构。要"持久"的东西放 FSceneViewState（第 4 节）。

4FSceneViewState：跨帧持久状态

FSceneViewState（ScenePrivate.h）是唯一需要跨帧保存的 View 相关数据，由 ULocalPlayer（或编辑器视口）持有，每帧通过 FSceneViewInitOptions::SceneViewStateInterface 注入 FSceneView。它存的数据全是"需要历史帧才能计算"的效果：

  • TAA 历史缓冲（TemporalAAHistoryRT）——时间抗锯齿需要上一帧颜色重投影累积。
  • 遮挡查询历史——UE 用异步遮挡查询（GPU 查询结果延迟一两帧返回），结果存在 ViewState。
  • Eye Adaptation（自动曝光）历史——按前几帧平均亮度平滑调整曝光，必须跨帧积累。
  • Screen Space Reflections 历史、SSGI 历史等屏幕空间效果的时间积累缓冲。

如果没有持久 ViewState（例如截图渲染），这些效果的历史帧数据会丢失，TAA 第一帧会"抖"、自动曝光会"闪"——这正是打开 stat rendertargets 能看到 TAA history texture 的原因：它在 ViewState 里跨帧活着。

游戏线程 ULocalPlayer::CalcSceneView → FSceneViewInitOptions → new FSceneView → 装入 FSceneViewFamily
FSceneViewState 由 ULocalPlayer 持有，注入 ViewInitOptions（跨帧持久）
渲染线程 接收 FSceneViewFamily → FSceneView 上转为 FViewInfo（渲染器内部工作对象）
InitViews 填充可见性位图、动态 MeshBatch 等中间数据
TAA/曝光等效果读写 FSceneViewState 里的历史缓冲

检索练习

为什么 UE 需要 FSceneViewState，而不是把 TAA 历史、遮挡历史都放在 FViewInfo 里？

从 ULocalPlayer 开始，不看笔记说出 View 体系四个类的构造顺序、所属线程和各自核心职责。
① 游戏线程：ULocalPlayer::CalcSceneView 用 FSceneViewInitOptions（含 FSceneViewState*）构造 FSceneView，装入 FSceneViewFamily。FSceneView 携带视图/投影矩阵（FViewMatrices）、视口 Rect、后处理设置，是跨模块轻量接口。FSceneViewFamily 是帧级容器，存本帧全部 View 和 EngineShowFlags。
② 渲染线程：FSceneView 在渲染器内部被上转为 FViewInfo（SceneRendering.h）——继承 FSceneView，追加可见性位图、动态 MeshBatch 等渲染中间数据，是渲染器真正的工作对象，单帧生命周期。
③ 跨帧：FSceneViewState 由 ULocalPlayer 持有（渲染线程读写），存 TAA 历史缓冲、遮挡查询历史、自动曝光历史等跨帧效果数据。

带走这几条

  • FSceneView（SceneView.h）：跨模块轻量接口，由 ULocalPlayer::CalcSceneView 构造，携带 FViewMatrices + ViewRect + 后处理设置。
  • FSceneViewFamily：帧级容器，收本帧全部 View，携带 EngineShowFlags 和渲染时间；构造完即触发渲染线程工作。
  • FViewInfo（SceneRendering.h）：渲染器内部工作对象，继承 FSceneView，单帧生命周期；InitViews 填充可见性位图、动态 MeshBatch 等中间数据。
  • FSceneViewState（ScenePrivate.h）：唯一跨帧持久的 View 状态，由 ULocalPlayer 持有，存 TAA 历史、遮挡历史、自动曝光历史——无此对象则时间积累效果失效。
  • "轻 vs 重"的设计原则：对外接口保持轻量（FSceneView），渲染器内部随意扩展（FViewInfo），持久状态单独隔离（FSceneViewState）。
