Part 6 · 后处理 · Lesson 0033

泛光与自动曝光

泛光通过高斯下采样/上采样金字塔提取高亮并柔和扩散；自动曝光用亮度直方图计算曝光值，在色调映射前把 SceneColor 缩放到合适范围。两者都在 HDR 线性域工作，都在色调映射之前完成。

时长 ~20 分钟 前置 0031（后处理概述）、0032（TAA） 产出 能解释泛光金字塔的下/上采样链、自动曝光直方图的工作机制及两者相对色调映射的顺序

泛光（Bloom）和自动曝光（Eye Adaptation）是最能影响游戏画面"感觉"的后处理效果，但它们的底层都是直接操作 HDR 线性颜色值。理解它们需要先接受一个前提：在物理摄影中，传感器会对过曝区域产生光晕（Bloom），人眼在明暗环境间需要适应（Eye Adaptation）。UE 的这两个 Pass 是在渲染管线层面对这两个物理/感知现象的模拟。

1泛光的整体流程：高斯金字塔

UE 4.26 的泛光实现基于高斯金字塔（Gaussian Pyramid）：先对 SceneColor 做一系列下采样（每级分辨率减半），再反向做上采样并叠加，产生具有不同频率宽度的柔和光晕。

代码路径：PostProcessBloom.cpp，着色器：PostProcessBloom.usf。整个泛光链不修改主 SceneColor 缓冲——它产生一张独立的泛光纹理，在色调映射 Pass 内以加法方式叠加到 SceneColor 上。这很重要：泛光 Pass 本身不输出最终结果，它只是为色调映射准备一个"光晕分量"。

2亮度提取：保留高于阈值的部分

金字塔的第一步是从 SceneColor 中提取"超亮"部分。实现是一个简单的线性阈值滤波：

// PostProcessBloom.usf — 亮度提取（示意）
float3 SceneColorRGB = SceneColorTexture.SampleLevel(Sampler, UV, 0).rgb;
// 用感知亮度（dot 为 BT.709 权重）
float  Luminance  = dot(SceneColorRGB, float3(0.2126, 0.7152, 0.0722));
// BloomThreshold 是 FPostProcessSettings::BloomThreshold，默认 -1（禁用阈值，提取全部）
float  BloomBlend = max(0, Luminance - BloomThreshold) / max(Luminance, 0.0001);
float3 Extracted  = SceneColorRGB * BloomBlend;

当 BloomThreshold = -1（UE 默认）时阈值禁用，整个 SceneColor 都进入泛光处理，但金字塔本身的卷积会自然压制暗区的贡献。

UE 4.26 的泛光阈值（r.BloomThreshold）默认为 -1，意味着对整个 SceneColor 做泛光而非只对高亮处理——这种"全带宽泛光"更接近真实摄影的镜头光晕，但对暗场景也有轻微辉光。若要节省性能，设为正数（如 1.0）可跳过暗像素。

3下采样链：构建高斯金字塔

UE 4.26 泛光默认使用 6 级下采样（可由 r.Bloom.Quality 调整，0–5 级）。每级分辨率减半，使用一个 13-tap 下采样滤波器（来自 Siggraph 2014 "Next Generation Post Processing in Call of Duty: Advanced Warfare" 的双线性 13-tap 采样方案）：

// 13-tap 双线性下采样（4+8+1 中心对称布局，示意）
// 用 4 个 2×2 双线性采样点 + 8 个偏移点 + 1 中心 = 13 tap
float3 a = SampleBilinear(UV + float2(-HalfTexel.x,  HalfTexel.y));
float3 b = SampleBilinear(UV + float2( HalfTexel.x,  HalfTexel.y));
// ... 其余 tap 略 ...
// 加权平均合并所有 tap，权重和为 1
float3 Result = 0.5 * (a + b + ...) * Weight;

13-tap 滤波在每步下采样时提供了比简单 4-tap 双线性更好的频率响应，避免混叠；双线性采样使用硬件插值，等效采样数更高但 texel 抓取数只有 13。

6 级下采样对应的分辨率（以 1920×1080 为例）：

金字塔层级分辨率主要捕获的频率
Level 0（输入）1920×1080高频边缘高亮
Level 1960×540中高频光晕
Level 2480×270中频光晕
Level 3240×135中低频扩散
Level 4120×68低频大范围辉光
Level 560×34最宽频率，远距离光晕

4上采样链：叠加各级频率

下采样完成后，从最小的层级（Level 5）开始反向上采样，每级放大一倍，并与上一级叠加。UE 4.26 采用带Tent 滤波的 3×3 上采样（9-tap，双线性采样等效）：

// 3×3 Tent 上采样（示意）
float3 Result =
    SampleBilinear(UV + float2(-TexelSize.x,  TexelSize.y)) * 1.0 +
    SampleBilinear(UV + float2(           0,  TexelSize.y)) * 2.0 +
    SampleBilinear(UV + float2( TexelSize.x,  TexelSize.y)) * 1.0 +
    // ... 中间行权重 2/4/2 ...
    // ... 底行权重 1/2/1 ...
    ; // 除以 16（权重和）
Result = Result / 16.0;
// 叠加：当前级 + 放大后的下级
Result = Result + SampleLevel(DownsampleLevel, UV);

每个上采样级别叠加该级的下采样内容，最终 Level 0 处的结果是所有频率光晕的叠加：宽度不同的泛光分量混合在一起，产生"内紧外松"的自然光晕形态。

各层级的叠加权重由 FPostProcessSettings::Bloom1Tint…Bloom6Tint（颜色分层染色）和 Bloom1Size…Bloom6Size（尺寸比例）控制，美术可以对不同频率的泛光单独调色，产生更丰富的光晕效果。

5自动曝光：亮度直方图

自动曝光（Eye Adaptation，眼部适应）模拟人眼从暗处到亮处或反向的亮度适应过程。在 UE 4.26 里默认用亮度直方图方法实现，分两个计算着色器 Pass：

直方图构建CS：遍历 SceneColor
统计亮度分布
›
曝光值计算CS：从直方图
提取目标曝光 EV100
›
1×1 曝光缓冲输出给色调映射
乘 SceneColor

代码：PostProcessEyeAdaptation.cpp，着色器：PostProcessEyeAdaptation.usf（或 AutoExposureCommon.usf）。

直方图构建 Pass

第一个 CS Pass 对 SceneColor（通常是下采样后的低分辨率版本以节省性能）的每个像素计算对数亮度，然后用原子加法（InterlockedAdd）把它累加进一个共享的 64 桶直方图缓冲：

// 直方图构建 CS（示意）
float3 Color = SceneColorTexture.Load(PixelPos).rgb;
float  LogLuminance = log2(dot(Color, LuminanceWeights) + 1e-5);
// 映射到直方图 Bin [0, 63]
float  NormLuminance = (clamp(LogLuminance, MinLogLuminance, MaxLogLuminance)
                        - MinLogLuminance) / (MaxLogLuminance - MinLogLuminance);
uint   BinIndex = (uint)(NormLuminance * 63.0);
InterlockedAdd(HistogramBuffer[BinIndex], 1);

对数亮度域的直方图对高动态范围场景比线性域更均匀：暗区和亮区都能获得足够的桶宽，避免全部样本堆在少数几个线性桶里。

曝光值计算 Pass

第二个 CS Pass（通常是 1 个线程组，64 个线程，每个线程处理一个 Bin）从直方图中计算出目标曝光值（EV100）：

  1. 丢弃最暗的 N% 和最亮的 M% 的像素（AutoExposureLowPercent / AutoExposureHighPercent，默认 80%/95%），避免极端像素（黑角、高光源）主导曝光；
  2. 对剩余桶计算加权平均亮度；
  3. 转换为目标曝光 EV100，与上一帧的 EV100 做时间平滑（速度由 AutoExposureSpeedUp/AutoExposureSpeedDown 控制）；
  4. 将最终 EV100 转换为线性曝光倍数，写入一个 1×1 的 R32F 缓冲。
EV100、曝光倍数与 SceneColor 缩放的关系

EV100 是摄影中的曝光值（Exposure Value），以 ISO 100、光圈 f/1.0 为基准。EV100 增大 1 = 曝光减半（场景变暗），减小 1 = 曝光翻倍（场景变亮）。转换公式：ExposureScale = 1 / (2^EV100 × 1.2)（其中 1.2 是 UE 的经验系数）。色调映射 Pass 把 SceneColor 乘以这个 ExposureScale，使平均亮度落在色调映射曲线的理想工作点。

6曝光方法：直方图 vs. 基础（Basic）

UE 4.26 的自动曝光有两种模式，通过 AutoExposureMethod（FPostProcessSettings）选择：

模式实现优点缺点
Histogram（默认）64-桶对数亮度直方图 + 百分位提取抗极端像素干扰，可精细控制百分位两个 CS Pass，共享内存原子操作
Basic对 SceneColor 的小尺寸 mip 做简单平均亮度极低开销，移动平台可用容易被单个极亮/暗区域拉偏

7泛光与曝光相对色调映射的顺序

理解顺序的关键问题：泛光和曝光在色调映射之前还是之后？

关键洞见 · 泛光和曝光必须在色调映射之前

为什么泛光在色调映射之前：色调映射是非线性压缩，如果先色调映射再加泛光，两个亮度不同的像素混合后的结果会与物理不符（因为非线性压缩打破了叠加性）。在线性 HDR 域加泛光，高亮部分的溢出（超过 1.0 的值）才能正确"扩散"出光晕。色调映射把这个扩散后的 HDR 整体压缩到 LDR。

为什么曝光在色调映射之前：曝光是对 SceneColor 的整体线性缩放（乘一个标量），它需要在色调映射的非线性曲线之前工作，否则同一个 EV100 在不同输入亮度下的效果会不一致（因为色调映射曲线不是线性的）。

最终进入色调映射 Pass 的输入是：

  • SceneColor × ExposureScale（已曝光校正的 HDR 线性色）；
  • BloomTexture（独立的泛光纹理，在色调映射 PS 里以加法叠加到 SceneColor）；
  • 颜色分级 LUT（0034 会详细介绍）。

手动曝光 vs. 自动曝光

当 AutoExposureMethod = Manual 时，曝光由 AutoExposureBias（EV 偏移）和 CameraISO、CameraAperture、CameraShutterSpeed 的物理摄影参数固定，不做直方图计算，曝光缓冲值是一个常数。这适合需要精确控制画面亮度的影视/建筑可视化场景。手动曝光下，自动曝光的两个 CS Pass 跳过，只在 CPU 算出一个固定 ExposureScale 传入色调映射。

检索练习

UE 4.26 泛光 Pass 的输出是直接叠加到 SceneColor，还是产生一张独立纹理供色调映射使用？为什么这样设计？

说出泛光金字塔的 4 个步骤（亮度提取、下采样、上采样、输出），每步的具体操作和输入/输出是什么。
  1. 亮度提取：输入全分辨率 SceneColor HDR；用亮度阈值滤波器保留高于阈值的部分（默认阈值 -1 即全部通过），输出第一级提取纹理。
  2. 下采样（6 级）：逐级分辨率减半，每步使用 13-tap 高斯卷积滤波（抗混叠）；产生从 1/2 到 1/64 分辨率的 6 张纹理，存储不同尺度的高亮信息。
  3. 上采样（6 级）：从最小层级开始反向，使用 Tent 3×3 滤波上采样，每级放大后与上一级叠加，逐步把所有频率的泛光合并。
  4. 输出：最终产生一张与 SceneColor 同分辨率的泛光纹理（独立于 SceneColor），传入色调映射 Pass 作为加法输入。

带走这几条

  • 泛光通过高斯下采样→上采样金字塔（默认 6 级）产生多频率光晕，输出是独立泛光纹理（不直接写 SceneColor）。
  • 下采样用 13-tap 双线性滤波，上采样用 Tent 3×3，在上采样时逐级叠加，保证不同尺度泛光混合。
  • 自动曝光默认用亮度直方图（64 桶对数空间），两个 CS Pass：构建直方图 → 百分位提取曝光 EV100 → 时间平滑 → 写 1×1 R32F 缓冲。
  • 泛光和曝光都必须在色调映射之前完成，因为它们需要在线性 HDR 域工作——色调映射的非线性压缩会破坏加法叠加的物理正确性。
  • 色调映射 Pass 的输入是：SceneColor × ExposureScale + 泛光纹理（加法）+ 颜色分级 LUT。
