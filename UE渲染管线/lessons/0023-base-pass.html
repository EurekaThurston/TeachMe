Base Pass：写 GBuffer 与速度

Base Pass 是材质着色器真正运行的地方。它的任务只有一件：把每个可见片元的材质属性编码进 GBuffer（MRT），同时可选地输出速度向量。它不做任何光照计算。

时长 ~16 分钟 前置 0021（GBuffer 布局）、0022（Prepass / Early-Z）、0001（MRT） 产出 能描述 BasePass 如何运行材质着色器并把属性写入 GBuffer（MRT），以及速度/贴花的处理

Prepass 建立了 SceneDepth，BasePass 拿着这张深度图开启 Early-Z，让每个像素着色器调用只发生在真正可见的片元上。然后，材质着色器读取 Texture 采样、法线贴图、材质参数，输出 BaseColor/Roughness/Metallic/Normal 等属性，经 EncodeGBuffer 编码写进 MRT 的 GBufferA/B/C/D——这一步没有任何光照求和，光照是光照 Pass 的事（0025）。这就是延迟着色"先写后读"承诺的 CPU 侧。

1代码入口：RenderBasePass

FDeferredShadingSceneRenderer::RenderBasePass（BasePassRendering.cpp）在 RDG 里注册一个写入所有 GBuffer + SceneDepth 的 Pass。Pass 内部，每个图元经由 FBasePassMeshProcessor 生成绑定了完整材质着色器的 FMeshDrawCommand。

// BasePassRendering.cpp（架构示意）
void FDeferredShadingSceneRenderer::RenderBasePass(
    FRDGBuilder& GraphBuilder,
    FExclusiveDepthStencil DepthStencilAccess,
    FRDGTextureRef SceneColorTexture,
    FRDGTextureRef SceneDepthTexture, ...)
{
    // RDG Pass：MRT = [SceneColor, GBufferA, GBufferB, GBufferC, GBufferD]
    // DepthStencil = SceneDepth（只读深度用于 Early-Z，写 stencil）
    RenderBasePassInternal(GraphBuilder, ...);
}

2BasePassPixelShader.usf 的核心流程

Engine/Shaders/Private/BasePassPixelShader.usf 是材质实例生成 PS 的模板。简化流程：

// BasePassPixelShader.usf（核心路径，简化）
void MainPS(FVertexFactoryInterpolantsVSToPS Interpolants, ...)
{
    // 1. 从 Vertex Factory 取插值数据（UV, 法线切线空间等）
    FMaterialPixelParameters MaterialParameters =
        GetMaterialPixelParameters(Interpolants, SvPosition);

    // 2. 调用材质图节点生成的代码，求出材质属性
    FPixelMaterialInputs PixelMaterialInputs;
    CalcMaterialParameters(MaterialParameters, PixelMaterialInputs, ...);

    // 3. 把材质输出填进 FGBufferData
    FGBufferData GBuffer;
    GBuffer.BaseColor  = GetMaterialBaseColor(PixelMaterialInputs);
    GBuffer.Roughness  = GetMaterialRoughness(PixelMaterialInputs);
    GBuffer.Metallic   = GetMaterialMetallic(PixelMaterialInputs);
    GBuffer.WorldNormal = TransformTangentNormalToWorld(...);
    GBuffer.ShadingModelID = GetMaterialShadingModel(...);
    GBuffer.Emissive   = GetMaterialEmissive(PixelMaterialInputs);

    // 4. 编码进 MRT 输出
    EncodeGBuffer(GBuffer,
        OutTarget0, // GBufferA = WorldNormal
        OutTarget1, // GBufferB = Metallic/Specular/Roughness/ShadingModelID
        OutTarget2, // GBufferC = BaseColor + AO
        OutTarget3, // GBufferD = CustomData
        SceneColorOut); // 写 Emissive 到 SceneColor（不含间接光照）
}
Emissive 直接进 SceneColor

自发光（Emissive）不需要光照计算，BasePass 直接把它累加到 SceneColor（RT0）。光照 Pass 之后再把直接光贡献叠加进去。因此 SceneColor 在 BasePass 之后已经包含 Emissive，光照 Pass 只是继续写入。

3速度缓冲：r.BasePassOutputsVelocity

速度向量（屏幕空间运动向量）供 TAA（0032）重投影历史帧和运动模糊（0035）使用。UE 4.26 里有两种输出时机，由 cvar 控制：

cvar 值行为优劣
r.BasePassOutputsVelocity=1在 BasePass 的 MRT 中直接输出 Velocity（额外绑定一个 Velocity RT）省去单独 Velocity Pass 的几何遍历；代价是 BasePass 的 MRT 更宽，带宽略增
r.BasePassOutputsVelocity=0在 BasePass 之后单独跑 Velocity Pass（RenderVelocities）BasePass 带宽更窄；但多一次几何遍历

默认行为因平台而异：PC 路径通常为 1（在 BasePass 内输出），以节省一次额外的遍历。

4DBuffer 贴花的注入顺序

贴花（Decal）在 BasePass 之前注入到 DBuffer（Decal Buffer），BasePass 的 PS 在采样材质属性之前会先读 DBuffer，把贴花的 BaseColor/Normal/Roughness 叠加进去。这让贴花无需第二次光照计算就能正确影响材质属性。

5前向与移动端路径（简述，超出延迟主线）

当材质设置为 Forward Shading 或目标平台为移动端时，BasePass 的 PS 直接在片元上计算光照并写进 SceneColor，不走 GBuffer。UE 4.26 的 Forward Renderer（ForwardShadingRenderer.cpp）就是这条路径。本课程延迟主线不展开，提一句作为参照：两者 BasePass 的几何遍历相同，差别在 PS 的输出目标（GBuffer vs. 直接 SceneColor）。

UE5 超纲

UE5 的 Substrate 材质系统改变了 BasePass 的 GBuffer 写入逻辑；Nanite 的 Visibility Buffer 路径则完全绕过传统 BasePass。这些均超出本课程范围。

检索练习

BasePass 的像素着色器在 EncodeGBuffer 调用之前，主要做了什么？

Emissive 为什么不需要光照 Pass 处理？DBuffer 贴花为什么在 BasePass 之前就写入？
Emissive 是材质自发光，本身就是能量来源，不依赖任何入射光线，所以 BasePass 直接把它累加到 SceneColor 即可，不需要 BRDF 积分。DBuffer 贴花在 BasePass 之前写入是因为贴花应该修改物体表面的材质属性（BaseColor、Normal、Roughness）——如果在 BasePass 之后再处理，就只能修改已写入 GBuffer 的值（可行但需要额外 Pass），而在 BasePass 内读 DBuffer 则让贴花的影响自然混入材质计算，贴花的光照完全由 GBuffer 里的最终材质属性决定，无需单独光照。

带走这几条

  • BasePass 的核心职责：运行材质着色器（BasePassPixelShader.usf），把材质属性经 EncodeGBuffer 写进 GBufferA/B/C/D（MRT）。不做光照。
  • 入口：FDeferredShadingSceneRenderer::RenderBasePass + FBasePassMeshProcessor（BasePassRendering.cpp）。
  • Emissive 直接写 SceneColor；ShadingModelID 写 GBufferB.a。
  • r.BasePassOutputsVelocity=1：在 BasePass 内输出速度向量（UE 4.26 PC 默认）；=0 则单独 Velocity Pass。
  • DBuffer 贴花在 BasePass 之前写入，BasePass PS 读 DBuffer 混合进材质属性，让贴花参与完整 GBuffer 光照。
