Part 3 · 渲染线程 · Lesson 0013

场景渲染入口：BeginRenderingViewFamily → SceneRenderer

从视口 Draw 调用到 FDeferredShadingSceneRenderer::Render() 在渲染线程上开跑，中间有七八个关键函数。这节课把这条调用链逐步展开，让你知道"谁触发了渲染、在哪里创建了 SceneRenderer、它被如何投递到渲染线程"。

时长 ~20 分钟 前置 0011（FViewInfo 创建）、0012（ENQUEUE_RENDER_COMMAND） 产出 能描述从视口绘制到创建 FDeferredShadingSceneRenderer 并在渲染线程跑 Render() 的完整链路

0012 讲清楚了命令队列的传送带。这节课的问题是：是谁把"渲染这一帧"这件大事推进队列的？ 答案是一条从 UGameViewportClient::Draw 到 FRendererModule::BeginRenderingViewFamily 再到 ENQUEUE_RENDER_COMMAND(FDrawSceneCommand) 的调用链，最终把一个完整构造好的 FSceneRenderer* 交给渲染线程去执行 Render()。

2游戏线程侧：从 Viewport 到 ViewFamily

每帧 Slate 刷新时调用 UGameViewportClient::Draw(FViewport* InViewport, FCanvas* SceneCanvas)。它创建一个 FSceneViewFamily，再根据当前激活的 APlayerController / ULocalPlayer 构建一到多个 FSceneView（分屏时可有多个）。FSceneViewFamily 包含渲染这一帧所需的全部视图参数：时间戳、屏幕尺寸、后处理设置、Feature Level、全部 View 列表。

// GameViewportClient.cpp（简化示意）
FSceneViewFamilyContext ViewFamily(
    FSceneViewFamily::ConstructionValues
        .SetWorld(GetWorld())
        .SetScene(GetScene())
        .SetRealtimeUpdate(true));

// 为每个本地玩家构建 FSceneView 并加入 ViewFamily
for (ULocalPlayer* Player : GetLocalPlayers())
{
    Player->CalcSceneView(&ViewFamily, /* out */ View);
}

// 把整个 ViewFamily 踢给渲染器模块
GetRendererModule().BeginRenderingViewFamily(SceneCanvas, &ViewFamily);
FSceneViewFamilyContext vs FSceneViewFamily

FSceneViewFamilyContext 是 FSceneViewFamily 的 RAII 子类，析构时自动清理视图分配的资源。真正传入渲染器的是基类指针 FSceneViewFamily*。游戏线程把整个 ViewFamily 所有权交给渲染器模块后，不应再碰这块数据。

3FRendererModule::BeginRenderingViewFamily

这个函数是游戏线程侧的"最后一步"，定义在 SceneRendering.cpp。它做三件事：

  1. 创建 SceneRenderer：调用 FSceneRenderer::CreateSceneRenderers()，按 Feature Level 和 Shading Path 实例化正确的子类。
  2. 把 SceneRenderer 移动进 lambda：确保渲染线程拿到所有权，游戏线程不再访问。
  3. ENQUEUE_RENDER_COMMAND(FDrawSceneCommand)：把 lambda 投入渲染线程队列。
// SceneRendering.cpp — 概念性示意
FSceneRenderer* SceneRenderer =
    FSceneRenderer::CreateSceneRenderers(ViewFamily, CanvasCanvas);

ENQUEUE_RENDER_COMMAND(FDrawSceneCommand)(
    [SceneRenderer, Canvas](FRHICommandListImmediate& RHICmdList)
    {
        RenderViewFamily_RenderThread(RHICmdList, SceneRenderer);
        delete SceneRenderer;   // 渲染完后在渲染线程销毁
    });

4CreateSceneRenderers：按路径选类

FSceneRenderer::CreateSceneRenderers 定义在 SceneRendering.cpp，根据 FSceneViewFamily::Scene 的 GetFeatureLevel() 选择实现：

4.26 中 FDeferredShadingSceneRenderer 定义在 Engine/Source/Runtime/Renderer/Private/DeferredShadingRenderer.h。它持有 FScene*、TArray<FViewInfo> Views，以及整帧渲染所需的所有中间状态。

5渲染线程侧：RenderViewFamily_RenderThread → Render()

渲染线程消费 FDrawSceneCommand 后调用 RenderViewFamily_RenderThread(FRHICommandListImmediate&, FSceneRenderer*)（同在 SceneRendering.cpp）。这个函数做最后的全局状态准备（设置当前渲染器、设置 GPU mask 等），然后调用：

SceneRenderer->Render(RHICmdList);

对于 PC 延迟路径，这就是 FDeferredShadingSceneRenderer::Render(FRHICommandListImmediate& RHICmdList)——整条 GPU 管线的总编排者。它的内容是下一课（0014）的主题。

SceneRenderer 的所有权转移

SceneRenderer 在游戏线程创建，被移动进 lambda 后所有权完全属于渲染线程。渲染线程执行完 Render() 后在 lambda 末尾 delete SceneRenderer。游戏线程在 ENQUEUE 之后不应持有这个指针——否则是典型的 UAF（Use After Free）隐患。这也是为什么 lambda 捕获的是裸指针而非引用计数：语义上是"单次所有权转移"。

检索练习

FDeferredShadingSceneRenderer 是在哪个线程、哪个函数里被创建的？

不看图，按顺序说出从 UGameViewportClient::Draw 到 FDeferredShadingSceneRenderer::Render() 的完整调用链，指出每个跨线程边界在哪里。

游戏线程：UGameViewportClient::Draw → 构建 FSceneViewFamily + FSceneView → FRendererModule::BeginRenderingViewFamily（SceneRendering.cpp）→ FSceneRenderer::CreateSceneRenderers（选 FDeferredShadingSceneRenderer）→ ENQUEUE_RENDER_COMMAND(FDrawSceneCommand)。

跨线程边界在 ENQUEUE_RENDER_COMMAND：lambda 入队后游戏线程返回，渲染线程异步消费。

渲染线程：消费 FDrawSceneCommand → RenderViewFamily_RenderThread（SceneRendering.cpp）→ SceneRenderer->Render(RHICmdList)（FDeferredShadingSceneRenderer::Render，DeferredShadingRenderer.cpp）。

带走这几条

  • 触发渲染的源头是 UGameViewportClient::Draw（游戏线程），经 FRendererModule::BeginRenderingViewFamily（SceneRendering.cpp）完成 SceneRenderer 创建与入队。
  • FSceneRenderer::CreateSceneRenderers 按 Feature Level 选择 FDeferredShadingSceneRenderer（SM5，PC 延迟）或 FMobileSceneRenderer（ES3.1，移动超纲）。
  • 跨线程边界是 ENQUEUE_RENDER_COMMAND(FDrawSceneCommand)——SceneRenderer 所有权从游戏线程转移到渲染线程 lambda，渲染完后在渲染线程 delete。
  • 渲染线程入口是 RenderViewFamily_RenderThread，最终调用 FDeferredShadingSceneRenderer::Render(RHICmdList)——这是整条 GPU 管线的起点，0014 展开它。
