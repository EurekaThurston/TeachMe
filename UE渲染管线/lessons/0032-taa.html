Part 6 · 后处理 · Lesson 0032

时间抗锯齿（TAA）

每帧抖动投影矩阵的子像素偏移，把多帧的超采样样本累积进历史缓冲；速度缓冲驱动运动重投影；邻域夹紧拒绝鬼影。TAA 是 UE 4.26 的默认抗锯齿方案。

时长 ~22 分钟 前置 0031（后处理概述）、0022（Base Pass 速度输出） 产出 能解释子像素抖动、历史缓冲重投影、邻域夹紧的工作原理，以及 TAA 为什么需要速度缓冲

传统 MSAA 在单帧内用多个覆盖采样点消除边缘锯齿，代价是每帧要多次写 GBuffer——这在延迟着色下显存成本极高。TAA 换了一个思路：每帧只用一个采样点，但把采样点在 2×2 或 Halton 序列的子像素位置上逐帧抖动，然后把多帧的样本在历史缓冲里累积，相当于把时间维度变成超采样维度。最终效果：几乎零显存开销的 8× 乃至更高效等效超采样，但要解决运动带来的鬼影问题。

1子像素抖动：投影矩阵的偏移

TAA 的抖动在渲染管线的最早端发生——修改投影矩阵。具体做法是在投影矩阵的平移分量（行/列 [2][0] 和 [2][1]，取决于约定）上加一个子像素偏移，使整个场景在屏幕上整体位移不超过半个像素。

偏移序列的选取影响最终效果：

  • 2 帧序列（Halton 2,3）：最轻量，收敛快，但累积采样数少，仍有残留锯齿；
  • 8 帧 Halton 序列：UE 4.26 默认，兼顾收敛速度和质量；
  • 可配置：r.TemporalAASamples 控制序列长度（默认 8），r.TemporalAAFilterSize 控制时间滤波核。
Halton 序列是什么

Halton 序列是一种低差异序列（Low-Discrepancy Sequence）：给定维度 d 和基数 b，第 n 项把 n 的 b 进制表示按小数点镜像展开。对 2D 抖动，用基数 (2,3)——Halton(2) 和 Halton(3)。相比伪随机，低差异序列在有限样本数下覆盖更均匀，抖动分布更好，同样 8 帧下质量优于纯随机抖动。

投影矩阵修改发生在 SceneView.cpp 的视图设置阶段（FSceneView 构造时）。之后整个场景的所有 VS 输出的裁剪空间位置都带有这个子像素偏移，光栅化结果自然产生子像素级别的偏移。这不需要修改任何材质或着色器。

2历史缓冲：FSceneViewState 上的跨帧存储

TAA 的核心数据结构是历史缓冲（History Buffer），它存储上一帧（或多帧）TAA 处理后的场景颜色，格式通常与 SceneColor 相同（PF_FloatRGBA 或 PF_FloatR11G11B10）。

历史缓冲不能是 RDG 瞬态资源（因为它要跨帧存活），它被存储在 FSceneViewState 上——这是与 FSceneView 对应的持久状态对象，生命周期与视口（Viewport）相同，而不是与单帧相同。

// ScenePrivate.h（结构示意）
class FSceneViewState : public FSceneViewStateInterface
{
public:
    // TAA 历史：上一帧输出的颜色纹理（External Resource 注入 RDG）
    TRefCountPtr<IPooledRenderTarget> PrevFrameViewInfo.TemporalAAHistory;

    // 速度缓冲历史（用于静止场景的历史失效检测）
    TRefCountPtr<IPooledRenderTarget> PrevFrameViewInfo.VelocityHistory;

    // 相机矩阵历史（用于相机运动的重投影）
    FMatrix PrevViewMatrix;
    FMatrix PrevProjectionMatrix;
    // ...
};

历史缓冲通过 GraphBuilder.RegisterExternalTexture() 以 External Resource 方式注入当帧的 RDG 图，TAA Pass 读取它，并在帧末将新的输出回写给 ViewState，供下帧使用。

3速度缓冲与运动重投影

静止相机时，上一帧的像素和当前帧的像素一一对应——直接比较同一屏幕位置即可。但一旦物体移动或相机旋转，需要重投影（Reprojection）：对当前帧每个像素，找到上一帧它对应的屏幕位置，从历史缓冲的那个位置采样。

重投影需要两类速度信息：

  • 相机速度：相机运动产生的全屏速度，可以从当前帧和前一帧的 View/Projection 矩阵差值计算得到，不需要专用缓冲；
  • 物体速度：场景中运动物体（骨骼动画、物体移动）产生的逐像素速度，必须在 Base Pass 时由顶点着色器计算并写入 Velocity GBuffer（SceneVelocity，PF_G16R16 格式，存储屏幕空间位移）。
Velocity GBuffer 的写入时机

UE 4.26 在 Base Pass 的顶点着色器中计算每个顶点的"当前帧裁剪空间位置"与"上一帧裁剪空间位置"的差，转换为屏幕空间位移后写入 SV_Target 中的速度通道。对静止物体，速度为 0（或跳过写入），只写相机运动引起的部分。这条速度数据同时被 TAA 和动态模糊（0035）消费——是它们的共同依赖。

TAA Pass 的重投影核心逻辑：

// TemporalAA.usf — 重投影采样（示意）
float2 Velocity = SceneVelocityTexture.SampleLevel(Sampler, UV, 0).xy;
// 速度缓冲存的是屏幕空间位移（相对于上一帧）
float2 PrevUV   = UV - Velocity;
float3 HistorySample = HistoryBuffer.SampleLevel(Sampler, PrevUV, 0).rgb;
// HistorySample 是上一帧对应位置的颜色

若 PrevUV 超出 [0,1] 范围（物体移出屏幕），历史无效，需要丢弃历史、用当前帧颜色替代（History Invalidation）。

4邻域夹紧（Neighborhood Clamping）：拒绝鬼影

重投影给出了上一帧的颜色，但这个颜色不一定和当前帧吻合——场景内容可能变化、遮挡关系改变、光照变化等。直接把历史颜色混入会产生鬼影（Ghosting）：之前帧的颜色"留"在当前位置。

解决方案是邻域夹紧（Neighborhood Clamping / Variance Clipping）：

  1. 对当前帧像素，采样其 3×3 邻域（9 个像素）；
  2. 计算邻域颜色的最小/最大包围盒（Bounding Box Clamp）或方差椭球（Variance Clip，精度更高）；
  3. 把历史颜色夹紧（Clamp）到这个包围盒内——如果历史颜色超出当前帧邻域颜色范围，说明它来自一个和当前帧不同的场景区域，强制拉回边界。
// 邻域包围盒夹紧（示意）
float3 NeighborMin = float3(1e10, 1e10, 1e10);
float3 NeighborMax = float3(-1e10, -1e10, -1e10);
for (int i = -1; i <= 1; i++) for (int j = -1; j <= 1; j++) {
    float3 c = SceneColorTexture.Load(int3(PixelPos + int2(i,j), 0)).rgb;
    NeighborMin = min(NeighborMin, c);
    NeighborMax = max(NeighborMax, c);
}
float3 ClampedHistory = clamp(HistorySample, NeighborMin, NeighborMax);

Variance Clip 用均值和标准差而非极值构建椭球，对亮度剧变区域（如高光边缘）夹紧更精准，但计算量稍高。UE 4.26 默认使用方差夹紧。

关键洞见 · 夹紧越激进，鬼影越少但时间稳定性越差

邻域夹紧的本质是"不信任历史"。夹紧范围越小，拒绝的鬼影越多，但也会把合理的历史信息丢弃，导致时间累积效果减弱、画面出现"闪烁（Flicker）"。r.TemporalAACurrentFrameWeight 控制当前帧颜色在混合时的权重（默认约 0.1），值越大倾向当前帧（少鬼影、多闪），值越小倾向历史（稳定、易鬼影）。这是 TAA 质量调优的核心 trade-off。

5混合权重与最终输出

夹紧后，TAA 把历史颜色和当前帧颜色按权重混合：

// 指数移动平均混合
float BlendFactor = saturate(CurrentFrameWeight); // 约 0.1 默认
float3 Output = lerp(ClampedHistory, CurrentColor, BlendFactor);
// Output 写回 SceneColor 并存入 HistoryBuffer 供下帧使用

这是一个指数移动平均（Exponential Moving Average）：历史贡献约 90%，当前帧约 10%。等效于对过去多帧做加权平均，权重随时间指数衰减。

混合权重还可以根据像素速度自适应调整：运动速度大的区域历史可信度低，可以动态增大 BlendFactor（更倾向当前帧），减少运动鬼影；静止区域则保持低 BlendFactor，充分利用历史累积。

TAA 与 Alpha 透明度的冲突

TAA 历史缓冲混合假设颜色在相邻帧间连续变化。透明粒子、半透明材质在每帧的渲染顺序和覆盖关系可能突变，导致夹紧失效、鬼影明显。UE 4.26 默认把半透明 Pass 的结果在 TAA 之后以单独方式合并（不经历史混合），但这意味着半透明不受 TAA 的抗锯齿保护。r.SeparateTranslucency 控制这个行为。

7为什么 TAA 必须放在后处理链最前面

TAA 消费的速度缓冲在 Base Pass 时写入，是整个 GBuffer 中生命周期最"易污染"的资源——任何后续写 SceneColor 的 Pass（泛光、景深）都可能产生和速度不匹配的颜色值，让历史重投影对不准。因此规则是：TAA 必须在所有修改 SceneColor 的后处理 Pass 之前运行，直接消费来自几何/光照阶段的原始 SceneColor。

这也说明了为什么泛光中出现的高亮边缘不应该"泄漏"进 TAA 历史——泛光是 TAA 之后才运行的，两者的时序保证了历史缓冲里存的永远是未加泛光的干净场景颜色。

检索练习

TAA 的邻域夹紧（Neighborhood Clamping）解决的是什么问题，它的核心操作是什么？

说出 TAA 从抖动到最终输出的完整数据流：投影矩阵→速度缓冲→重投影→夹紧→混合，每步的输入和输出是什么？
  1. 子像素抖动：在 FSceneView 构建时修改投影矩阵，加入 Halton 序列偏移（<0.5 像素）。输入：帧计数；输出：带偏移的投影矩阵，影响所有 VS 输出位置。
  2. Base Pass 写速度：顶点着色器计算当前帧与上一帧裁剪空间位置差 → 屏幕空间位移写入 Velocity GBuffer（G16R16）。
  3. TAA Pass 读速度，重投影：对当前帧每像素，读速度缓冲得到 ΔUV，从历史缓冲的 UV-ΔUV 处采样，得到历史颜色。
  4. 邻域夹紧：采样当前帧 3×3 邻域，计算颜色包围盒/方差椭球，历史颜色被 clamp 到范围内。
  5. 混合：lerp(ClampedHistory, CurrentColor, ~0.1)，输出写入新 SceneColor，并拷贝给 FSceneViewState 的 TemporalAAHistory 供下帧。

带走这几条

  • TAA 通过逐帧移动投影矩阵的子像素偏移（Halton 序列）积累多帧超采样，等效于高质量 MSAA 但无显存开销。
  • 历史缓冲存在 FSceneViewState 上，跨帧持久，以 External Resource 方式注入 RDG。
  • 速度缓冲（Base Pass 写入的 GBuffer Velocity 通道）是重投影的核心；它同时被 TAA 和动态模糊消费。
  • 邻域夹紧用当前帧 3×3 邻域包围盒拒绝失效历史，解决鬼影；夹紧越激进，鬼影越少但闪烁越多。
  • 混合权重约 0.1（当前帧），约 0.9（历史），形成指数移动平均；r.TemporalAACurrentFrameWeight 可调。
  • TAA 必须是后处理链的第一个 Pass——在泛光、景深等所有修改 SceneColor 的 Pass 之前。
