阴影：ShadowDepth、CSM、阴影贴图

阴影在 UE 4.26 里是独立的渲染阶段：先从光源视角渲染深度，再把深度贴图的遮挡结果投影回屏幕空间，供延迟光照整合。

时长 ~20 分钟 前置 0025（延迟直接光照）、0001（深度缓冲与 Early-Z） 产出 能描述阴影深度渲染、CSM 级联、阴影投影到屏幕遮罩的完整流程

延迟着色在光照阶段（0025）需要知道"这个像素是否在阴影中"。这个信息不能从 GBuffer 直接读出，必须另外生成——UE 的做法是在光照之前，从每个光源的视角把场景深度渲染进阴影贴图，然后用一个投影 Pass 把它转化为屏幕空间的阴影遮罩（Shadow Mask），让光照着色器查询。理解这条路径，就能理解为什么方向光的阴影精度随距离下降，以及 r.Shadow.* 那些 cvar 调的是什么。

主要代码文件：ShadowDepthRendering.cpp（深度 Pass）、ShadowRendering.cpp（投影 Pass），核心数据结构 FProjectedShadowInfo 贯穿始终。

2FProjectedShadowInfo：阴影的"配置表"

FProjectedShadowInfo 是整个阴影系统的中枢数据结构，每个需要渲染阴影的灯光/阴影类型都有一个（或多个）实例。它记录了：

  • 灯光视角的投影矩阵（ShadowViewMatrix、SubjectAndReceiverMatrix）——把世界坐标变换到光源的 clip 空间；
  • 阴影贴图在图集（shadow depth atlas）中的位置（X/Y 偏移与尺寸）——多个阴影共享一张大纹理；
  • 阴影类型标志：是否为 CSM 级联、是否为全场景阴影、是否为逐物体阴影（Per-Object Shadow）；
  • Subject Primitives：哪些图元要向阴影贴图投影（cast shadow），哪些要接收（receive shadow）。

FDeferredShadingSceneRenderer::InitDynamicShadows() 在 InitViews 阶段为每个灯光创建并初始化这些实例，决定本帧需要哪些阴影贴图，分配其在图集里的区域。

全场景阴影 vs 逐物体阴影

全场景阴影（Whole Scene Shadow）：一张阴影贴图覆盖整个可见场景，供方向光 CSM 使用，场景里所有 cast shadow 的图元都渲染进去。逐物体阴影（Per-Object Shadow）：每个动态物体/角色有自己的小阴影贴图，精度高但贴图数量多；用于点光、聚光灯，或当方向光 CSM 无法覆盖某物体时的补充。两者可共存，都挂在 FProjectedShadowInfo 上，只是标志位不同。

3ShadowDepth Pass：从光源视角渲染深度

函数入口：FDeferredShadingSceneRenderer::RenderShadowDepthMaps()（位于 ShadowDepthRendering.cpp）。它遍历本帧所有 FProjectedShadowInfo，对每个实例：

  1. 设置渲染目标为阴影深度图集（shadow depth atlas）的对应区域；
  2. 用灯光视角的投影矩阵代替相机矩阵，执行一次几何渲染；
  3. 像素着色器极简：只写深度，不需要材质颜色——这是最便宜的 Pass 之一；
  4. 启用 Early-Z（ShadowDepth 材质域使用 SE_BLEND_Masked 时才需要 PS，默认完全不跑 PS）。

HLSL 着色器路径：ShadowDepthVertexShader.usf / ShadowDepthPixelShader.usf。顶点着色器把顶点变换到光源的裁剪空间；像素着色器在蒙版材质时输出深度偏移，否则为空。

// ShadowDepthVertexShader.usf（简化示意）
void Main(
    FVertexFactoryInput Input,
    out float4 OutPosition : SV_POSITION)
{
    float4 WorldPos = VertexFactory.GetWorldPosition(Input);
    // 用光源的 ViewProjection 矩阵，不是相机矩阵
    OutPosition = mul(WorldPos, ShadowDepthPass.WorldToClip);
}

关键差异只有一行：用的是灯光的 WorldToClip 矩阵。

4CSM：级联阴影贴图

CSM（Cascaded Shadow Maps，级联阴影贴图）是方向光（Directional Light）的标准阴影方案。它解决了一个根本矛盾：方向光覆盖整个可见场景，若用单张阴影贴图，近处精度低；但若把贴图全给近处，远处就没有阴影了。

CSM 的做法：把相机视锥按距离切成多个 Sub-Frustum（级联），每个级联各自有一张方向光的正交投影阴影贴图，近处的级联贴图覆盖范围小（精度高），远处的覆盖范围大（精度低）。

UE 4.26 默认 4 个级联（可通过 r.Shadow.CSM.MaxCascades 调整上限）。每个级联对应一个 FProjectedShadowInfo，且标有 bWholeSceneShadow = true。级联分割点用 PSSM（Parallel-Split Shadow Maps）公式，结合线性与对数分割——控制台变量 r.Shadow.CSM.TransitionScale 调过渡柔化。

为什么远处阴影会"糊"或消失

最远的级联贴图覆盖面积最大，同样分辨率下每个 texel 对应的世界面积也最大，因此阴影边缘呈现块状（texel aliasing）。超过 r.Shadow.CSM.MaxCascadeDistance 的距离不再渲染 CSM——方向光不会有 per-object shadow 兜底（点/聚光灯才有），所以方向光阴影在极远处直接消失。这是 CSM 的固有局限，UE5 VSM（Virtual Shadow Maps）解决了此问题——但那是超纲内容。

5Shadow Projection：把深度贴图投影成屏幕遮罩

深度贴图生成后，还需要一个 Pass 把它"问询"结果写成屏幕空间的 Shadow Mask（阴影遮罩纹理），供延迟光照（0025）查询。这个 Pass 叫 Shadow Projection，着色器在 ShadowProjectionPixelShader.usf（对应 C++ 入口 ProjectShadows()）。

每个屏幕像素的处理步骤：

  1. 从深度缓冲重建该像素的世界坐标（反投影，与 0025 光照 Pass 相同的手法）；
  2. 用光源的 WorldToLight 矩阵把世界坐标变换到光源 UV 空间，得到采样坐标 (u, v) 和深度 d_receiver；
  3. 对阴影深度贴图采样得到 d_occluder；
  4. 比较 d_receiver 与 d_occluder（加 bias 防自阴影 acne）：若 d_receiver > d_occluder + bias，该像素在阴影中；
  5. 用 PCF（Percentage Closer Filtering）在周围多点采样取平均，得到 [0,1] 的软阴影值，写入 Shadow Mask 对应通道。
// ShadowProjectionPixelShader.usf（逻辑简化）
float ShadowTerm = ManualPCF(
    ShadowDepthTexture,
    ShadowUV,          // 世界坐标 → 光源 UV
    ReceiverDepth,     // 当前像素在灯光空间的深度
    DepthBias          // 防自阴影偏移
);
OutColor = float4(ShadowTerm, 0, 0, 0); // 写 Shadow Mask R 通道

Shadow Mask 是一张屏幕分辨率的纹理；对于方向光，R 通道存 CSM 的遮蔽值，光照 Pass 乘以它即得到阴影光照。

6距离场阴影（Distance Field Shadows）

UE 4.26 支持 Ray Traced Distance Field Shadows（DistanceFieldShadowing.cpp）：预计算每个 Mesh 的有向距离场（SDF），在阴影投影阶段用计算着色器向距离场发射光线，求遮挡距离。相比 CSM，距离场阴影：

  • 不受贴图分辨率限制，远处也能有清晰边缘（软阴影通过锥形追踪实现）；
  • 适合方向光中远距离的软阴影补充；
  • 需要额外显存存储每个 Mesh 的 SDF 图集，且在场景几何极复杂时构建耗时；
  • 开关：r.GenerateMeshDistanceFields 1，光源组件勾选 Distance Field Shadows。
UE5 超纲：Virtual Shadow Maps

UE5 引入了 Virtual Shadow Maps（VSM），用虚拟纹理思路把阴影贴图分辨率提升到极限，完全替代 CSM + Per-Object Shadow 的组合。VSM 是 UE5 Lumen/Nanite 套件的一部分，在 UE 4.26 中不存在，仅此提及以防混淆。

检索练习

CSM 中，近处级联的阴影贴图与远处级联相比，以下哪项描述正确？

说出从"光源深度贴图"到"延迟光照看到的阴影系数"之间经历的所有步骤（不看图）。
  1. ShadowDepth Pass：从灯光视角（用灯光的 ViewProjection 矩阵），把场景几何渲染到阴影深度图集，只写深度，不跑材质颜色。
  2. Shadow Projection Pass：对屏幕每个像素，从 GBuffer 深度重建世界坐标，变换到光源 UV 空间，与深度贴图比较（加 depth bias），PCF 多点采样取平均，得到 [0,1] 阴影系数，写入 Shadow Mask 纹理。
  3. 延迟光照（0025）：读取 Shadow Mask，把灯光辐照度乘以阴影系数，输出带阴影的光照结果。
对于方向光，步骤 1 重复 N 次（每个 CSM 级联一次），步骤 2 按像素深度选择对应级联的贴图采样。

带走这几条

  • FProjectedShadowInfo 是阴影的核心数据结构，记录灯光视角矩阵、图集位置、类型标志。
  • ShadowDepth Pass（RenderShadowDepthMaps）：从灯光视角只渲染深度，极度轻量。
  • CSM（Cascaded Shadow Maps）：按距离把视锥分 N 段，每段一张方向光深度贴图，近精远粗。
  • Shadow Projection：深度比较 + PCF 输出屏幕空间 Shadow Mask，供延迟光照乘以遮蔽系数。
  • 距离场阴影（Distance Field Shadows）不依赖贴图分辨率，适合远处软阴影补充。
  • UE5 Virtual Shadow Maps 解决了 CSM 远处精度问题，UE 4.26 中不存在（超纲）。
