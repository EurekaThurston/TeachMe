体积雾与大气

体积雾用 Froxel 3D 网格的计算着色器积分散射/消光；大气系统用物理散射模型模拟天空颜色与空中透视；指数高度雾叠加廉价近似。

时长 ~22 分钟 前置 0024（Light Grid / Froxel 概念）、0002（计算着色器） 产出 能解释体积雾的 Froxel 计算流程，以及 SkyAtmosphere、ExponentialHeightFog 的原理与叠加方式

雾与大气是两个层面的效果：体积雾在视锥内按物理散射方程逐 froxel 积分，得到光在雾中的散射（云隙光、体积光）和消光结果，最终以一张 3D 纹理供所有物体采样；大气则是天空本身颜色与空中透视（远处物体偏蓝）的物理模拟；指数高度雾是廉价的高度指数衰减近似，三者可组合使用。

2Froxel 网格：视锥空间体素

UE 4.26 的体积雾把相机视锥切成一个Froxel（frustum voxel）网格：

  • X / Y 轴：屏幕分辨率的子采样（默认 1/8 至 1/4，如 240×135）；
  • Z 轴：深度方向，按指数分布切片（近处切片薄，远处厚，默认 64 层到 r.VolumetricFog.Distance 限定的距离）；
  • 每个格子是视锥空间的一个小锥形体积，存储散射系数（Scattering）和消光系数（Extinction）。

这个网格与 0024 的 Light Grid 一脉相承——Light Grid 是屏幕空间的聚簇，Froxel 是视锥空间的体素，两者在深度轴上的分布策略类似（指数分布，近细远粗）。

3注入阶段：向 Froxel 写散射/消光

注入（Voxel Injection）计算着色器：对每个 froxel，计算该体积内的散射系数和消光系数：

  • 来自 AVolumetricFog 组件的全局均匀雾参数（散射颜色、消光比例）；
  • 来自 ULocalFogVolume（UE 4.26 的局部体积雾，可设置不同密度区域）；
  • 大气组件（SkyAtmosphere）的散射贡献（可选）。

4光照积分阶段：每格的散射光

第二个计算着色器对每个 froxel 计算该格内的入射散射辐亮度（In-Scattered Luminance）：

  1. 从 Light Grid（0024）取得影响该格的灯光列表；
  2. 对每盏灯，计算灯光到 froxel 中心的辐照度（含衰减）；
  3. 乘以相位函数（Phase Function，控制散射方向性，Henyey-Greenstein 模型）得到向相机方向的散射量；
  4. 对方向光，再乘以阴影贴图采样结果（cloud volumetric shadow，体积阴影）。
// VolumetricFog.usf 光照积分（示意）
float3 InScattering = 0;
for (uint LightIdx = 0; LightIdx < LightCount; LightIdx++)
{
    FLocalLightData Light = GetLocalLightData(LightIdx, GridIndex);
    float3 LightVec  = Light.Position - FroxelCenter;
    float  Atten     = GetDistanceAttenuation(LightVec, Light.InvRadius);
    float  Phase     = HenyeyGreenstein(CosTheta, FogPhaseG);
    InScattering    += Light.Color * Atten * Phase * Light.Luminance;
}
// 方向光乘阴影
InScattering += DirectionalLight * DirectionalShadow * DirPhase;
Froxel.InScattering = InScattering * Froxel.Scattering;

相位函数控制光在雾中是各向同性散射还是偏向某方向（正向散射产生云隙光/体积光效果）。

5散射积分阶段：前向累积

最后一个计算着色器沿Z 轴（从近到远）逐格积分，计算从相机看向每个 froxel 时光的透过率（Transmittance）和累积散射：

// 前向散射积分（Ray March along Z）
float3 AccumScattering  = 0;
float  AccumTransmittance = 1.0;
for (int z = 0; z < NumSlices; z++)
{
    float4 Froxel = FroxelBuffer[z];
    float3 InScatter  = Froxel.rgb;
    float  Extinction = Froxel.a;
    float  SliceThickness = GetSliceThickness(z);
    // Beer-Lambert 消光
    float  SliceTrans = exp(-Extinction * SliceThickness);
    AccumScattering  += InScatter * AccumTransmittance * (1 - SliceTrans) / Extinction;
    AccumTransmittance *= SliceTrans;
    // 写入结果 3D 纹理
    ResultVolume[int3(xy, z)] = float4(AccumScattering, AccumTransmittance);
}

Beer-Lambert 定律描述光通过介质的指数衰减；累积散射是沿视线路径中雾对光的贡献总和。

结果写入一张 3D RGBA 纹理（RGB=累积散射颜色，A=透过率）。之后不透明和半透明物体的 PS 只需按像素的视图空间深度在这张 3D 纹理里做一次采样，就能得到雾对该像素的影响——O(1) 查询，无论场景几何多复杂。

6SkyAtmosphere：物理大气散射

UE 4.26 新增的 ASkyAtmosphere 组件实现基于物理的大气散射：

  • 瑞利散射（Rayleigh Scattering）：小分子（氮/氧）对短波（蓝光）散射强，造成晴天蓝天、日落红橙天空；
  • 米氏散射（Mie Scattering）：大粒子（气溶胶、雾霾）的各向散射，造成朦胧感和云隙光白光；
  • 空中透视（Aerial Perspective）：远处物体因光在大气中散射和消光，颜色偏向天空色（蓝化/雾化），随距离指数增强。

实现上，SkyAtmosphere 预计算一组查找纹理（LUT）（透射率 LUT、散射 LUT、多次散射 LUT），运行时着色器查这些 LUT 得到任意方向/距离的大气散射值，而不是实时做光线步进。空中透视的结果写入一张屏幕空间（或 froxel）纹理，叠加到 SceneColor。

SkyAtmosphere 与 SkyLight 的分工

SkyLight 捕获天空的全局光照贡献（IBL，供地面物体接受间接光，见 0028），SkyAtmosphere 渲染天空本身的颜色（太阳盘、天空渐变、空中透视）。两者配合：SkyLight 在 SkyAtmosphere 驱动下实时重新捕获，保证 IBL 颜色与大气天空颜色一致。

7指数高度雾（Exponential Height Fog）

AExponentialHeightFog 是廉价的高度衰减雾近似：雾密度随高度指数衰减（低处密，高处稀），在 PS 中用一个解析公式计算视线路径上的积分雾量，不需要体积纹理。它比 VolumetricFog 便宜得多，但：

  • 没有局部密度变化；
  • 不计算真实灯光散射（无云隙光）；
  • 适合大场景的背景雾感，或作为 VolumetricFog 的廉价替代/补充。

8三者的叠加顺序与适用

技术原理开销适用
ExponentialHeightFog 解析高度衰减公式，无体积积分 极低 大场景背景雾感，廉价近似
VolumetricFog Froxel CS 积分，Beer-Lambert + 相位函数 中等（受 Froxel 分辨率） 云隙光、体积光、局部密度变化
SkyAtmosphere LUT 预计算瑞利/米氏散射 + 空中透视 低（LUT 查表） 天空颜色、日出日落、远景雾化

典型组合：ExponentialHeightFog 提供基础雾感 + VolumetricFog 加体积光 + SkyAtmosphere 驱动天空与远景大气——三者可同时启用，在 SceneColor 中按序叠加。

VolumetricFog 的性能热点

体积雾最贵的部分是 Z 轴切片数（r.VolumetricFog.GridSizeZ，默认 64）和覆盖距离（r.VolumetricFog.Distance）。切片数翻倍 = 光照积分 CS 工作量翻倍。降低移动平台性能的首选操作：减少 Z 切片（32）+ 缩短距离。体积雾结果 3D 纹理的分辨率由 r.VolumetricFog.GridPixelSize（屏幕空间下采样比）控制。

检索练习

体积雾的散射积分结果写入一张 3D 纹理后，不透明物体的像素着色器如何使用它？

说出体积雾从 Froxel 初始化到最终应用到像素的四个阶段及每阶段的核心计算（不看图）。
  1. 注入阶段：计算着色器对每个 froxel 写入散射系数（Scattering）和消光系数（Extinction），来源包括全局雾参数、局部雾体积等。
  2. 光照积分阶段：从 Light Grid 取得影响该格的灯光列表，对每盏灯计算散射辐亮度（衰减 × 相位函数 × 阴影），累积为格内的入射散射量。
  3. 散射积分阶段：沿 Z 轴从近到远逐格前向积分（Beer-Lambert），计算从相机到每格的累积散射颜色和透过率，写入结果 3D RGBA 纹理。
  4. 应用阶段：不透明和半透明物体的像素着色器按视图空间深度在 3D 纹理中采样一次，得到雾的散射（加）和透过率（乘），叠加到最终像素颜色。

带走这几条

  • 体积雾（VolumetricFog.cpp）：视锥 Froxel 网格 → 计算着色器三步（注入→光照积分→散射积分）→ 3D 纹理 → 物体 PS O(1) 采样。
  • 光照积分用 Light Grid（0024）取灯光，乘以相位函数（Henyey-Greenstein），积分用 Beer-Lambert 衰减。
  • SkyAtmosphere：LUT 预计算瑞利/米氏散射，空中透视叠加远景大气；驱动 SkyLight 的 IBL。
  • ExponentialHeightFog：解析高度衰减，极低开销，适合背景雾感。
  • 三者可组合叠加，VolumetricFog 最贵，性能热点在 Z 切片数和覆盖距离。
