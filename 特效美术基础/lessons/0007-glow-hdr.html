辉光、HDR 与 additive

为什么最热的火焰核心是「白」的而不是红的?为什么能量特效叠在亮背景上就消失了?这一课讲清发光的物理与两种混合的取舍。

本课目标(一个收获) 你能解释高能量为什么读作「白热」(value clipping 向白溢出),理解 HDR(值 >1)+ bloom 如何制造发光,并知道 additive(叠加)何时用、何时会害你(亮背景前消失、无暗部无实体)。

一、为什么核心是白的

火、能量、爆炸的核心,温度/能量最高的地方,在画面里几乎总是白的,外围才显出橙红。原因是: 能量越高 → 明度 value 越高 → 一旦超出屏幕能显示的最高亮度,颜色就向白溢出,这叫 value clipping(明度截断)。 最热处所有通道都顶满 = 白;离核心远一点、能量低一点,才"显"出它本来的色相。

这和真实的黑体辐射 blackbody色温梯度一致——温度从低到高:

暗红 红 橙 黄 近白 白热

暗红 → 橙 → 黄 → 白。核心白热 white-hot,向外按色温降到橙红。这条梯度就是火/能量"热感"的来源。

二、HDR 与 bloom

普通颜色的明度上限是 1.0(屏幕全白)。HDR(高动态范围)允许值 >1.0——比如 emissive 强度 ×4。 屏幕显示不出"比白还白",于是引擎的后期 bloom 把这部分超额的光溢出到周围像素,形成光晕。这就是"发光"在屏幕上的实现:bloom = value 溢出到周围。

拖动 HDR 强度,看核心怎么从饱和橙红 → 向白去饱和,同时 bloom 怎么变大:

强度 = ×1.6
核心向白去饱和中
回扣 0003 · 焦点 bloom 要适度。值给太高、阈值放太低 → 全屏泛白、整个画面被洗白,焦点丢失(就是 0003 讲的"明度层次塌掉")。bloom 是给该亮的核心用的,不是给全屏用的。控制 bloom 阈值,让只有真正超亮的部分才溢出。

三、additive vs alpha blend:两种混合

这是特效师每天都在选的开关,理解它能省你无数次"为什么看不见":

Additive(叠加):把特效的颜色加到背景上。只会让画面更亮、永不变暗。天生适合光、火、能量——因为光本来就是"加"上去的。代价:它没有暗部、没有实体感,而且在亮背景前会消失(背景已经很亮,再加一点点看不出)。

Alpha blend(普通半透明):特效按透明度盖在背景上,可以有暗部、有实体遮挡感。适合烟、碎片、尘——这些是"挡住光"的实体,不是"发光"。

试试 Additive + 亮背景:火球淡到几乎看不见 → 回扣 0001 的明度教训,additive 特效在亮背景前是高风险。这时该换 alpha、或给它压暗边。

四、颜色随强度变(Color-over-life)

真实的火/能量,强处趋白、弱处显橙红。所以你的颜色不该是一个固定色,而是随强度/寿命变化的一条梯度:核心白 → 中段黄橙 → 边缘暗红 → 透明。 这条梯度同时承担了"热感"和"明度衰减"两件事——它就是上面那条黑体色温条。

五、接回 Niagara

  • 材质 Emissive × (>1) HDR 强度:自发光通道乘一个 >1 的强度,配合后期 Bloom 才"发光"。强度推动 value clipping,核心自然趋白。
  • additive 材质做能量光:激光、闪光、能量罩用叠加,只加光。但记得它在亮场景会消失,做之前先想背景。
  • alpha / translucent 做烟:烟、尘、碎片用半透明,有暗部和实体遮挡。
  • 颜色梯度给热感:Color-over-life 用"白→黄→橙→暗红"的梯度,核心向白。
  • 控制 bloom 阈值:别让 bloom 阈值太低导致全屏泛白;只让真正超亮的核心溢出。

六、常见误区

  • additive 特效在亮背景前看不见:最常见的坑,回扣 0001。亮背景下改用 alpha 或加暗边。
  • 全屏 bloom:bloom 阈值太低/强度太高,画面被洗白,焦点全丢。
  • 核心不够白:核心还停在饱和橙红,没顶到白 → 读不出"高温/高能",显得"温"。
  • 纯饱和色核心:把核心做成纯饱和的红或蓝,而不向白去饱和 → 像染色玻璃,不像在"发热发光"。
