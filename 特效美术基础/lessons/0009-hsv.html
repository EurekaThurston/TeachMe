色彩三要素 HSV:把一个颜色拆成三个旋钮

老手看颜色不是看「这是什么色」,而是同时拧三个旋钮——色相、饱和度、明度。而其中,明度仍然为王。

本课目标(一个收获) 你能把任何一个颜色在脑里拆成 Hue(色相)+ Saturation(饱和度)+ Value(明度) 三个独立旋钮,并且明白:改 hue / sat 不改可读性,只有改 value 才改——所以「看不清」永远先动 value,不是动 saturation。

三个独立的旋钮

第 1 课我们只盯住 value。现在补齐另外两个。任何一块颜色都能拆成三件互不相干的事:

  • Hue 色相——它是「什么颜色」:红、橙、黄、绿、青、蓝、紫。在色环上转一圈,用角度 0–360° 表示。
  • Saturation 饱和度——它有多「纯 / 鲜」:从灰扑扑(0)到最浓最艳(100%)。降饱和就是往里掺灰。
  • Value 明度——它有多「亮 / 暗」:从纯黑(0)到最亮(100%)。这就是第 1 课那条黑白灰。

关键是「分别看」:看到一块颜色,不要笼统说「偏暗的脏蓝」,而要拆成「hue 蓝偏青、sat 中、value 低」。能分别拧这三个旋钮,是色彩这一整个模块的核心功——后面讲配色、冷暖、饱和度作工具,全建立在这上面。

为什么明度仍然为王

三个旋钮地位不平等。判断一张图,永远先看 value。 因为眼睛分辨形体、读出「谁在前、谁是焦点、边界在哪」,靠的是明度差(第 1 课定律)。hue 和 sat 负责「好不好看、什么情绪、什么属性」,但它们不负责读不读得清。

分工 Value 管「读得清」(结构、可读性、焦点);Hue + Saturation 管「好看 / 情绪 / 属性」(美感、冷暖、能量感)。两件事别混。一个特效可以色彩很美却读不清,也可以颜色很素却读得清——决定后者的永远是 value。

动手:三个旋钮 + 去色证明

下面是真·三旋钮。拖 H / S / V,左边是颜色,右边是它的去色(灰度)预览——右边那块灰的深浅,就是这个颜色的真实 value(用第 1 课的 relativeLuminance() 算出来,显示成 0–100 的数字)。

现在做个实验:把 V 固定不动,只猛拉 S 从 0 到 100。看右边的灰度块和那个 value 数字——几乎不动。这就证明了:饱和度救不了可读性。再反过来只拉 V,看灰度块立刻跟着亮 / 暗。

高饱和 ≠ 亮:新人最常踩的混淆

这是第 1 课易错点的根:很多人把「不够亮」误诊成「不够饱和」,于是猛加 saturation——结果颜色更艳了,却还是看不清。因为饱和和明度是两个旋钮,加错了旋钮当然没用。

更反直觉的是:纯色的固有明度天差地别。同样拉到 100% 饱和、100% value,纯黄本身就很亮(value 高),而纯蓝本身就很暗(value 低)。你可以在上面那个 demo 里验证:把 S 和 V 都拉满,只转 H——右边灰度块会跟着 hue 一起变深变浅,黄那一段最亮,蓝紫那一段最暗。

一句话 「这块纯蓝很饱和」和「这块纯蓝很暗」同时成立、毫不矛盾。饱和说的是纯不纯,明度说的是亮不亮。把纯色当亮色用(比如指望纯蓝核心很显眼),会翻车。

Chroma 与色彩的「视觉重量」

Munsell 色彩系统把三要素叫 hue / value / chroma(chroma 近似饱和度,指色彩的「纯度 / 强度」)。高 chroma 的纯色有视觉重量——它会「喊」,会抢注意力。

所以纯色不能乱铺:满屏都是高饱和纯色,它们会互相打架,谁都想当主角,结果谁都不突出,画面又吵又累。高手的做法是大面积压低饱和、把高 chroma 留给小面积的关键点——这正是后面 0012(饱和度作工具)、0013(主色+点缀)要展开的。这里先建立直觉:纯色是稀缺资源,不是默认值。

接回 Niagara

Niagara 里 Color 是 RGB(线性),但你应该在脑里换成 HSV 三旋钮来调,而不是瞎拖 RGB 滑块。具体:

  • HDR 强度 / Emissive 主要动的是 value。 把颜色的 RGB 整体乘以一个 >1 的强度,本质是抬 value(同时多少抬一点视觉饱和的「发光感」),所以「想更读得清」第一杠杆永远是它,不是 saturation。
  • 想换情绪 / 属性但保持读感,动 hue。 火从橙改成蓝白(冰火),只要 value 不变,可读性就不变——这是 hue / value 分工的实用价值。
  • 想更「实 / 厚」或更「梦幻 / 淡」,动 saturation,但记住它不改可读性。别再用加饱和来救「看不清」。
  • 调色时养成习惯:先定 value(读得清),再定 hue(什么属性),最后用 saturation 微调质感。

常见误区

  • 用饱和度救可读性。 「看不清 → 加饱和」是错诊。饱和度不进入明度结构,加再多也跳不出背景。先动 value。
  • 把纯色当亮色用。 指望一个 100% 饱和的纯蓝 / 纯紫核心很显眼——它很「艳」但很「暗」,在亮背景前照样溶掉。
  • 不分别看,只说「这是个脏紫」。 拆不出 hue/sat/value,就没法精确地改;说得清三个旋钮各在哪,才改得准。
  • 到处堆高 chroma 纯色。 纯色互相打架,画面吵且没焦点。纯色留给小面积关键处。
