明度归组与层次:焦点放在最大反差

把画面眯成 3 块明度。眼睛会奔向最强的明度对比——所以「焦点」是你用反差摆出来的,不是碰运气。

本课目标(一个收获) 你能把任意画面在脑中简化成 3–4 个大明度块(notan 思维),并据此主动安排焦点:把最大的明暗反差放在你想让玩家先看到的地方(通常是技能本身)。

notan:先把画面砍成 3 块

Notan(日语「浓淡」)指把一幅复杂画面归并成几大块明度来看——通常是暗 / 中 / 亮三档。专业画师在动笔前会先画一张拇指大的 3-value 缩略图(thumbnail),只用三个灰,定下「哪块暗、哪块中、哪块亮」的大关系。如果这张三色缩略图就立不住、读不清,放大、加细节、上颜色也救不回来。

对你而言,「眯眼看」就是最快的 notan 工具:眯到看不清细节时,画面自然塌成几个明度团块。能塌成清楚的 3 块 = 明度结构成立;塌成糊糊一片 = 没层次。

最朴素的 3-value 关系:暗(背景)、中(次要)、亮(主体)。先用这三档想清楚,再谈颜色。

焦点定律:最强明度对比 = 视觉焦点

这是本课的硬核心:眼睛被画面里最强的那处明度对比吸引过去。 不是被最饱和、也不一定是最大,而是明暗反差最剧烈的地方。

所以「让玩家看技能」翻译成明度操作就是:让技能与它周围背景形成全画面最大的明度差。两条路:

  • 亮核暗景:技能是白热高明度,背景压暗——最常见,最直给。
  • 暗描边亮景:在亮背景前,反过来给技能压一圈更暗的边/烟,用「暗咬亮」制造反差。

读图层次 hierarchy:谁满对比、谁退场

一幅画面里不该人人都喊。按重要性分配对比:

  • 主体(主技能):给满对比——最大明度差,占住焦点。
  • 次要元素(拖尾、环境特效):减对比,明度往背景靠一点,自觉退到第二层。
  • 背景:压成一团,明度收窄、别有抢眼的局部反差,给主体让路。
对比要节制 到处都是高对比 = 没有焦点。 如果画面里每个角落都有强烈明暗反差,眼睛被同时往十个方向拽,结果谁都看不进去。对比是一种稀缺资源:把它集中在主角身上才有焦点。战斗里多特效叠加时,这条尤其要命——必须有人是主角,其余压住。

动手一:把场景量化成 N 个明度(posterize)

左边是一个模拟战斗帧(暗背景、几个中明度元素、一个亮技能核)。拖滑块把它量化(posterize)成右边的 N 个明度档:档数越少,越接近 notan 缩略图。一路拖到 3,看画面塌成哪三块——以及亮技能核是否始终待在最高那一档(那就是焦点)。

应该看到什么 量化到 3 档:背景并进最暗块、几个元素并进中灰块、技能核单独占最亮块——三档分明、焦点清楚。这说明这帧的明度结构立得住:就算砍到三个灰,主体依然是全场最亮、反差最大,眼睛照样奔向它。如果到了 3 档技能核和别的东西并进了同一块,那就是焦点没拉开。

动手二:背景越暗,主体越跳

技能核本身不变,只拖动滑块改背景明度。看同一个核,如何随着背景变暗而越来越「跳」——这就是焦点由明度差决定,而非核自己变亮。

背景明度 70
应该看到什么 背景越亮,核与背景明度差越小,核越「平」、越溶进画面;背景越暗,明度差越大,同一个核越往外跳。你一根手指都没动核,只动了背景——焦点强度由两者的明度差说了算。

接回 Niagara

把「焦点 = 最大明度反差」翻成你引擎里的旋钮:

  • emissive 强度分层。 主技能的 emissive / HDR 强度拉到最高;拖尾、环境特效、命中残留压低。让明度有梯队,而不是齐刷刷一样亮。
  • 给特效让路。 背景、地面贴花、环境光适当压暗,腾出暗场,主特效才有反差可言。在很亮的场景里,这一步比继续加亮特效更有效。
  • 别让每个粒子都最亮。 如果所有 sprite 的 value 都挤在高位,画面就是一片白——全亮 = 没层次 = 没焦点。要有亮核、有压住的中间层、有暗的烟和边。
  • 多特效同屏时排座次。 战斗里同时炸开好几个,先定谁是主角(这一下技能),给它最大反差;其余主动减对比退场。

常见误区

  • 全屏 bloom 拉满。 整画面都泛光发亮,value 全挤到顶,反差被抹平——焦点丢失,变成一片亮糊。bloom 该是给最亮的核点睛,不是给整屏打光。
  • 次要特效和主技能一样亮。 拖尾、环境、buff 光和主技能一个亮度,它们就在抢戏打架,玩家不知道该看哪。次要的必须减对比、退一层。
  • 只顾把主体加亮,忘了压背景。 焦点是相对的。主体到顶了还不够跳?往往是背景太亮——压暗背景常比硬加主体更省、更有效。
