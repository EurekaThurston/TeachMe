gameplay 的色彩语言:约定与可达性

游戏里的颜色不只是好看——它是一套玩家被训练过的语言。而这套语言,不能只靠 hue 来说。

本课目标(一个收获) 把特效色当成一套约定/UI 语言来设计:一致的色彩约定让玩家一眼读到信息(红=危险、绿=友/治疗…)。同时记住可达性铁律——约 8% 的男性是红绿色盲,关键信息绝不能只靠 hue,必须叠加 value(明度)、形状/剪影、动态。这一条直接服务目标①:让特效在系统层面"读得清"。

颜色是一套约定 / UI 语言

玩家不是看到红色才"觉得"危险——他们是被无数游戏训练成:红=敌/危险、绿=友/治疗/安全、黄=可交互/暴击/拾取、蓝=魔法/护盾/冷却。这是一套约定(convention),跟红绿灯是一回事。

你的特效,就是这套语言的载体。当一个 AOE 危险区是红色、一个治疗光环是绿色,玩家不需要读文字就懂——前提是你遵守了约定。可读性在这里不来自"单个特效好不好看",而来自"约定一致 + 与游戏整体调色板形成对比"。

红 — 敌方 / 危险 / 受击 / AOE 警告
绿 — 友方 / 治疗 / 安全 / 增益
黄 — 可交互 / 暴击 / 拾取 / 高亮
蓝 — 魔法 / 护盾 / 冷却 / 法力
紫 — 减益 / 中毒 / 精英 / 稀有

常见(但非绝对)的游戏色彩约定。具体含义随项目而定,关键是「一致」。

项目色彩规范:一致性 > 好看

好的项目都有一份 VFX 配色规范(color spec):伤害类型(火=橙、冰=青、电=紫白)、阵营(红队/蓝队)、状态(中毒=紫绿、燃烧=橙)各自有固定色。 这份规范的价值不在"每条单独好看",而在整套一致:玩家学一次,处处适用。

所以当你做第 37 个特效时,正确的问题不是"这个火球什么颜色最炫",而是"本项目的火属性是什么色,我有没有遵守"。自创一个更漂亮但不合规范的配色,等于在一套语言里说方言——单看没问题,放进系统就制造混乱。readability 来自一致性,不来自单个的好看。

与游戏整体调色板对比

约定之外还有第二条:特效要能从场景和 UI 里跳出来。重要信息应该用背景里少见、高对比的色。比如一个 AOE 危险区铺在地面上,用与地面互补的色(土黄地面 → 紫红警告)最跳;如果危险区的色和地面撞色,玩家就看不见警告——再"正确"的红也没用。

这把第 10 课(互补)、第 13 课(按背景选色)和本课的"约定"接在了一起:先选符合约定的色,再在约定允许的范围内,挑与背景对比最强的那一版。

动手:色盲模拟(可达性铁律)

下面是一个典型场景:红色 = 敌、绿色 = 友。两个 token 只靠 hue 区分(我故意让它们明度接近、形状相同)。正常色觉下你一眼能分。

现在点「红绿色盲模拟」(deuteranopia 近似)——这是约 8% 男性玩家看到的画面。你会发现红和绿几乎变成同一种土黄,敌友难辨。然后点「补上明度 + 形状」:给敌人压暗 + 改成尖锐多边形 + 加图标,给友方提亮 + 保持圆形 + 加图标——即使在色盲模拟下,两者也重新可分。

看到了什么 只靠 hue 时,色盲模拟一开,红绿糊成一团——这就是"红敌绿友"对约 8% 玩家直接失效的真相。一旦叠上明度差(深=危险/亮=安全)+ 形状差(尖=敌/圆=友)+ 图标,信息就有了三条独立通道,任何一条都能读懂。这正是第 1 课"明度为王"在可达性上的回扣:把信息编码在明度和形状里,色盲、低对比、繁杂背景下都不丢。

接回 Niagara

把"色彩语言 + 可达性"落到 Niagara 的具体决策:

  • 危险预警别只靠红。 用高 value + 高对比色 + 形状/闪烁多通道编码:危险区给强明度脉冲、清晰的边界形状、节奏闪烁,而不仅是把 Color 调红。
  • 阵营区分叠加明度/形状差异。 红队/蓝队除了 hue,再给一个稳定的明度差或剪影差异(如不同的拖尾形状),让色盲玩家也能分敌我。
  • 遵循项目 VFX 配色规范。 火/冰/电、阵营、状态都按规范的固定色走,别为了"这个更炫"自创配色破坏一致性。
  • 与场景/UI 对比。 关键信息用背景里少见的色;AOE 危险区取与地面互补的色,确保从环境里跳出来(回扣第 10、13 课)。
  • 交付前两道检查:① 去色(第 1 课的去色测试)——去掉颜色后,敌友/危险还分得清吗?② 色盲模拟——在红绿色盲滤镜下,关键信息还读得出吗?两道都过,才算"系统层面读得清"。

常见误区

  • 只用 hue 编码信息。 红敌绿友在约 8% 男性玩家、低对比、繁杂背景下都会失效。关键信息必须叠 value + 形状 + 动态。
  • 特效色与场景/UI 撞色。 危险警告和地面同色 → 看不见;特效和 UI 抢同一个色 → 信息混乱。要主动与背景/UI 拉开对比。
  • 无视项目规范自创配色。 单看更漂亮,放进系统是"说方言",破坏玩家学过的约定,整体可读性下降。
  • 以为"调成正确的红"就够了。 正确的 hue 只是第一步;没有明度/形状/对比的支撑,正确的红照样可能读不清。
