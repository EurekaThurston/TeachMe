# Working Notes — 美术基础(for VFX)

## 用户画像 / 教学偏好
- 3D 游戏特效师，主力 **Unreal / Niagara**，做实时特效(爆炸、魔法、能量、烟火等)。
- 美术理论基础**零散**:听过一些名词(互补色、冷暖)但串不起来、用不熟。
- 动手型创作者，有强 VFX 直觉:发光/自发光、叠加混合、渐变、HDR、颜色曲线、时间曲线。用这些作类比桥梁。
- **中文授课**;课程 HTML 内文用中文，美术术语保留英文原文 + 中文解释(value、hue、chroma 等)。
- 目标三件套:① 特效在游戏里**读得清**(第一优先) ② 更好看/更有冲击力 ③ 能**评判并指导**他人作品。

## 教学手法(本工作区约定)
- **课程深度(用户明确要求,2026-06-21):每课要"深入、量大但不堆重复"。** 比技能默认的"极短"更厚:充分的概念讲解(带引用)+ 交互 demo + Niagara 应用 + 较多测验。但仍**单课单核心**,避免一课塞多概念造成超载。
- 用户希望**一次多出几节课**,尽量把"现阶段目标所需的全部章节"产出,他慢慢看 → 见 `SYLLABUS.html`(完整 15 章路线图)。
- 偏**交互 + 即时反馈**:去色/眯眼测试、看图判断、配色判断,而非纯阅读。
- 把每个美术概念**对应到 Niagara 的具体操作**(颜色曲线、Emissive 强度、渐变、Sprite 明度)。
- 教学顺序遵循经典法则:**Value first, color second**(明度是地基,色彩是上层)。
- 共享组件:`assets/styles.css`(Tufte 风格 + 美术调色板,**所有课共用、勿改**)、`assets/quiz.js`(等长选项、即时反馈测验)、`assets/value-tools.js`(去色切换/明度计算)。新课一律复用,不内联重复。新交互组件按模块命名放进 `assets/`(如 `light-tools.js`、`color-tools.js`),避免文件冲突。
- **JS 字符串坑(必查):** 测验文案里的引号必须用中文「」或弯引号"",**严禁 ASCII 直引号**(会截断 JS 字符串、整段测验失效)。出课后用 node 跑 `new Function` 验证语法。
