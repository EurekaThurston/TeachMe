std::string 与常见标准库类型

代码里的文字、和"名字 → 数据"的查表,在 C++ 里长什么样。这正是蓝图里 String 变量和 Map 变量背后的东西。

本课目标(一个收获) 读懂代码里的文本(std::string)和常见容器(std::pair / std::map),并理解 std:: 前缀就是在说"这东西来自标准库"。

热身 · 回忆第 9 课(别翻笔记)

先凭记忆答一题,把上次 std::vector 的东西捞回来——主动回忆比重看一遍记得牢。

std:: 是什么:一个"标准库工具架"

你已经在第 9 课见过 std::vector 了。每次看到 std:: 这个前缀,先在心里翻译成一句话:"这是 C++ 标准库(Standard Library)提供的现成工具。" std 念作 "standard"。冒号那两个点 :: 叫作用域解析(scope resolution),意思是"vector 这个名字属于 std 这个命名空间(namespace)"。

命名空间的完整机制留到第 14 课细讲,这里你只需要建立直觉:看到 std:: 不要慌,它不是某个项目自己写的怪东西,而是全世界 C++ 都通用的标准件。

std::string:代码里的"文字"

std::string 就是一段文本——一串字符,比如 "Health"、"BaseColor"、"render_high.exr"。在 VFX 里这太常见了:参数名 / 变量名("Health"、"Velocity"、"BaseColor")、文件路径、资产名、通道名,全是字符串。(Unreal 里你还会看到 FString / FName 这两个引擎自己的字符串类型,概念和 std::string 一致。) 

#include <string>   // 用 std::string 前要包含这个头文件

std::string paramName = "Health";   // 一个参数名
std::string channel   = "diffuse";  // 一个通道名

它比 C 的 char* 安全好用

你在老代码里还会撞见另一种写法:char*(或 const char*),这是 C 语言时代的字符串——本质是"一串字符的裸地址",长度要自己数到结尾的 '\0',拼接、求长度都得手动且容易出错。std::string 把这些麻烦全包好了:它自己记得有多长、自己管理内存、能直接用 + 拼接。读现代 C++ 时,默认你看到的文本就是 std::string。

三个你一定会看到的操作

std::string base = "render";
std::string ext  = ".exr";

std::string file = base + "_v01" + ext;  // + 拼接 → "render_v01.exr"

int n = file.size();                       // .size() 给出字符个数(这里是 14)

const char* raw = file.c_str();              // .c_str() 转回老式 C 字符串
  • + 拼接:把两段文字接成一段,像把字符串"焊"在一起。
  • .size():返回里面有几个字符(也有等价的 .length())。注意它是个成员函数,要带括号。
  • .c_str():把 std::string 交还成老式 const char*。为什么需要它? 很多老 C API(比如某些渲染器、文件库的接口)只认 const char*,这时就用 .c_str() 把现代字符串"翻译"给它们用。

std::pair:一对绑在一起的值

std::pair<A, B> 就是把两个值打包成一对。尖括号里的 A、B 是这一对里两个成员各自的类型(这种 <...> 模板写法你在 std::vector<float> 里已经见过)。取值用 .first 和 .second: 

#include <utility>

std::pair<std::string, float> entry = {"density", 0.8f};

std::string name  = entry.first;    // "density" —— 第一个成员
float       value = entry.second;   // 0.8     —— 第二个成员

把它想成一个只有两栏的小卡片:左边一个名字、右边一个值。记住 .first / .second 这两个名字,你下面读 std::map 的迭代时立刻用得上。

std::map:名字 → 数据 的查表

重头戏。std::map<Key, Value> 是一张查找表:给它一个键(Key),它还你对应的值(Value)。这正是蓝图里 Map 变量的样子(C++ 里对应的引擎类型是 TMap)——你拿一个名字 "Health" 去查,它还你对应的那份数据。

读法 读到 std::map<std::string, T> 时,脑子里直接浮现"名字 → 数据 的查表":键是一个名字字符串,值是它对应的数据。这跟你在蓝图里建一个 Key 为 String、Value 为某类型的 Map 变量是同一件事。 
#include <map>
#include <string>

std::map<std::string, int> paramSize;  // 参数名 → 每个参数占几个 float

paramSize["Position"] = 3;   // 位置是 3 个 float (x,y,z)
paramSize["Normal"]   = 3;   // 法线也是 3 个
paramSize["Color"]    = 3;   // 颜色 RGB,3 个
paramSize["UV"]       = 2;   // UV,2 个

int s = paramSize["Position"];   // 用名字查:s == 3

用 [] 取值,vs. 用 .find() 安全地问"在不在"

m["P"] 是最直观的取值方式:方括号里放键。但有个读代码必须知道的坑:如果这个键不存在,[] 会悄悄新建一条默认值的条目。想"只查询、不创建"时,代码会改用 .find(): 

auto it = paramSize.find("Color");   // 去表里找 "Color"

if (it != paramSize.end())          // .end() 表示"没找到"的边界标记
{
    int s = it->second;             // 找到了:it->second 就是对应的值 3
}
else
{
    // 没有 "Color" 这一项
}

这里 it 指向表里的一个条目,而每个条目本身就是一个 std::pair:it->first 是键("Color"),it->second 是值(3)。看,前面学的 .first / .second 在这儿接上了。(it->second 的箭头 -> 留到后面讲指针时细说,现在只当它是"取这个条目的 second"。)

迭代:把整张表走一遍

for (const auto& kv : paramSize)   // kv 依次是表里的每个 pair
{
    std::string name = kv.first;   // 键:参数名
    int         size = kv.second;  // 值:占几个 float
}

这种 for (... : 容器) 的写法叫范围 for(range-based for):不用管下标,直接"对容器里的每个元素轮一遍"。每轮拿到的 kv 又是个 pair,所以照样 .first / .second。

你会在真实代码里看到

把今天三样东西拼起来,这就是一个"按名字查数据"的函数——和你在引擎、插件代码里读到的形状几乎一致:

#include <string>
#include <map>
#include <vector>

// 一张"参数名 → 这个参数的一串值"的表
using ParamTable = std::map<std::string, std::vector<float>>;

// 给定参数名,取出那一项的数据;没有就返回一个空 vector
std::vector<float> getParam(const ParamTable& params, const std::string& name)
{
    auto it = params.find(name);       // 用名字去查表
    if (it != params.end())
        return it->second;              // 找到 → 返回那份数据

    return {};                         // 没找到 → 返回空 vector
}

一眼读出它的"合同":喂进去一张参数表和一个参数名字符串,吐出来一个 std::vector<float>(那一项的数值)。using ParamTable = ... 只是给那个又长又啰嗦的 map 类型起了个别名,方便人读——真实代码很爱这么干。

头文件,先点一下 用 std::string 要 #include <string>,用 std::vector 要 <vector>,用 std::map 要 <map>。#include 就是"把这个工具的说明书先搬进来"。它的细节(头文件到底是什么)留到第 13 课;现在你只要会认:文件顶部那几行 #include <...> 告诉你这段代码用到了哪些标准库工具。

读代码易错点

  • m["P"] 不是"只读"——它在键不存在时会偷偷新建条目。 读到对一张 map 反复用 [] 的代码,要留神它可能正在往表里加东西,而不只是查询。想纯查询看 .find()。
  • .size() / .first / .second 别和函数调用混淆。 .size() 带括号(是个成员函数);.first / .second 不带括号(是成员变量,直接取)。多写或漏写括号都会读错。
  • 看到 .c_str() 不是"在改字符串",而是"在交给老 C API"。 它把 std::string 转成 const char* 喂给只认老式字符串的接口,常出现在调用第三方库的那一行。
  • it != m.end() 是"找到了吗"的标准判断。 .end() 是一个"过了末尾"的边界标记,不是最后一个元素。find 没找到时就返回 .end(),所以判 != end() 等于在问"到底有没有这条"。
  • 字符串的双引号 vs. 字符的单引号。 "P"(双引号)是一段 std::string 文本;'P'(单引号)是单个 char 字符。两者不是一回事,看错了类型就会跟着读错。
