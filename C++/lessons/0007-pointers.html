C++ for VFX TAs · 第 7 课

指针 pointers —— 一个存"地址"的变量

这是整门课最硬的一堵墙。翻过去,你就能读懂 Houdini HDK、Unreal、各种渲染器里那些满屏的 * 和 ->。慢慢来,这一课值得多花时间。

本课目标(一个收获) 你能读懂 T* 指针、& 取地址、* 解引用、nullptr,说清指针和引用的区别,并且分得清读真实 DCC/HDK 代码绕不开的 . 与 ->。读完这一课,你看真实 C++ 不会再被那一堆符号劝退。

热身 · 回忆第 6 课(别翻笔记)

上一课讲的是引用 references 和 const&。先凭记忆答一题,把它捞回来——等下你会发现指针和引用是一对"近亲",先把引用想清楚,指针就好懂一半。

先从一个你已经懂的画面出发

在 Houdini 里,一根连线从来不"装"数据。它只是告诉下游节点:"你要的几何在那个 SOP 的输出口。"数据本体待在那个节点里,连线只是指向它的一根线。你顺着这根线,就能去拿、去改那份几何。

指针(pointer)就是这根线。 一个普通变量 float mass = 7.0; 自己装着值 7.0;而一个指针变量不装值,它装的是"那个值在内存里的地址"——相当于一张写着门牌号的纸条。纸条本身不是房子,但顺着门牌号你能找到房子。

VFX 类比 普通变量 = 节点里那份真正的几何数据。指针 = 指向那份数据的一根连线(它只记着"数据在哪")。后面要讲的 ->,就是"顺着这根连线钻进去,拿里面的东西"。

三个符号,一次说清:T* / & / *

指针其实只有三个动作,对应三个符号。我们一个一个拆。

① 声明一个指针:T*

在类型后面加一个 *,就是"指向 T 的指针"。int* 读作"指向 int 的指针",float* 读作"指向 float 的指针"。

int count = 42;        // 普通变量:自己装着 42
int* ptr;             // 一个指针:能装"某个 int 的地址",现在还没指向谁

注意:int* 整体是类型。* 贴在类型那边还是变量那边,在 C++ 里都合法(int* ptr 和 int *ptr 一样),读代码时把它看成"这是一个指针"就好。

② 取地址:&x

在一个变量前面放 &,意思是"给我这个变量的地址"。这正好能喂给指针。

int count = 42;
int* ptr = &count;     // ptr 现在指向 count(装着 count 的地址)
最容易混的一点 —— 务必看懂 & 这个符号在两个位置意思完全不同,这是第 6 课和这一课最大的坑:
● 写在类型声明里,如 int& r = count; —— 这里 int& 是"引用类型"(上一课的内容),r 是 count 的别名。
● 写在表达式里、变量前面,如 ptr = &count; —— 这里 & 是"取地址运算符",意思是"拿 count 的地址"。
判断诀窍:& 紧跟在类型后面(int&)→ 引用;& 出现在等号右边、贴着一个变量名(&count)→ 取地址。

③ 解引用:*ptr

在一个指针前面放 *,意思是"顺着这个地址,去拿/改那里的值"。这叫解引用(dereference)。

int count = 42;
int* ptr = &count;     // ptr 指向 count

int x = *ptr;          // 解引用:顺着地址拿值 → x 变成 42
*ptr = 100;             // 解引用后赋值:改的是 count 本身 → count 现在是 100

留意:* 又是个"一符两义"的符号。在声明里(int* ptr)它表示"这是指针";在表达式里(*ptr)它表示"解引用、去拿值"。和 & 一样,看它出现在类型旁边还是表达式里。

把三件事连起来记:& 把"值"变成"地址"(进),* 把"地址"变回"值"(出)。它俩是一对反操作。

nullptr:一根没接到任何地方的线

一个指针可以"什么都不指"。这种空指针在现代 C++ 里写作 nullptr。它就像一根悬空、没插进任何端口的连线。

int* ptr = nullptr;    // 明确表示:现在什么都不指

if (ptr != nullptr)    // 读真实代码常见:先检查"线接上了吗"
{
    *ptr = 5;            // 确认非空,才敢顺着它去改值
}

为什么要有空指针?因为很多 API 会用"返回 nullptr"来表示"没找到 / 失败了"。比如"按名字找一个节点",找不到就还你一个 nullptr。所以真实代码里到处是"先判空,再解引用"的套路——这点下面"易错点"会重点讲。

指针 vs 引用:这对近亲到底差在哪

第 6 课的引用和这一课的指针都能"指向别的变量、并改它"。但它俩有三条关键区别,读代码时要分得清:

问题引用 int&指针 int*
能为空吗?不能,必须绑定一个东西能,可以是 nullptr
能改指向吗?不能,一旦绑定就锁死能,随时换指别的变量
取里面的值怎么写?直接用名字,就像本体要先 * 解引用

一句话记住:引用是"焊死的别名"(必须接、不能换、用起来像本体);指针是"可插拔的线"(可以悬空、可以改插口、用前常要先解引用或判空)。正因为指针能为空、能改指向,API 设计者爱用它来表达"可有可无"和"动态查找"。

你会在真实代码里看到:HDK 的几何指针

下面这段是示意性的 Houdini HDK 风格代码(真实头文件更复杂,这里框出来只为让你认得这个形状)。一个 SOP 节点的核心函数,常常长这样:

// 示意:一个 SOP 在 cook 时,Houdini 把"指向几何的指针"给你
GU_Detail* buildGeometry(GU_Detail* gdp)
{
    if (gdp == nullptr)        // ① 先判空:线接上了吗
        return nullptr;

    GA_Offset pt = gdp->appendPoint();   // ② 顺着指针调它的方法:加一个点
    gdp->setPos3(pt, UT_Vector3(0, 1, 0));  // ③ 再顺着指针设这个点的位置

    return gdp;                  // ④ 把这根指向几何的线还回去
}

读法:GU_Detail* 就是"指向一份几何细节(点/面/属性)的指针"。Houdini 不会把整份几何复制给你(太大了),而是给你一根指向它的线,你顺着线就地操作。函数最后 return gdp; 又把这根线还回去。这就是为什么 HDK、Unreal、渲染器 API 满屏都是 *:大对象都用指针传来传去,只传地址、不搬数据。

关键招式:. 和 -> 的区别(读 DCC 代码绕不开)

对象有成员(属性和方法)。怎么取成员,取决于你手里是对象本身还是指向对象的指针:

GU_Detail  geo;          // geo 是对象本体
GU_Detail* gdp = &geo;   // gdp 是指向它的指针

geo.appendPoint();        // 对象/引用 → 用点  .
gdp->appendPoint();        // 指针       → 用箭头 ->
(*gdp).appendPoint();     // 完全等价:先 * 解引用,再用点 .

p->member 就是 (*p).member 的简写。 因为"先解引用再取成员"太常用,C++ 给了 -> 这个快捷写法。所以记忆口诀很简单:
● 手里是对象或引用(没有 * 的那种)→ 用 .
● 手里是指针(类型带 * 的那种)→ 用 ->

VFX 类比 -> 这个箭头,就是"顺着连线钻进那个节点,去拿/调它里面的东西"。你手里只有一根线(指针),想用里面的功能,就得沿着箭头进去。手里直接就是节点本体(对象),那就用 . 当场取。

读代码易错点(这一节最值钱)

① 对 nullptr 解引用 = 崩溃。 一根悬空的线,你非要顺着它去拿东西,程序当场炸(crash)。所以真实代码里 if (p != nullptr) / if (p) 这种判空随处可见——它不是啰嗦,是保命。读代码时看到判空,就知道"作者在防这根线没接上"。

GU_Detail* gdp = nullptr;
gdp->appendPoint();    // ✗ 危险!顺着一根空线去操作 → 崩溃

② & 的两副面孔,别认错。 同一个 &,在 int& r(类型旁)是"引用",在 &count(变量前)是"取地址"。看位置,不看符号长相。

③ * 的两副面孔,也别认错。 int* p(类型旁)是"这是指针";*p(表达式里)是"解引用拿值"。同样看位置。

④ 改指针 ≠ 改它指的值。 p = &other; 是"让线改插到 other 上";*p = 5; 才是"顺着线把那头的值改成 5"。少一个 *,意思天差地别。
