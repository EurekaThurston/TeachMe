循环 —— 逐点处理的引擎

几何代码 99% 的时间在干一件事:对一堆点、一堆图元,各做一遍同样的处理。看懂循环,你就看懂了 HDK 的主干。

本课目标(一个收获) 读懂 for / while / range-based for 循环,能说出一个循环跑几次、对哪些数据各做一次什么——这是几何代码的核心模式。

热身 · 回忆第 4 课(别翻笔记)

先凭记忆答一题,把上次"运算符"的东西捞回来——主动回忆比重看一遍记得牢。

为什么循环就是 VFX 代码的全部

你在 Houdini 里丢一个 Point Wrangle,写一句 @P.y += 1;,然后它就对每一个点各跑一遍。你从没写过"遍历所有点"这件事——Houdini 帮你藏起来了。 而在 C++ / HDK 里,这个"对每个点跑一遍"是你要亲眼看到的代码:它就是一个循环(loop)。

循环只做一件事:把同一段代码重复执行很多次。读循环时,你脑子里要回答两个问题—— (1) 这段代码会跑几次?(2) 每一次它在对哪个数据做什么? 答出这两点,这个循环你就读懂了。

VFX 类比 for 循环 = Houdini 对每个点跑一遍 wrangle。循环的"次数"= 点的数量;循环体(花括号里那段)= 节点对单个元素做的那点事。Wrangle 把循环藏了,C++ 把它摊在你面前——仅此而已。

while 循环:最朴素的"条件成立就一直跑"

while 是最容易看懂的循环。结构就一句话:只要括号里的条件为真,就把花括号里那段再跑一遍。条件一旦变假,循环结束,代码继续往下走。

int i = 0;                 // 计数器,从 0 开始
while (i < 5)               // 闸门:i 还小于 5 吗?
{
    process(i);             // 循环体:对 i 做点事
    i = i + 1;             // 不更新 i 的话会永远卡在这里!
}

手跑一遍:i 依次是 0,1,2,3,4,每次都执行一遍 process(i);当 i 变成 5 时,5 < 5 为假,循环结束。 所以它跑了5 次,分别对 0,1,2,3,4 各做一次。注意:循环体里必须有某行让条件最终变假(这里是 i = i + 1),否则就是死循环,程序永远出不来。

do-while:先跑一次,再问条件

do-while 是 while 的孪生兄弟,唯一区别:它先执行一遍循环体,再检查条件。所以不管条件真假,循环体至少跑一次。读代码时看到 do { ... } while (...),记住"先做后问"即可。 

do
{
    askUser();              // 至少执行一次
} while (notConfirmed);     // 条件在最后才判断

经典 for 循环:三段结构与执行顺序

for 是几何代码里最常见的循环。它把"计数器初始化、判断条件、更新计数器"这三件事打包写在一行括号里,用两个分号隔开,所以特别紧凑。括号里三段的顺序是固定的: 

for (int i = 0;   i < n;   i = i + 1)
//   └─① 初始化─┘  └②条件┘  └─③ 更新──┘
{
    process(i);            // ④ 循环体
}

这三段加上循环体,执行顺序不是从左到右那么简单,务必记牢这条时间线:

① 初始化  ── 只在最开头跑一次(int i = 0)
② 条件    ── 每轮开始先判断;为真才进循环体,为假就退出
④ 循环体  ── 跑你那段代码(process(i))
③ 更新    ── 循环体跑完后才更新(i = i + 1),然后回到 ②

所以一轮的节奏是 ②→④→③→②→④→③…,直到 ② 为假。上面这个 for 跟前面那个 while 完全等价——只是把"初始化 / 条件 / 更新"三处收拢到了一行,读起来一眼就知道这是个数着数跑的循环。这也是为什么 C++ 代码里数组、点列表的遍历几乎都用 for。

range-based for:最像 VEX 的那种"自动逐点"

经典 for 要你管计数器 i、还要用 i 去取第 i 个元素。但很多时候你根本不在乎序号,只想"把这一堆东西挨个拿出来处理一遍"。这时就用 range-based for(范围 for): 

for (auto& p : points)     // 对 points 里的每个元素,起名叫 p
{
    p.y += 1.0f;            // 直接对这个元素做事,不用管它是第几个
}

读法:"对 points 里的每一个元素,把它叫做 p,然后执行花括号里那段"。冒号 : 右边是那一堆数据,左边是每次拿出来的那一个。 这跟 VEX 里 @P.y += 1; 自动对每个点跑一遍,是同一个心智模型——只不过 C++ 把"遍历哪一堆"明写成了 : points。

那个 auto& 先眼熟:auto 是"类型让编译器自己猜"(下一课讲引用 &,它让你真正改到原数据而不是改一份拷贝)。本课你只要能读出"逐个遍历 points"这层意思就达标了。

VFX 类比 range-based for 就是 wrangle 里的 @P 那种"自动逐点":你写的代码是针对单个点的(p.y += 1),框架负责把它套到每个点上。区别只在于:VEX 帮你隐藏了 : points,而 C++ 让你看见你正在遍历哪一堆几何。

break、continue 与嵌套循环

循环里有两个能改变流程的关键词,读真实代码经常撞见:

  • break:立刻跳出整个循环,后面几轮都不跑了。像找东西"找到了就收工"。
  • continue:跳过本轮剩下的,直接进下一轮。像"这个点不符合条件,略过,看下一个"。
for (int i = 0; i < n; i = i + 1)
{
    if (mask[i] == 0) continue;   // 这个点被屏蔽 → 跳过,处理下一个
    if (found)         break;      // 已经找到 → 整个循环收工
    process(i);
}

循环还能套循环(嵌套):外层每跑一轮,内层就整个跑一遍。比如对每个图元(外层),再遍历它的每个顶点(内层)。读嵌套循环时,把内层当成"外层循环体里的一个完整动作"看即可——别被两层 i、j 吓到。 

for (int prim = 0; prim < numPrims; prim++)        // 外层:每个图元
{
    for (int v = 0; v < vertsOf(prim); v++)      // 内层:该图元的每个顶点
    {
        shadeVertex(prim, v);
    }
}

(i++ 只是 i = i + 1 的简写,以后会大量见到,先眼熟。)

你会在真实代码里看到

下面这段是 HDK / Houdini 几何代码里最高频的样子:遍历一个几何体的所有点,把每个点的位置抬高。std::vector<UT_Vector3> 你现在只要读成"一排位置(vector 是个能装很多元素的容器,下一课细讲)";.size() 读成"里面有多少个"。 

// positions 是这块几何所有点的位置,排成一长串
std::vector<UT_Vector3> positions = mesh.getPositions();

// 经典 for:用序号 i 一个个取出来处理
for (size_t i = 0; i < positions.size(); i++)
{
    positions[i].y += 1.0f;     // 把第 i 个点抬高 1
}

// 同样的事,range-based for 写法更短、更像 @P
for (UT_Vector3& P : positions)
{
    P.y += 1.0f;             // 对每个点 P,抬高 1
}

看到这种结构,你的脑内独白应该是:"它在遍历所有点,对每个点的 y 加 1。跑的次数 = 点的数量。" 真实的 HDK 还常写成 for (GA_Offset ptoff : range) 这种用偏移量遍历点的形式——名字不同,但骨架完全一样:冒号右边是"一堆点",左边是"当前这一个"。认出骨架,你就能读懂八成几何循环。

读代码易错点

  • off-by-one(差一错误)。 i < n 跑 0..n-1 共 n 次;但 i <= n 会跑 0..n 共 n+1 次,多跑一次、还可能越界访问 positions[n](不存在的第 n+1 个)。读到 <= 时要警觉:是不是多跑了一轮?
  • 死循环。 while / for 的条件如果永远为真(比如忘了更新计数器,或更新方向写反),程序就卡死。读 while 时一定要找:哪一行会让条件最终变假? 找不到就是 bug。
  • == 写成 =。 循环条件里要比较用 ==;写成赋值 = 会改变行为(第 4 课提过)。
  • range-based for 里漏了 &。 for (auto p : points)(没有 &)拿到的是每个元素的拷贝,你改 p 改的是副本,原数据纹丝不动;for (auto& p : points)(有 &)才真正改到原数据。下一课专门讲这个 &,现在先把它当一个"改没改到原件"的开关记住。
