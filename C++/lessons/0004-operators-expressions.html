运算符与表达式

代码里 99% 的"在算什么",都藏在一行行表达式里。读懂运算符,你就读懂了计算本身。

本课目标(一个收获) 读懂代码里的算术、比较、逻辑与三元表达式,并避开两个经典陷阱:整数除法(5 / 2 居然等于 2)和 = 与 == 混淆。看到一行 a * b + c > 0 ? x : y 你能不慌、能拆。

算术运算符:和你想的几乎一样,除了一个坑

五个基本算术运算符你早就认识:+ 加、- 减、* 乘、/ 除,外加一个 % 取模(求余数)。 它们组合出来的东西叫表达式(expression)——一段"能算出一个值"的代码。 

int   total   = count * 4 + 1;     // 先算乘 count*4,再加 1
float area    = width * height;     // 两个 float 相乘,结果还是 float
int   leftover = 17 % 5;            // 取模:17 除以 5 余 2 → leftover = 2

%(取模 / modulo)对你这种做索引数学的人特别有用:它给你"除法剩下的余数"。 比如在一圈点上循环取色、做棋盘格、每 N 个做一次事——全靠它。

怎么想 i % count 就是"循环取索引":当 i 一路涨上去,i % count 会在 0, 1, 2, …, count-1, 0, 1, 2, … 之间周而复始,永远落在 0 .. count-1 这个范围里。HLSL 里也是同一个 %。

整数除法陷阱:本课最值钱的一节

这是 C++(以及 HLSL、几乎所有类 C 语言)里最常坑到 TA 的一个点。规则只有一句话,但后果很大:

记住这条 当 / 两边都是整数 int 时,做的是整数除法——结果会把小数部分直接丢掉(向零截断),而不是四舍五入。 
5 / 2      // == 2   (不是 2.5!小数 .5 被直接砍掉)
7 / 2      // == 3   (3.5 截断成 3,不是四舍五入的 4)
1 / 2      // == 0   (0.5 截断成 0 —— 很多人第一次被这个坑)

5.0 / 2    // == 2.5 (只要有一边是 float,就走浮点除法,小数保留)
5 / 2.0    // == 2.5 (同上,2.0 让整个表达式变成浮点)

为什么是这样?因为 int / int 的结果类型还是 int,而 int 装不下小数,只能截断。 要拿到 2.5,你得让至少一边是浮点数:把字面量写成 5.0,或者把某个变量转成 float。 （LearnCpp 6.2 把这条讲得很细。) 

int   a = 5, b = 2;
float bad  = a / b;              // 先算 5/2=2(整数除法!),再存进 float → 2.0,坑!
float good = (float)a / b;       // 先把 a 变成 5.0,再除 → 2.5,对了

注意 bad 那行:就算你把结果存进 float,也救不回来——因为截断发生在除法那一刻,等存进 float 时小数早没了。 读代码时看到 int / int,你应该条件反射地警觉:"这里会不会丢小数?"

复合赋值与自增自减:同一件事的简写

你会大量看到 += 这类复合赋值(compound assignment)。它们只是简写,等号右边那串先算完,再写回左边的变量: 

x += 3;     // 等同于 x = x + 3;
x -= 1;     // 等同于 x = x - 1;
x *= 2;     // 等同于 x = x * 2;   (翻倍)
x /= 4;     // 等同于 x = x / 4;

还有更短的:++(自增,加 1)和 --(自减,减 1)。 

i++;     // 等同于 i += 1;  即 i = i + 1;
i--;     // 等同于 i -= 1;

++ 可以写在变量前面(++i,前缀)或后面(i++,后缀)。 单独成一行时,两者效果一模一样,都是把 i 加 1。差别只在它们当作一个值被用到时:

  • 前缀 ++i:先加 1,再把加完的值拿去用。
  • 后缀 i++:先把原来的值拿去用,之后才加 1。
int i = 5;
int a = ++i;   // 先加 → i 变 6,a 拿到 6
int j = 5;
int b = j++;   // 先用 → b 拿到 5,之后 j 才变 6

读代码阶段你不必纠结这个微妙差别——绝大多数你看到的 i++ 都是循环里单独一句,就理解成"i 加 1"即可。知道有前后缀之分、别被它吓到,就够了。

比较与逻辑:把多个条件拼起来

比较运算符你在第 2 课见过,这里快速回顾,它们算出来的结果是 bool(true/false):

写法意思
a < b / a > b小于 / 大于
a <= b / a >= b小于等于 / 大于等于
a == b相等(两个等号!)
a != b不相等

要把多个条件组合起来,用逻辑运算符:

  • &&(逻辑与 / and):两边都真才真。
  • ||(逻辑或 / or):有一边真就真。
  • !(逻辑非 / not):把真假翻过来。
if (age >= 18 && hasLicense)        // 成年「并且」有执照,两者都要满足
if (isHidden || opacity == 0)       // 被隐藏「或者」透明度为 0,任一即可
if (!isVisible)                     // 「不」可见(等同于 isVisible == false)

一个值得知道的细节叫短路求值(short-circuit):对于 &&,如果左边已经是 false,右边根本不会算(反正结果注定是 false);对于 ||,左边已经 true 时右边也跳过。这常被用来"先检查安全、再访问": 

if (ptr != nullptr && ptr->count > 0)   // 左边为假就短路,不会去碰右边的 ptr->count

三元运算符:一个迷你 if,真实代码里满地都是

这个你一定会反复遇到,所以单独讲。三元运算符(ternary / conditional operator)写法是: 

条件 ? 真时的值 : 假时的值

它整体算出一个值——条件为真就取 ? 后面那个,为假就取 : 后面那个。等价于一个超浓缩的 if/else: 

int bigger = (a > b) ? a : b;     // a 大就取 a,否则取 b —— 等于求两者最大值

// 完全等价于下面这段,只是它压成了一行:
int bigger;
if (a > b) bigger = a;
else      bigger = b;

如果你写过 HLSL,这眼熟得很——HLSL 里的 ?: 是同一个东西。 （LearnCpp 6.6 专讲它。）读代码时,把 cond ? x : y 在心里读成"如果 cond,就 x,否则 y"即可。

你会在真实代码里看到

下面这段是 Unreal / 渲染器风格里很典型的写法:遍历一组元素、按高度算个混合权重、再用三元决定走哪条分支。逐行的注释就是你"读代码"该有的内心独白: 

// Unreal 风格:遍历一组高度值,按高度做颜色混合
for (int32 i = 0; i < NumPoints; ++i)
{
    float h   = Heights[i];
    float t   = (h - MinH) / (MaxH - MinH);     // 归一化到 0..1(注意:这些都是 float,安全)
    float mix = t < 0.5f ? t * 2.0f : 1.0f;       // 三元:低于一半就线性拉伸,否则封顶到 1

    int swatch = i % NumColors;                 // 取模:在调色板里循环取色,永不越界
    Colors[i] = Palette[swatch] * mix;
}

看懂这一段,你就具备读真实 Unreal / 引擎代码的基本功了:它无非是算术 + 取模 + 三元 + 比较的组合。每个零件你现在都认识。

够用的运算符优先级

"先算谁"基本和小学数学一致,记住两条就够读代码:

  • 乘除模 * / % 先于 加减 + -(和数学一样)。
  • 算术 先于 比较 < > == ,比较 又先于 逻辑 && ||。
a + b * 2 > 10 && flag

// 拆开实际是这样优先:
((a + (b * 2)) > 10) && flag
//      ①乘    ②加  ③比较     ④逻辑与(最后)

不确定时,作者通常会加括号讲清楚——你读到括号,就照括号里先算。你不需要背完整的优先级大表,只要知道"乘除先、比较中、逻辑后",剩下交给括号。

读代码易错点

  • = 不是 ==。 单个 = 是赋值(把右边塞进左边),两个 == 才是比较是否相等。 if (x = 5) 不会"判断 x 是不是 5",而是把 5 赋给 x 然后判断 5(非零)为真——这是经典 bug,读代码时务必看清几个等号。
  • 整数除法丢小数。 见到 int / int 先警觉:1 / 2 是 0 不是 0.5。要小数就让一边是 float。
  • 浮点相等不可靠。 float 有微小精度误差,0.1 + 0.2 == 0.3 可能是 false。所以代码里很少直接 == 比两个算出来的 float,常见的是比"差的绝对值是否够小"。看到 fabs(a - b) < 0.0001 这种写法,就是在做"近似相等"。
  • 整数会溢出(概念先眼熟)。 int 能表示的范围有限,数算得太大会"绕回"成负数或乱值。日常读代码不必深究,但看到大数累加时心里有个数。
