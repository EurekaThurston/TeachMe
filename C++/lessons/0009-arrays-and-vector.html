数组与 std::vector —— 一串同类型的数据

单个 float 是一个点的属性,一串 float 是整张几何的属性数组。读懂"一串"怎么存、怎么逐个取,你就读懂了 VFX 代码的主干。

本课目标(一个收获) 你能读懂代码怎么存一串同类型的数据并逐个访问:C 风格数组 float a[3]、动态数组 std::vector<T>、下标从 0 开始、用 .size() 拿长度、用循环遍历。这正是点/属性数组在代码里的样子。

热身 · 回忆第 8 课(别翻笔记)

先凭记忆答一题,把上次"内存与生命周期"的东西捞回来——主动回忆比重看一遍记得牢。

从你已经懂的东西出发:点云就是一串数据

在 Houdini 里,一个几何体有一堆点,每个点有 ptnum(0、1、2……)和一份 P(位置)。 "所有点的 P" 合在一起,就是一个属性数组——同一类型的值,排成一长串,用编号取。 C++ 里存这种"一串同类型数据"的东西,就叫数组(array)。先看最朴素的一种。 

// C 风格数组:固定长度的一串 float
float pos[3] = { 1.0f, 2.0f, 3.0f };  // 一个 xyz 位置

float x = pos[0];   // 取第 1 个 → 1.0
float y = pos[1];   // 取第 2 个 → 2.0
float z = pos[2];   // 取第 3 个 → 3.0

读这段只需抓三点:float pos[3] 是"3 个 float 排成一串",名字叫 pos; { ... } 是初始值;pos[i] 里的方括号叫下标(subscript),用来取第几个。 注意最要命的一条——下标从 0 开始数。所以 3 个元素的合法下标是 0、1、2,没有 pos[3]。这点我们等下专门拆。

VFX 类比 一个 float pos[3] 就是一个 vector(xyz);一个装了很多位置的数组就是 Houdini 的点云 / 几何属性数组。下标 i 就是点编号 ptnum——pos[0] 是第 0 号点。遍历这个数组,就是 Wrangle 里"逐点处理"那件事。

std::vector<T>:能增长的动态数组

C 风格数组的长度在写代码时就钉死了([3] 永远是 3)。但真实几何的点数是运行时才知道的、还会变。 所以现代 C++ 代码里你看到的几乎全是 std::vector<T>——一个能自己增长的动态数组。 <T> 里的 T 是"装什么类型"(承接前面课讲的模板尖括号):std::vector<float> 装一串 float,std::vector<int> 装一串 int。 

// 一串 float,可以动态增长
std::vector<float> masses = { 1.0f, 2.5f, 4.0f };

masses.push_back(9.0f);        // 末尾追加一个 → 现在有 4 个

float first = masses[0];      // 下标访问,和 C 数组一样,从 0 开始
std::size_t n = masses.size();   // .size() 返回元素个数 → 4

你需要认得的几个成员函数(它们紧跟在 变量名. 后面):

写法意思VFX 里类似
v[i]取第 i 个元素(0 起)@P 在 ptnum = i 的点
v.size()有几个元素@numpt(点总数)
v.push_back(x)末尾追加一个Add 节点新增一个点
v.empty()是不是一个都没有判断几何是否为空

遍历:逐个走一遍这串数据

"逐点处理"在代码里就是遍历(loop over)这串数据。两种写法你都会遇到。 第一种是 range-based for(承接第 5 课的循环),最干净——它替你逐个把元素拿出来,你不用碰下标: 

// 写法 A:range-based for —— "对每个 m,执行……"
float total = 0.0f;
for (float m : masses)   // 把 masses 里每个元素依次叫做 m
{
    total += m;            // 累加;total += m 等价于 total = total + m
}
// 循环结束后 total 是所有质量之和

第二种用下标 + 计数。它更啰嗦,但你在老代码里会大量看到,而且当你需要知道当前是第几个(比如要拿 ptnum)时必须用它: 

// 写法 B:下标循环 —— i 从 0 走到 size()-1
for (std::size_t i = 0; i < masses.size(); ++i)
{
    float m = masses[i];   // 第 i 个,i 就是"编号"
    total += m;
}

盯住条件 i < masses.size():用的是 小于,不是 <=。 因为 4 个元素的合法下标是 0,1,2,3,而 size() 是 4——i 必须在到达 4 之前停。这是"从 0 开始数"带来的、贯穿全课的关键算术。

你会在真实代码里看到

VFX 代码里几乎总是 一个 std::vector 装位置 / 属性,然后一个循环逐点处理。 下面是一个把所有点沿 Y 轴抬高、并累加质心的循环。UT_Vector3 是 Houdini HDK 里的"三维向量"类型(示意),你只要知道它有 .x .y .z 三个分量: 

// points 是一整张几何的点位置数组(就是点云)
std::vector<UT_Vector3> points = getPointPositions();

UT_Vector3 sum{ 0, 0, 0 };

for (std::size_t i = 0; i < points.size(); ++i)
{
    points[i].y += 1.0f;   // 第 i 号点抬高 1 个单位(逐点变换)
    sum += points[i];        // 累加所有位置,准备求平均
}

UT_Vector3 centroid = sum / (float)points.size();  // 质心 = 总和 / 点数

读这段,在脑子里就是一句话:"对每一号点 i,改它的 Y,再把它加进 sum。" 这和你在 Wrangle 里写 @P.y += 1; 是同一件事——只是 Wrangle 帮你隐藏了那个 for 循环和那个 i,而 C++ 把它们摆在明面上。看懂这个对应,90% 的 VFX C++ 循环就不陌生了。

读代码易错点(这一节最值钱)

① 下标从 0 开始,不是 1。 一个有 n 个元素的数组,合法下标是 0 到 n-1。第一个是 v[0],最后一个是 v[n-1]。看到 v[1] 别以为是"第一个"——那是第二个。

② 越界访问(out-of-bounds)是真正的危险。 对长度为 3 的数组写 pos[3] 或 pos[10],C++ 不会报错拦你——它会去读/写那块本不属于这串数据的内存。结果是垃圾值、随机崩溃,或更糟的静默错误(这正好接上第 8 课的内存话题)。读代码时,看到下标就反问一句:它会不会 ≥ size()?

float pos[3] = { 1, 2, 3 };
float bad = pos[3];   // ✗ 越界!合法下标只有 0,1,2 —— 这是 bug

③ .size() 返回的是无符号的 size_t。 v.size() 的类型不是普通 int,而是 std::size_t——一个永远非负的整数类型。这有个经典的小坑:倒着写循环时,无符号数减不到负数,而是"绕回"成一个巨大的数,循环就停不下来。

// ✗ 危险:i 是无符号,减到 0 再 --i 会绕回成超大数 → 死循环 + 越界
for (std::size_t i = points.size() - 1; i >= 0; --i) { /* ... */ }

读代码时看到 std::size_t(或 auto i = v.size())就要警觉:这个数不可能是负的。正向循环用 i < v.size() 永远安全;真要倒序,正确写法另有讲究,看到了就来问我。LearnCpp 专门有一节讲这个"无符号长度与下标"的坑。
