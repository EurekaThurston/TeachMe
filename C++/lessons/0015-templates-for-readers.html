模板作为读者 —— 别被 <> 吓到

现代 C++ 满屏都是尖括号。好消息:作为读者,你几乎只需要看懂它们,极少需要自己写。

本课目标(一个收获) 读懂代码里的 <> 尖括号——把模板理解成"对类型通用的代码模具",把 std::vector<float> 读成"用 float 浇出来的 vector",从而能淡定读过任何模板化的 API 签名。

热身 · 回忆第 14 课(别翻笔记)

先凭记忆答一题,把上次"命名空间与 ::"的东西捞回来——主动回忆比重看一遍记得牢。

模板解决什么问题:一份逻辑,适配所有类型

想象你要写一个"取两者较大值"的函数。只为 int 写一遍?那 float 怎么办?double 又得再抄一遍? 逻辑明明一模一样,只是类型不同。为每种类型重抄一遍,既啰嗦又容易出错。 

// 笨办法:同一段逻辑,被迫为每个类型各写一遍
int    maxValue(int a, int b)       { return (a > b) ? a : b; }
float  maxValue(float a, float b)   { return (a > b) ? a : b; }
double maxValue(double a, double b) { return (a > b) ? a : b; }

模板(template)就是为了消灭这种重复。你写一份逻辑,把"类型"留成一个空位,让编译器在用到时自动填进去。这正是 LearnCpp 对函数模板的定位:"一个用来生成函数的模子,其中某些类型被参数化了"。

函数模板:template<typename T> 怎么读

上面三行可以塌缩成一份模板。你读到的真实代码经常长这样:

template <typename T>
T maxValue(T a, T b)
{
    return (a > b) ? a : b;
}

把它逐块翻译成人话:

template <typename T>     // 声明:下面这段代码有一个"类型占位符",我叫它 T
T        maxValue(T a, T b)   // 返回 T,两个参数也是 T —— 三处必须是同一个类型
 │                  └─ 参数 a、b 都是那个待定的类型 T
 └─ 返回类型也是那个待定的类型 T

读法只有两步:① 看第一行 <...> 里有几个占位符(这里只有一个 T);② 把 T 当成"任意一种类型"通读下去。 当别处调用 maxValue(3.5f, 9.2f) 时,编译器看到喂进来的是 float,就把所有 T 替换成 float,临时生成一份 float 版本的函数——这个动作叫实例化(instantiation)。你不用关心生成细节,只要知道:同一份模具,喂不同类型,得到不同成品。

你熟悉的一个真实对照 你在 Unreal 里写的 TArray<float> 和 TArray<FVector> 就是同一个模具喂不同类型:TArray 这份"动态数组"逻辑只写了一遍,尖括号里填 float 还是 FVector,就生成对应类型的具体容器。template<typename T> 里的 T 就是那个占位符,调用时才被填实。

类模板:std::vector<float> 是"用 float 浇出来的 vector"

第 9 课你见过 std::vector——一个能装一串同类型东西的"动态数组"。它其实是个类模板:std::vector<T> 是模具,尖括号里填什么,就浇出装什么的成品。 

std::vector<float>   points;   // 用 float 浇出来:一串浮点数
std::vector<int>     ids;      // 用 int 浇出来:一串整数(比如索引)
std::vector<FVector> positions; // 用引擎的向量类型 FVector 浇出来:一串三维向量

读这种类型时,把尖括号读成"装着…的":std::vector<float> = "装着 float 的 vector"。 std::vector 这个词单独出现没有意义——它只是模具;必须配上 <...> 才是一个能用的具体类型。

你会在真实代码里看到

读真实头文件时,模板会和别的东西叠在一起出现。先眼熟,不用全懂:

// ① 智能指针也是模板:一个"管理某类型对象的指针"模具(后面课讲)
std::unique_ptr<Mesh> loadMesh(const std::string& path);

// ② 函数模板签名:对任意类型 T 的一整段数组求和,返回 T
template <typename T>
T sumAll(const std::vector<T>& values);

// ③ 标准库函数模板:把一段区间里的最大元素找出来(你只需读懂它"对什么工作")
template <typename It>
It max_element(It first, It last);

读 ② 的心法:别陷进 T 的细节。问自己两件事就够——它对什么类型工作?(任意 T,从参数 std::vector<T> 看出来)它返回什么?(一个 T)。 于是 sumAll<float> 接收"一串 float"、返回"一个 float"。这就够你看懂调用它的代码了。

读代码易错点

易错点 ①:把 < > 误读成"小于 / 大于"。 第 4 课里 a < b 是比较;但紧贴在类型名或函数名后面的 <...> 是模板参数,不是比较。判断很简单:左边是不是一个类型名 / 模板名? 是 → 那对尖括号在"填类型",不是在比大小。

value < minVal              // 比较:value 小于 minVal 吗(第 4 课)
std::vector<float>          // 模板:用 float 实例化的 vector —— 这里不是"小于"

易错点 ②:被嵌套模板唬住。 看到 std::vector<std::pair<int, float>> 别慌,从里往外拆:

std::vector< std::pair<int, float> >
              └── 最里层:一个 (int, float) 的"二元组"对 ──┘
└────────── 外层:一串这样的对 ───────────────────────────┘

先读最内层 std::pair<int, float>(一对值:一个 int、一个 float),再把它整个当成一种类型,套回外层 std::vector<…>:于是整体是"一串这样的二元组"。一层层剥,再吓人的尖括号也会塌成一句人话。

易错点 ③:以为每个 <T> 都得自己写。 不需要。std::vector、std::unique_ptr、std::pair 这些模具都是标准库早就写好的,你只是在使用它们时填一下类型。本课目标也只是"读懂",写模板是另一回事——大多数 TA 一辈子读模板远多于写模板。

易错点 ④:把 typename 和 class 当成两回事。 在模板参数里,template<typename T> 和 template<class T> 意思完全一样,都表示"T 是个类型占位符"。老代码爱写 class,新代码偏好 typename;读到哪个都按"类型占位符"理解即可,别以为 class 就一定是指某个具体的类。
