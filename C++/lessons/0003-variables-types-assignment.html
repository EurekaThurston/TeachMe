C++ for VFX TAs · 第 3 课

变量、类型与赋值

函数体里那些不停冒出来的小盒子,是怎么被命名、被赋值、被反复用的。读真实代码绕不开这一关。

本课目标(一个收获) 读代码时,看到一行变量,你能立刻拆出三件事:它的类型、它的名字、它的初始值。并且你能分清 =(赋值)和上一课的 ==(比较),还能说出为什么真实 HDK / 引擎代码里到处是 const。

热身 · 回忆上一课(别翻笔记)

先凭记忆答一题,把第 2 课的关键点捞回来——主动回忆比重看一遍记得牢得多。

一行变量,三个零件

一个变量就是一个带名字、带类型的盒子,里面装着一个值。这跟你天天打交道的属性 attribute 几乎是一回事:点上的 @P 是一个值、@Cd 是一个值、@pscale 是一个值——每个都有名字、有类型、有当前数值。C++ 的变量就是这个东西,只是类型要明写出来。 

int pointCount = 5;
 │        │          │
 │        │          └─ 初始值:盒子一开始装什么(这里是 5)
 │        └─ 变量名:后面代码用这个名字找到这个盒子
 └─ 类型:盒子只能装什么(这里是整数 int)

末尾那个分号 ; 表示"这句话说完了"。读到 类型 名字 = 值; 这个形状,你就知道:代码在这里造了一个新盒子并塞了第一个值进去。这个动作叫声明并初始化(declaration + initialization)。再看几个真实形状: 

float  radius   = 0.5f;       // 带小数的数,0.5f 的 f 表示这是 float 字面量
double mass     = 12.0;       // 也是小数,但精度更高(下面讲)
bool   isVisible = true;       // 真 / 假,只有 true 和 false 两个值
char   axis     = 'x';        // 单个字符,用单引号
int    seed;                    // 只声明、没给初始值 —— 盒子里是垃圾值,危险

注意最后一行:只写了 类型 名字; 没有 = 值。这叫未初始化变量,盒子里装的是内存里残留的随机垃圾。读代码时看到这种,要留个心:后面如果先读它再赋值,那很可能是个 bug。

基本类型:每个盒子能装什么

类型决定了盒子的"形状"——能装什么、占多少内存。VFX 里你早就在用这些概念了,只是叫法不同。

类型装什么字面量长这样VFX 里类似
int整数,无小数5、-12、0点编号 @ptnum、计数
float小数(单精度)0.5f、3.14f@pscale、坐标分量
double小数(双精度,更准)3.14、12.0大场景累加、解算
bool真 / 假true、false勾选框、可见性开关
char单个字符'a'、'x'轴向标记、分组首字母

顺带认一下字面量(literal):它就是直接写死在代码里的那个值本身。5 是 int 字面量,3.14 是 double 字面量,0.5f 是 float 字面量(末尾的 f 是关键标记),true 是 bool 字面量,'a' 是单字符(单引号),"text" 是字符串(双引号,后面课再细讲)。读代码时,字面量是你能直接看到的"具体数",变量则是会变的"盒子"。

= 是赋值,不是相等(承接第 2 课)

这是新手读代码最容易栽的地方,务必钉死。第 2 课里 ==(两个等号)是提问:"这两个相等吗?"算出来是 true 或 false。而 =(一个等号)是命令:"把右边的值塞进左边的盒子。" 

int x = 5;        // 赋值:把 5 装进 x。这是一个动作
x = 10;           // 再赋值:现在 x 盒子里是 10,旧的 5 被覆盖了

if (x == 10)      // 比较:问 x 现在是不是等于 10 → 这里是 true
{
    // ...
}

读代码时的判断诀窍:一个等号 = 出现在普通语句里 → 是在改盒子的值;两个等号 == 出现在 if / 条件里 → 是在问真假。看到 x = 10; 单独成行,它做的事是"覆盖",不是"判断"。

作用域 scope:盒子的活动范围就在 { } 里

一个变量并不是造出来就永远存在。它的"生命"被一对花括号 { } 框住——这叫作用域(scope)。变量在哪个 { } 里声明,就只在那个 { } 闭合之前有效;出了这对括号,这个名字就不存在了。 

float computeArea(float r)
{
    float pi = 3.14159f;     // pi 在这个函数的 { } 里诞生
    float area = pi * r * r;  // 这里能用 pi,没问题
    return area;
}                              // 函数的 } 一闭合,pi 和 area 都消失
// 在这一行写 pi 会报错:它已经不在作用域里了

为什么读代码时要在意作用域?因为同名变量在不同 { } 里是不同的盒子。你在一个函数里看到的 i,跟另一个函数里的 i 毫无关系。搞清楚"这个名字属于哪个 { }",你才不会把两段不相干的逻辑串在一起读。

VFX 类比 把一对 { } 想成一个 Houdini 的 Subnetwork / Compile Block:你在里面 spare 出来的中间属性,出了这个网络就不可见了。变量作用域就是这个"局部可见"的边界——里面的盒子,外面够不着。

const:只读盒子,真实代码里随处可见

const 是 "constant(常量)" 的缩写。在类型前面加上 const,意思是:这个盒子一旦装好值,就不准再改了。谁要是后面想给它重新赋值,编译器直接报错、拒绝编译。 

const float gravity = 9.8f;   // gravity 是只读的
gravity = 10.0f;                 // 编译错误!不能改一个 const 的值

为什么 HDK / 虚幻引擎 / 渲染器的代码里 const 多到像噪点一样?三个原因,读代码时都用得上:

  • 表达意图。const 在喊话:"这个值只是拿来读的,我不会动它。"你读到它,就知道这个盒子在下文不会被偷偷改掉,可以放心。
  • 保护输入。函数参数常写成 const,表示"我只看你的数据,不会改你的"。
  • 性能。大块几何数据(几百万个点)按 const 引用传进函数,既避免拷贝、又承诺不改原件。这正是 HDK 里 const GU_Detail * 满天飞的原因。
VFX 类比 const 就像 Houdini 节点上的一个只读输入端口:数据从这个口流进来给你用,但你这个节点不能往回写、改不了上游。看到 const,就把它读成"只读输入,放心用,但别指望它变"。

auto:让编译器替你填类型

有时你会看到 auto 顶在类型的位置上。它不是某种新类型,而是对编译器说:"右边是什么类型,这个盒子就是什么类型,你自己推。" 

auto count  = 5;        // 右边是 int 字面量 → count 被推成 int
auto radius = 0.5f;     // 右边是 float 字面量 → radius 被推成 float
auto ok     = true;     // 右边是 bool → ok 被推成 bool

读代码时遇到 auto,别慌:把目光移到 = 右边,右边那个表达式的类型,就是这个变量的类型。在真实代码里 auto 常用来接那些类型名字又长又啰嗦的返回值(比如迭代器),省得人手抄一长串类型。

隐式转换 与 float vs double 的精度

当你把一个类型的值塞进另一个类型的盒子,C++ 会悄悄(隐式)帮你转换。最常见的是整数和小数之间: 

float r = 5;        // int 5 被隐式转成 float 5.0f,安全
int   n = 5.9f;     // float 5.9 被转成 int → 小数部分被砍掉,n 变成 5(不是四舍五入!)

读代码踩坑提醒:小数 → 整数会直接截断(truncate),不是四舍五入。5.9f 进 int 盒子变 5,-1.9f 进 int 盒子变 -1。很多"为什么少了 1"的 bug 就藏在这种不起眼的转换里。

再说 float 和 double 的区别:两个都装小数,但 double 用更多内存换来更高精度(更多有效位)。在 VFX 里这个权衡天天发生:Houdini 的点位置 @P 分量通常是 float(几百万个点,内存和带宽要省);而某些大坐标累加、或反复迭代的解算,会用 double 防止误差越滚越大。读代码时看到 double,常常是个信号:这里在意精度。

VFX 类比 float vs double = 精度 vs 内存的老权衡,跟你选纹理位深(8-bit 还是 16/32-bit EXR)是同一种取舍。点位置用 float 够用又省内存;大世界坐标或长链解算用 double,免得远处物体"抖动"——和远处几何因 float 精度不足而跳动是一个道理。

你会在真实代码里看到

下面是一段贴近 HDK 风格的代码(遍历几何上的点、读写一个 float 属性)。现在你应该能逐个认出每个变量的类型、初始值、作用域和 const 的用意:

// gdp 是几何;只读传入,所以是 const(我们不改原始几何的拓扑)
void scalePscale(const GU_Detail *gdp, GA_RWHandleF pscaleH, const float factor)
{
    for (GA_Offset ptoff : gdp->getPointRange())
    {
        const float old = pscaleH.get(ptoff);  // 读出来就只读:old 不会再变
        float scaled = old * factor;        // 新盒子,装算好的结果
        pscaleH.set(ptoff, scaled);            // 写回属性
    }
}

读法演示:factor 是 const float ——只读的小数输入(像只读端口)。old 是 const float,在循环每一圈里诞生、用完即弃,作用域只在循环的 { } 内。scaled 是普通 float,因为它要被算出来后赋值。整段你不需要懂 GA_RWHandleF 具体是什么,光靠"类型 / 名字 / 初值 / const / 作用域"这五件事,就能读懂数据怎么流。

读代码易错点

  • 把 = 看成"相等"。单独一行的 a = b; 是赋值(把 b 装进 a),不是在判断它们相等。判断相等是 ==,且几乎只出现在 if / while 这类条件里。
  • 以为小数转整数会四舍五入。不会。int n = 7.99f; 得到的是 7,小数部分被直接砍掉。
  • 忽略 const。const 不是装饰,它是承诺"此盒子下文不改"。读到它,你可以放心这个值不会被偷偷动;反过来,没有 const 的盒子才要警惕被改写。
  • 把不同作用域的同名变量当成同一个。两个函数里都有 i,它们是两个无关的盒子。先确认"这个名字属于哪对 { }"。
  • 漏看 0.5f 末尾的 f。有 f 是 float 字面量,没有 f 的 0.5 是 double 字面量。在精度敏感的代码里,这个细节有时正是重点。
  • 遇到 auto 就以为"没类型"。它有类型,只是让你去 = 右边读出来。
