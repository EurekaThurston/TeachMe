枚举、switch 与位标志

真实 DCC 代码里满地都是 GROUP_POINT、case 分支和 FLAG_A | FLAG_B。这一课把这三样一次读穿。

本课目标(一个收获) 你能读懂三样东西:用名字代替裸数字的命名常量 enum、按值分流的多分支 switch,以及 DCC API 参数里常见的位标志组合 FLAG_A | FLAG_B——并且知道怎么用 & 检查某个标志有没有被设上。

热身 · 回忆上一课(别翻笔记)

先凭记忆答一题,把第 15 课"模板与尖括号"捞回来——主动回忆比重看一遍记得牢。

① enum:给一组数字取有意义的名字

你在 Houdini 的节点参数里见过下拉菜单:Group Type 让你选 Points / Primitives / Edges。底层其实就是 0 / 1 / 2 这几个整数。直接在代码里写 if (type == 1) 没人看得懂——1 是啥?enum 就是给这些整数贴上人话标签。 

enum GroupType
{
    GROUP_POINT,   // = 0(默认从 0 开始,自动 +1)
    GROUP_PRIM,    // = 1
    GROUP_EDGE     // = 2
};

读法:enum GroupType 定义了一个新"类型",它的合法取值只有花括号里那几个名字。每个名字背后还是一个整数(默认 0, 1, 2, …),但你读代码时看到的是 GROUP_PRIM 而不是冷冰冰的 1。这就是下拉菜单的选项名,而不是让你去记菜单第几项。 

GroupType t = GROUP_PRIM;        // 像在下拉菜单里选了 "Primitives"
if (t == GROUP_PRIM) { /* 处理面 */ }   // 一眼就懂,不用查 1 是什么
VFX 类比 enum 就是节点参数上那个下拉菜单(Ordered Menu):你看到的是 "Points / Primitives / Edges" 这些选项名,而底层存的是 0 / 1 / 2。enum 让 C++ 代码也能"按名字"读,而不是"按编号"猜。

② enum class:更安全的现代写法

现代 C++ 代码里你会越来越多看到 enum class(也叫 scoped enum,作用域枚举)。它和传统 enum 干同一件事,但更"严":必须用 类型名::值 的形式访问——这正是第 14 课讲过的作用域运算符 ::。 

enum class GroupType
{
    Point,
    Prim,
    Edge
};

GroupType t = GroupType::Prim;   // 必须带前缀 GroupType:: ,不能只写 Prim

两点区别,读代码时认得出就行:

看到这个传统 enumenum class
怎么访问GROUP_PRIMGroupType::Prim
名字会撞车吗会(都堆在外层)不会(关在自己作用域里)
能当整数直接用吗能(自动变 int)不能(更安全,要显式转)
读代码记住这条 看到 GroupType::Prim 这种带 :: 前缀的取值,就知道它是个 enum class;看到光秃秃的 GROUP_PRIM,多半是传统 enum。两者意思一样,只是写法严不严。

③ switch:按值分流的多岔路口

当一个值有好几种可能、每种都要做不同的事,与其堆一长串 if / else if,C++ 常用 switch。它特别爱和 enum 搭配——拿一个枚举值,跳到对应的 case。 

switch (groupType)
{
    case GROUP_POINT:
        processPoints();
        break;            // 处理完就跳出 switch
    case GROUP_PRIM:
        processPrims();
        break;
    default:                 // 以上都不匹配时走这里
        reportUnknown();
        break;
}

逐块读法:

  • switch (groupType):拿这个值去和下面每个 case 比。
  • case GROUP_PRIM::如果值等于 GROUP_PRIM,从这里开始执行。
  • break;:做完跳出整个 switch,不再往下。
  • default::所有 case 都没匹配上时的兜底分支(类似 else)。

易踩的坑:漏写 break 会"贯穿"(fall-through)

这是 switch 最反直觉的地方:如果某个 case 末尾忘了写 break,程序不会停,而是继续往下一个 case 里执行——哪怕那个 case 的标签根本不匹配。这叫 fall-through(贯穿)。 

switch (groupType)
{
    case GROUP_POINT:
        processPoints();
        // ← 这里漏了 break!
    case GROUP_PRIM:
        processPrims();    // GROUP_POINT 也会"顺手"跑到这一句!
        break;
}

有时这是故意的——让几个标签共用一段代码(下面会看到)。但更多时候,漏 break 是 bug。读代码时,看到某个 case 没有 break,要立刻警觉:这是有意贯穿,还是写漏了? 

// 故意贯穿:GROUP_POINT 和 GROUP_VERTEX 共用同一段处理
switch (groupType)
{
    case GROUP_POINT:
    case GROUP_VERTEX:        // 两个标签叠在一起,都走下面这段
        processPerPoint();
        break;
    case GROUP_PRIM:
        processPerPrim();
        break;
}

④ 位标志(bit flags):一排可同时勾选的复选框

enum 是"单选下拉菜单",但很多 DCC API 的选项是可以同时勾好几个的——比如"建组时:既保留已有点、又包含未使用的点、还顺便归一化"。这种"多选"用位标志表示:每个标志占二进制里的一个位(bit),互不干扰,可以自由组合。 

// 每个标志只点亮一个二进制位:1, 2, 4, 8, ...(都是 2 的幂)
enum BuildFlags
{
    BUILD_NONE      = 0,        // 0000 — 什么都不勾
    BUILD_KEEP      = 1,        // 0001 — 第 0 位
    BUILD_UNUSED    = 2,        // 0010 — 第 1 位
    BUILD_NORMALIZE = 4,        // 0100 — 第 2 位
    BUILD_REVERSE   = 8         // 1000 — 第 3 位
};
VFX 类比 位标志就是参数面板里那一排复选框(checkboxes):每个框独立勾或不勾。| 把你勾上的框合并成一个整数参数传给函数;& 再反过来问"这个框勾了没"。DCC 节点参数里的"组类型/标志位"就是这套机制。

用 | 组合标志(勾上几个框)

按位或 |(注意是一个竖线)把多个标志"叠"成一个整数:只要任意一个标志点亮了某个位,结果里那个位就是 1。 

int flags = BUILD_KEEP | BUILD_NORMALIZE;   // 0001 | 0100 = 0101 = 5
// 现在 flags 同时"勾上了" KEEP 和 NORMALIZE 两个框

用 & 检查某个标志(这个框勾了没)

按位与 &(一个 &)拿来"问"某个标志在不在:把组合值和单个标志 & 一下,如果那个位是 1,结果就非 0(真),否则是 0(假)。 

if (flags & BUILD_NORMALIZE)   // 0101 & 0100 = 0100,非 0 → 真
{
    normalizeGroup();          // NORMALIZE 这个框勾了,执行
}
if (flags & BUILD_REVERSE)      // 0101 & 1000 = 0000,是 0 → 假
{
    reverseOrder();            // REVERSE 没勾,跳过
}

顺带认全这几个按位运算符在 flag 参数里的含义:

符号名字在 flag 里干嘛
|按位或组合标志(勾上)
&按位与检查标志(问勾没)
~按位取反配合 & 清掉某标志
<<左移定义标志值,如 1 << 2 即 4

你常会看到标志这样定义:1 << 2 读作"把 1 左移 2 位",结果是 0100 = 4——比手写 4 更能表达"这是第 2 个位"。清掉一个标志则是 flags = flags & ~BUILD_KEEP:~ 把 KEEP 那个位反成 0、其余全 1,再 & 一下就只把那个位关掉。

你会在真实代码里看到

这是 Houdini HDK(C++ 开发包)里建组时很典型的一行——把多个选项用 | 拼成一个参数传进函数。现在你应该能读懂它在说什么: 

// HDK 风格:创建一个 primitive 组,组合多个构建标志
GA_PrimitiveGroup *grp = gdp->newPrimitiveGroup("myGroup");

geo->buildGroup(grp,
    GEO_GROUP_KEEP_EXISTING | GEO_GROUP_INCLUDE_UNUSED);
//        ↑ 第一个框                ↑ 第二个框
// 用 | 把"保留已有成员"和"包含未使用元素"两个选项合成一个参数

读这行时,把 A | B 在脑子里翻译成"同时打开 A 和 B 两个开关"就对了。函数内部多半会用 if (flags & GEO_GROUP_KEEP_EXISTING) 去逐个检查你勾了哪些。这也正是为什么 DCC / HDK API 爱用位标志:一个 int 参数就能携带十几个独立开关,既省参数个数,又能随意组合。

读代码易错点

  • 按位 & | ≠ 逻辑 && ||(第 4 课)。 单个 & / | 是逐位运算,处理标志;双写的 && / || 是真假运算,处理条件。看到 flags & FLAG 是"取那个位",看到 a && b 是"两个都为真"。一个字之差,意思完全不同。
  • switch 漏 break 会贯穿。 每个 case 末尾默认会"漏"到下一个 case。读到没有 break 的 case,先判断是有意共用还是写漏了。
  • enum class 忘加前缀。 enum class GroupType 的取值必须写 GroupType::Prim,光写 Prim 编译器不认。读代码看到 :: 前缀,就知道这是个作用域枚举。
  • 检查标志别写成 ==。 检查"勾没勾"要用 if (flags & FLAG),不是 if (flags == FLAG)——后者只在恰好只勾了这一个时才为真,组合标志时会判断错。
