先看全貌:一个完整(已精简)的 Actor

这是一个会自转、并在开场时打印一串路标点的 Actor。代码拆成两个文件:头文件 SpinningBeacon.h 声明"有什么",源文件 SpinningBeacon.cpp 写"怎么做"。这正是第 13 课讲的 .h / .cpp 分工。先整体扫一遍,不懂的地方下面逐块拆。

SpinningBeacon.h

// SpinningBeacon.h
#pragma once

#include "CoreMinimal.h"
#include "GameFramework/Actor.h"
#include "SpinningBeacon.generated.h"

UCLASS()
class MYGAME_API ASpinningBeacon : public AActor
{
    GENERATED_BODY()

public:
    ASpinningBeacon();                              // 构造函数

    UPROPERTY(EditAnywhere, Category = "Beacon")
    float SpinSpeed = 90.0f;                      // 每秒转多少度

    UPROPERTY(VisibleAnywhere, Category = "Beacon")
    TArray<FVector> Waypoints;                  // 一组路标点

protected:
    virtual void BeginPlay() override;

public:
    virtual void Tick(float DeltaTime) override;
};

SpinningBeacon.cpp

// SpinningBeacon.cpp
#include "SpinningBeacon.h"

ASpinningBeacon::ASpinningBeacon()
{
    PrimaryActorTick.bCanEverTick = true;        // 允许每帧 Tick
}

void ASpinningBeacon::BeginPlay()
{
    Super::BeginPlay();                          // 先让父类做它的开场

    Waypoints.Add(FVector(0.0f, 0.0f, 0.0f));
    Waypoints.Add(FVector(100.0f, 0.0f, 50.0f));

    for (const FVector& Point : Waypoints)
    {
        UE_LOG(LogTemp, Warning, TEXT("Waypoint Z = %f"), Point.Z);
    }
}

void ASpinningBeacon::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);

    FRotator Step(0.0f, SpinSpeed * DeltaTime, 0.0f);
    AddActorLocalRotation(Step);             // 绕自身 Yaw 轴转一点
}

下面把它切成几块,一块一块读。每读一块,我都会指出"这用到了第几课"。

第 1 块:头文件顶部的 #include

#pragma once

#include "CoreMinimal.h"
#include "GameFramework/Actor.h"
#include "SpinningBeacon.generated.h"

这用到了第 13 课的头文件与编译。 #pragma once 是"这个头文件只让编译器读一遍"的防重复声明。三个 #include 把别处定义好的名字搬进来:CoreMinimal.h 带来 Unreal 最常用的基础类型(比如 FVector、TArray),GameFramework/Actor.h 带来父类 AActor 的定义,这样下面才能写 : public AActor。

第三行 SpinningBeacon.generated.h 比较特别:它是 Unreal 的代码生成工具(UHT,Unreal Header Tool)自动生成的文件,你不会手写它。只要类里用了 UCLASS() 等反射宏,就必须把这个 .generated.h 放在 #include 列表的最后一行。记住这条规则即可,不用纠结它内部是什么。[Unreal 文档:Gameplay Classes]

第 2 块:UCLASS() 装饰的类,继承自 AActor

UCLASS()
class MYGAME_API ASpinningBeacon : public AActor
{
    GENERATED_BODY()

这用到了第 11 课和第 12 课的类与继承。 把它和普通 C++ 对照着看,你会发现核心就是一句你已经学过的话:class ASpinningBeacon : public AActor——定义一个叫 ASpinningBeacon 的类,公有继承自父类 AActor。继承意味着它自动拥有 AActor 的所有能力(能被放进关卡、有位置、能被 Tick),你只在它基础上加自己的东西。

剩下三个零件是 Unreal 包在外面的"约定层":

  • UCLASS() 放在 class 那一行上面,告诉 Unreal:"这个类要登记进引擎的反射系统",这样编辑器、Blueprint、序列化才认得它。它不是普通 C++,是 Unreal 的宏。
  • MYGAME_API 是个导出宏,跟模块编译有关,读代码时可以略过它,直接看后面的类名。
  • GENERATED_BODY() 必须放在类体最开头(花括号后第一行)。它会被展开成一堆 Unreal 自动生成的样板代码。所有 gameplay 类都必须有它。[Unreal 文档]
前缀速记(只此一处类比) 类名前面那个大写字母是 Unreal 的命名前缀约定,看到就能猜出它是哪一类东西:A = 能放进关卡的 Actor(AActor),U = 不能独立存在、必须挂在别人身上的对象/组件(UStaticMeshComponent),F = 普通数据结构(FVector、FRotator),T = 模板容器(TArray)。前缀只是名字的一部分,不改变它是不是 C++ 类。

第 3 块:UPROPERTY 成员变量

    UPROPERTY(EditAnywhere, Category = "Beacon")
    float SpinSpeed = 90.0f;

    UPROPERTY(VisibleAnywhere, Category = "Beacon")
    TArray<FVector> Waypoints;

这用到了第 3 课(变量与类型)和第 9 课(数组与 vector)。 先把 UPROPERTY(...) 那一行盖住,你看到的就是两句最普通的成员变量声明:

  • float SpinSpeed = 90.0f; ——一个 float 变量,声明时直接赋了初值 90.0f(末尾 f 表示这是 float 字面量)。
  • TArray<FVector> Waypoints; ——一个动态数组。TArray 是 Unreal 版的 std::vector,尖括号里 <FVector> 说明它装的是 FVector(三维向量)。这正是第 15 课讲的模板尖括号读法:容器<元素类型>。

再把 UPROPERTY(...) 揭开:它把这个变量也登记进反射系统,括号里的说明符决定它在编辑器里的行为。EditAnywhere 表示美术能在编辑器面板里改它;VisibleAnywhere 表示只能看不能改;Category = "Beacon" 决定它在面板里归到哪个分组下。[Unreal 文档:UProperties] 对读代码而言,关键是认出:变量本体是普通 C++,UPROPERTY 只是给它贴的一张 Unreal 标签。

第 4 块:声明 BeginPlay 与 Tick(带 virtual / override)

protected:
    virtual void BeginPlay() override;

public:
    virtual void Tick(float DeltaTime) override;

这用到了第 1 课(读函数签名)和第 12 课(类的 public/protected 与虚函数)。 抛开 virtual 和 override,这两行就是普通的函数声明,用第 1 课的"读合同"方法看:

  • void BeginPlay() ——返回 void(不返回值),无参数。
  • void Tick(float DeltaTime) ——返回 void,接收一个 float 参数 DeltaTime(距上一帧过了多少秒)。

virtual ... override 是第 12 课的内容:父类 AActor 里本来就有 BeginPlay 和 Tick,override 表示"我用自己的版本覆盖父类那个"。这两个名字是 Unreal 生命周期的固定钩子:BeginPlay() 在关卡开始、这个 Actor 进入游戏时被引擎调用一次;Tick(float DeltaTime) 在之后每一帧被调用。[Unreal 文档:Actor Lifecycle] 你不用手动去调它们,引擎到点会替你调。

第 5 块:构造函数

ASpinningBeacon::ASpinningBeacon()
{
    PrimaryActorTick.bCanEverTick = true;
}

这用到了第 12 课的构造函数和第 14 课的作用域符号 ::。 ASpinningBeacon::ASpinningBeacon() 里的 :: 读作"属于":它说"这是 ASpinningBeacon 这个类的、名字也叫 ASpinningBeacon 的函数"——名字和类同名、且没有返回类型,这就是构造函数的特征,在对象创建时自动跑一次。

里面只有一句:PrimaryActorTick.bCanEverTick = true;。PrimaryActorTick 是从父类继承来的一个成员,通过点号 . 访问它内部的开关 bCanEverTick,把它赋值为 true(这是第 4 课的赋值 =,不是比较 ==)。意思是"允许这个 Actor 每帧被 Tick"。必须在构造函数里设,因为它要在 BeginPlay 之前就定好。[Unreal 文档:Actor Ticking](顺带一提:bCanEverTick 前缀 b 是 Unreal 给 bool 变量的命名约定。)

第 6 块:BeginPlay() 里的 Super 与遍历 TArray

void ASpinningBeacon::BeginPlay()
{
    Super::BeginPlay();

    Waypoints.Add(FVector(0.0f, 0.0f, 0.0f));
    Waypoints.Add(FVector(100.0f, 0.0f, 50.0f));

    for (const FVector& Point : Waypoints)
    {
        UE_LOG(LogTemp, Warning, TEXT("Waypoint Z = %f"), Point.Z);
    }
}

这块最长,但全是前面学过的东西拼起来。逐行读:

  • Super::BeginPlay(); ——Super 是"父类"的代称(等于 AActor)。这一句先把父类版本的 BeginPlay 跑一遍,再做自己的事。覆盖父类方法时通常都要这么做,免得漏掉父类的初始化。又是第 14 课的 ::。
  • Waypoints.Add(FVector(0.0f, 0.0f, 0.0f)); ——往数组里追加一个元素。Waypoints.Add(...) 是第 11 课的"对象 . 方法"调用;括号里 FVector(0.0f, 0.0f, 0.0f) 当场造一个三维向量 (X, Y, Z)。这是第 9 课动态数组的 Add(对应 std::vector::push_back)。
  • for (const FVector& Point : Waypoints) ——这是第 5 课的范围 for 循环:把 Waypoints 里的元素一个个取出,每次叫 Point。const FVector& 用了第 6 课的引用 &(不拷贝,直接借用)和 const(循环里不改它),这是只读遍历的标准写法。
  • UE_LOG(LogTemp, Warning, TEXT("Waypoint Z = %f"), Point.Z); ——往日志打印。Point.Z 用点号取出该向量的 Z 分量(第 11 课读成员)。TEXT("...") 是 Unreal 的字符串字面量写法,%f 是占位符,会被后面的 Point.Z 填进去。

第 7 块:Tick() 每帧自转

void ASpinningBeacon::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);

    FRotator Step(0.0f, SpinSpeed * DeltaTime, 0.0f);
    AddActorLocalRotation(Step);
}

这用到了第 4 课(表达式)和第 3 课(局部变量)。 第一句 Super::Tick(DeltaTime) 还是先调父类版本。然后 FRotator Step(...) 声明一个局部变量 Step,类型是 FRotator(一个旋转量,三个参数依次是 Roll、Pitch、Yaw)。

中间那个参数 SpinSpeed * DeltaTime 是第 4 课的表达式:转速(度/秒)乘以本帧用了多少秒,得到"这一帧该转多少度"。乘上 DeltaTime 是游戏代码里极常见的套路——这样不管帧率高低,转动速度都一致。最后 AddActorLocalRotation(Step) 把这点旋转加到自己身上。每帧加一点,看起来就是匀速自转。

读代码易错点

  • 反射宏不是普通 C++。 UCLASS()、UPROPERTY()、UFUNCTION()、GENERATED_BODY() 都是 Unreal 自己的宏,普通 C++ 编译器并不认识它们;它们要靠 Unreal Header Tool 先处理。读代码时把它们当成"贴在普通声明上的 Unreal 标签",先看被它们包住的真正声明。
  • 前缀只是名字。 A/U/F/T 不是关键字,只是命名约定。ASpinningBeacon 和 SpinningBeacon 文件名对得上但不是同一个词——类名带 A,文件名常常不带。看到前缀能帮你判断它大概是什么(能否放进关卡、是不是组件),但别把它当语法。
  • 忘了 GENERATED_BODY() 会整类报错。 它必须在类体第一行;漏了它,反射系统不认这个类,编译直接失败。读别人代码时若看到一个 UCLASS 类却找不到 GENERATED_BODY(),基本可以断定那段代码不完整。
  • Tick 不写 bCanEverTick = true 就不会被调。 光声明并实现 Tick() 不够;若构造函数里没把 PrimaryActorTick.bCanEverTick 设为 true,引擎根本不会每帧调它。看到 Tick 没生效,先回构造函数找这一句。
  • = 不是 ==。 构造函数里的 bCanEverTick = true 是赋值;第 2 课提醒过,比较相等才用 ==。读条件判断时尤其要分清。
