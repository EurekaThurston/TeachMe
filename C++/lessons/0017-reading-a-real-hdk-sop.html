热身 · 回忆第 16 课(别翻笔记)

先凭记忆答一题,把上一课的 enum / switch 捞回来——它马上就会在真实代码里出现。

我们要读的真实文件:SOP_Star

这是 SideFX 官方 HDK 自带的示例插件 SOP_Star:一个"生成器"SOP,在视口里画出一颗星星。你给它"分段数"和"内/外半径",它就摆出对应数量的点,连成一个多边形。[1] 为了教学,我把它轻微精简了(去掉了部分错误处理和注释),但所有 API 都是真实存在的,没有一个是我编的。

它分成两个文件,这是 C++ 的常规拆法(第 13 课的"头文件 / 源文件"): 头文件 .h 声明这个类长什么样,源文件 .C 写出每个方法具体怎么做。先把整体过一遍,后面再逐块放大。

头文件 SOP_Star.h(精简)

// SOP_Star.h —— 声明这个插件类
#include <SOP/SOP_Node.h>
#include <UT/UT_StringHolder.h>

namespace HDK_Sample {

class SOP_Star : public SOP_Node
{
public:
    static OP_Node *myConstructor(OP_Network *net,
                                  const char *name,
                                  OP_Operator *op);

    static const UT_StringHolder theSOPTypeName;

protected:
    SOP_Star(OP_Network *net, const char *name, OP_Operator *op);
    virtual ~SOP_Star();

    virtual OP_ERROR cookMySop(OP_Context &context) override;
};

} // namespace HDK_Sample

源文件 SOP_Star.C(精简)

// SOP_Star.C —— 实现每个方法
#include "SOP_Star.h"
#include <GU/GU_Detail.h>
#include <OP/OP_Operator.h>
#include <OP/OP_OperatorTable.h>
#include <UT/UT_Interrupt.h>
#include <SYS/SYS_Math.h>

using namespace HDK_Sample;

const UT_StringHolder SOP_Star::theSOPTypeName("hdk_star");

// Houdini 启动时调用这个,把插件登记进节点表
void newSopOperator(OP_OperatorTable *table)
{
    table->addOperator(new OP_Operator(
        SOP_Star::theSOPTypeName,   // 内部类型名
        "Star",                     // 菜单里显示的名字
        SOP_Star::myConstructor,    // 怎么造一个出来
        SOP_Star::buildTemplates(), // 它有哪些参数
        0, 0,                       // 最少/最多输入数
        nullptr,
        OP_FLAG_GENERATOR));        // 标志:这是个生成器
}

// 工厂方法:造一个 SOP_Star 实例
OP_Node *
SOP_Star::myConstructor(OP_Network *net, const char *name, OP_Operator *op)
{
    return new SOP_Star(net, name, op);
}

// 构造函数:节点被创建时跑一次
SOP_Star::SOP_Star(OP_Network *net, const char *name, OP_Operator *op)
    : SOP_Node(net, name, op)
{
    mySopFlags.setManagesDataIDs(true);
}

SOP_Star::~SOP_Star() {}

// 核心:每次需要重算几何体时,Houdini 调用这个
OP_ERROR
SOP_Star::cookMySop(OP_Context &context)
{
    fpreal now = context.getTime();

    // 1) 读参数
    int   divisions = evalInt("divs", 0, now);
    float outerRad  = evalFloat("rad",  0, now);
    float innerRad  = evalFloat("rad",  1, now);
    int   plane     = evalInt("orient", 0, now);

    int npoints = divisions * 2;
    if (npoints < 4)
        npoints = 4;

    // 2) 决定星星画在哪个平面上
    int xc, yc, zc;
    switch (plane)
    {
        case 0: xc = 0; yc = 1; zc = 2; break;  // XY
        case 1: xc = 1; yc = 2; zc = 0; break;  // YZ
        default: xc = 0; yc = 2; zc = 1; break; // ZX
    }

    // 3) 清空旧几何,造 npoints 个新点
    gdp->clearAndDestroy();
    GA_Offset start = gdp->appendPointBlock(npoints);

    // 4) 一圈一圈摆点
    UT_AutoInterrupt boss("Building Star");
    float tinc = M_PI * 2 / (float)npoints;

    for (int i = 0; i < npoints; i++)
    {
        if (boss.wasInterrupted())
            break;

        float angle = (float)i * tinc;
        bool  odd   = (i & 1);
        float rad   = odd ? innerRad : outerRad;

        UT_Vector3 pos(SYScos(angle) * rad, SYSsin(angle) * rad, 0);
        pos = UT_Vector3(pos(xc), pos(yc), pos(zc));

        GA_Offset ptoff = start + i;
        gdp->setPos3(ptoff, pos);
    }

    return error();
}

看起来很多?别慌。这就是前 16 课的所有东西堆在一起而已——没有一个新语法。下面我们一块一块放大,每块都告诉你"这是第几课的"。

放大 ① · 那一堆 #include —— 第 13 课

#include "SOP_Star.h"
#include <GU/GU_Detail.h>
#include <UT/UT_Interrupt.h>
#include <SYS/SYS_Math.h>

这用到了第 13 课的 #include 与编译单元。每一行都是"把另一个文件的声明抄进来,这样我才能用它定义的类型和函数"。

  • "SOP_Star.h" 用双引号——自己项目里的头文件。
  • <GU/GU_Detail.h> 用尖括号——来自 HDK 库。GU_Detail 就是"一坨几何体"(点、面、属性),后面的 gdp 就是它。
  • <SYS/SYS_Math.h> 给我们 SYScos / SYSsin / M_PI——也就是第 10 课讲的"标准库 / 库函数"在 HDK 里的对应物。

放大 ② · 类声明 —— 第 11、12 课

class SOP_Star : public SOP_Node
{
public:
    static OP_Node *myConstructor(...);
protected:
    SOP_Star(...);
    virtual OP_ERROR cookMySop(OP_Context &context) override;
};

这一整块是第 11 课(读懂类)和第 12 课(类的深入)。逐个认:

  • class SOP_Star:定义一个新类型,名字叫 SOP_Star。
  • : public SOP_Node:继承。读作"SOP_Star 是一种 SOP_Node"。它白白拿到了 SOP_Node 已有的全部本事(比如那个 gdp 成员、evalFloat 方法),只需补上自己特别的部分。这正是 第 12 课的核心。
  • public: / protected::访问控制(第 12 课)。public 是对外开放的门,protected 是只给"自己和子类"用的内部门。
  • virtual ... cookMySop(...) override:virtual + override 表示"我重写了父类的这个方法"。SOP_Node 说'每个 SOP 都得有 cookMySop',这里就是 SOP_Star 给出自己的版本。

注意签名 OP_ERROR cookMySop(OP_Context &context)——这正是第 1 课的"读函数签名":返回 OP_ERROR(成功/失败的状态码),收一个 OP_Context& 参数。那个 & 是第 6 课的引用——不复制整个 context,直接借用原件。

放大 ③ · 构造函数 —— 第 12 课

SOP_Star::SOP_Star(OP_Network *net, const char *name, OP_Operator *op)
    : SOP_Node(net, name, op)   // ← 先让父类初始化
{
    mySopFlags.setManagesDataIDs(true);
}

构造函数(第 12 课)是节点被创建那一刻自动跑一次的"开机程序"。读它的两个关键点:

  • SOP_Star::SOP_Star 那个 :: 是第 14 课的作用域解析符:"这是属于 SOP_Star 类的那个 SOP_Star 函数"。
  • : SOP_Node(net, name, op) 这段叫初始化列表:在干自己的活之前,先把收到的参数转交给父类,让父类先把自己那一半搭好。
  • 那几个 OP_Network *net、const char *name 是第 7 课的指针。const char * 就是 第 10 课提过的"C 风格字符串"——指向一串字符的指针。

放大 ④ · 读参数 —— 第 1、3、9 课

fpreal now = context.getTime();

int   divisions = evalInt("divs", 0, now);
float outerRad  = evalFloat("rad",  0, now);
float innerRad  = evalFloat("rad",  1, now);

这是你最熟的部分——读节点上的参数。这几行同时用到了好几课:

  • int divisions = ...:第 3 课的变量、类型、赋值——开一个 int 盒子,把右边算出来的值塞进去。fpreal 只是 HDK 给高精度 float 起的别名(类型,第 3 课)。
  • evalFloat("rad", 0, now) / evalFloat("rad", 1, now):这就是 VEX 里的 ch("rad")!三个参数分别是"参数名、第几个分量、什么时刻"。注意同一个参数 "rad" 用 index 0 和 1 取到两个值——因为 rad 是个二维向量参数(外半径、内半径)。这个"按下标取分量"正是第 9 课(数组下标从 0 开始)的味道。
  • 调用 evalFloat(...) 本身,就是第 1 课读签名的反面:你现在会用这个签名了。
VFX 类比 cookMySop 就是这个节点的 VEX Wrangle 代码。evalFloat("rad", 0, now) = Wrangle 里的 ch("rad");gdp = 你 Wrangle 操作的那份几何;setPos3(ptoff, pos) = 给 @P 赋值。整段 cook 做的事,就是"每次时间或参数变了,重新把点摆好"——和你天天写的属性 Wrangle 一模一样,只是用 C++ 写出来、提前编译好,所以更快。

放大 ⑤ · switch 选平面 —— 第 16 课

int xc, yc, zc;
switch (plane)
{
    case 0: xc = 0; yc = 1; zc = 2; break;  // XY
    case 1: xc = 1; yc = 2; zc = 0; break;  // YZ
    default: xc = 0; yc = 2; zc = 1; break; // ZX
}

正是你刚热身回忆的第 16 课!plane 是那个 "Orientation" 下拉菜单的整数值(0=XY、1=YZ、2=ZX)。switch 根据它跳到对应 case,给三个坐标轴下标赋值。

官方原版这里其实用的是 enum(SOP_StarParms::Orient::XY)而不是裸数字 0/1/2——这也是第 16 课讲的"用 enum 给数字起名字、让代码自解释"。我为教学换成了数字,但你要认得:真实代码里 case 后面常常是 enum 名,不是数字。[1]

放大 ⑥ · 每点循环 —— 第 5、4、7、9 课

for (int i = 0; i < npoints; i++)
{
    if (boss.wasInterrupted())
        break;

    float angle = (float)i * tinc;
    bool  odd   = (i & 1);
    float rad   = odd ? innerRad : outerRad;

    UT_Vector3 pos(SYScos(angle) * rad, SYSsin(angle) * rad, 0);
    GA_Offset ptoff = start + i;
    gdp->setPos3(ptoff, pos);
}

这一块信息最密,慢慢拆——它把好几课同时用上了:

  • for (int i = 0; i < npoints; i++):第 5 课的 for 循环。把 i 从 0 数到 npoints-1,每个点跑一遍。这正是 VEX 里"逐点运行"的本质。
  • angle = (float)i * tinc:第 4 课的运算符与表达式。(float)i 是类型转换(把整数 i 变成 float 再算)。
  • odd = (i & 1):第 4 课 / 第 16 课的位运算 &。i & 1 取 i 的最低位——偶数得 0、奇数得 1。这里用它来交替"外半径/内半径",这就是星星尖角与凹角交错的来历。
  • rad = odd ? innerRad : outerRad:三目运算符(第 4 课)。读作"odd 为真就取 innerRad,否则取 outerRad"。
  • UT_Vector3 pos(...):第 9 课。UT_Vector3 就是 HDK 的 vector——三个 float 的组合,和 VEX 的 vector @P 同一个概念。
  • gdp->setPos3(ptoff, pos):gdp 是个指针(第 7 课),所以用 -> 而不是 . 来调它的方法。GA_Offset 是 HDK 里"第几个点"的句柄(类似 VEX 的点号 @ptnum,但更底层)。这一行 = 给第 ptoff 个点写入位置 = VEX 里的 @P = pos;。

读代码易错点

读代码易错点
  • gdp-> 用箭头不是点。 因为 gdp 是指针(GU_Detail *,第 7 课)。看到 -> 就提醒自己"左边是个指针"。如果它是普通对象才用 .(第 11 课)。
  • evalFloat("rad", 0, now) 里的 0 不是"时间"。 它是第几个分量(component index),时间是第三个参数 now。看错就会以为只取了一个值,其实 index 0 和 1 取的是两个不同的半径。
  • i & 1 是位与,不是逻辑与 &&。 单个 & 在按位运算(第 16 课);两个 && 才是"并且"。读串了含义就全错。
  • 循环里的 break 不是"返回"。 if (boss.wasInterrupted()) break; 只是跳出 for 循环(第 5 课),不是退出整个函数。函数最后那句 return error(); 才是真正的返回点。
  • cookMySop 没有 gdp = ...。 gdp 是从父类 SOP_Node 继承来的成员(第 12 课),早就存在了;你只管往里塞点。别去找它在哪声明——它在父类里。

放大 ⑦ · 收尾 return —— 第 1、2 课

    return error();
}

函数最后 return error();:把累计的错误状态(OP_ERROR)交回去,Houdini 据此判断这次 cook 成不成功。这就回到了第 1 课(返回类型是 OP_ERROR)和第 2 课(return 是函数的出口)。从签名到返回,你刚刚完整走通了一个真实函数体。

VFX 类比 · 全景回顾 把整份文件想成一个 HDA 的内部:newSopOperator 是"把这个数字资产注册到 TAB 菜单";buildTemplates 是节点的参数界面;构造函数是"放下节点时跑一次的初始化";cookMySop 是节点的计算图本身,每次上游或参数变化就重跑。你平时搭 HDA + Wrangle 做的事,C++ 插件只是把它写成了源码、编译成 .so/.dll。
