读懂函数签名:C++ 代码的入口

学会读一行,就能猜出整段代码的"接口"。这是阅读 C++ 最高杠杆的第一项技能。

本课目标(一个收获) 看到任意一个 C++ 函数的第一行,你能立刻说出三件事:它返回什么类型、叫什么名字、要喂给它哪些输入。

从你已经懂的东西出发

你在 Houdini 里写过 clamp(value, 0, 1),或者在着色器里见过它:把一个数限制在最小和最大值之间。 你已经懂它做什么。现在我们只看它在 C++ 里长什么样。 

float clamp(float value, float minVal, float maxVal)
{
    if (value < minVal) return minVal;
    if (value > maxVal) return maxVal;
    return value;
}

先别管 { } 里面的逻辑。只看第一行——这一行叫函数签名(function signature)。它是这个函数对外的"合同":告诉你能塞进去什么、会拿到什么。

签名的四个零件

float  clamp  (float value,  float minVal,  float maxVal)
 │       │        └──────────── 参数列表:每个都是「类型 名字」 ──────────┘
 │       └─ 函数名:你调用时用的名字
 └─ 返回类型:这个函数算完会"吐出"什么类型的值

关键差别,记住这一点就够本课了:在 C++ 里,类型是明写出来的。 每个值前面都贴着它的类型标签——float 表示"带小数的数"。Houdini 的 VEX 里其实也是这样(float、vector、int),只是你可能没太留意。

VFX 类比 把函数想成一个 Houdini 节点:参数列表就是节点左上角的输入端口(每个端口有规定的数据类型),返回类型就是节点底部的输出端口。读签名 = 看一个节点收什么、出什么。

常见的类型标签(够你起步了)

C++ 写法意思VFX 里类似
int整数(没有小数)VEX 的 int、点编号
float带小数的数VEX 的 float、属性值
bool真/假(true/false)开关、勾选框
void不返回任何值只"做事"、不输出的节点
float* / float&(后面课讲)指向某个 float引用一份数据,先眼熟
