struct 与 class 读法 —— 把数据和功能打包

到目前为止你读的都是"单个值"。真实代码里,数据是成捆出现的。这一课教你拆包。

本课目标(一个收获) 自上而下读懂一个 struct / class 定义——认出数据成员、成员函数(方法)、用 . 访问成员,以及 public / private 各自的含义。

热身 · 回忆第 10 课(别翻笔记)

先凭记忆答一题,把上次的标准库类型捞回来——主动回忆比重看一遍记得牢得多。

从你已经懂的东西出发:一个 Vector3

你天天在用 向量(vector):一个点的位置 P 有 x、y、z 三个分量。 这三个数属于同一个东西,所以把它们捆成一捆才合理。C++ 里捆数据的工具就是 struct: 

struct Vector3
{
    float x;   // 数据成员:横向分量
    float y;   // 数据成员:纵向分量
    float z;   // 数据成员:深度分量
};

逐块读:struct 是关键字,意思是"我要定义一个新类型";Vector3 是这个新类型的名字(以后它就能像 float 一样当类型用);{ } 里列的 x、y、z 叫数据成员(data members),每行还是熟悉的「类型 名字」写法。 最后那个 }; 的分号别漏——struct 定义结尾必须有分号,这是 C++ 的硬规矩。

VFX 类比 一个 struct 就像 Houdini 里一个自定义属性组:你把 x/y/z(或者 P、N、Cd)这些相关属性捆在一个名字下面,整组一起传递、一起命名,而不是散着拎三个孤零零的 float。

类型 ≠ 实例:先有模具,再有零件

上面那段 struct Vector3 { ... }; 只是定义了一种类型,它本身不占数据——就像 Houdini 里"属性的定义"不是某个具体点的值。 要真正得到一份数据,你得创建一个对象(object,也叫实例 instance): 

Vector3 pos;        // 创建一个 Vector3 对象,名字叫 pos
pos.x = 1.0f;       // 用 . 访问成员 x,给它赋值
pos.y = 2.5f;
pos.z = 0.0f;

float height = pos.y;  // 读出 pos 的 y 分量(= 2.5)

核心动作就是那个点号 .:对象名.成员名。 pos.y 读作"pos 的 y"。Vector3 是模具(类型),pos 是用模具做出来的一个零件(实例)。同一个模具能做出无数个零件:Vector3 a, b, c; 各有各自独立的 x/y/z。

成员函数(方法):打包进类型里的函数

向量不只是三个数,它还有能对自己做的运算:求长度、归一化。C++ 允许你把这些函数写进 struct 内部,它们就叫成员函数(member functions),口语里常叫方法(method): 

struct Vector3
{
    float x, y, z;                // 三个数据成员,可以写在一行

    float length() const          // 成员函数:求自己的长度
    {
        return std::sqrt(x*x + y*y + z*z);   // 直接用 x/y/z,不写对象名
    }

    void normalize()             // 成员函数:把自己缩放到长度为 1
    {
        float len = length();    // 成员函数之间可以直接互相调用
        x /= len; y /= len; z /= len;
    }
};

读法上和普通函数几乎一样——还是「返回类型 名字(参数)」加一个 { } 函数体。两个新现象要认得:

① 函数体里直接出现了 x、y、z,前面没有对象名。 因为成员函数天生"长在"某个对象身上,它里面裸写的 x 就是指调用它的那个对象自己的 x。

② length() 后面那个 const 表示"我只读、不改对象的数据"(normalize() 没有 const,因为它会改 x/y/z)。现在认得它是个"只读保证"就够了,细节后面课再展开。

VFX 类比 成员函数就像这个数据类型自带的"内置节点操作":Vector3 自己就懂怎么 length() 和 normalize(),就像 Houdini 的 vector 自带 length()、normalize() VEX 函数一样。数据和"能对它做的操作"打包在了一起。

怎么调用方法:还是那个点号

调用成员函数,语法和访问数据成员一模一样,只是名字后面多了括号 (): 

Vector3 v;
v.x = 3.0f; v.y = 4.0f; v.z = 0.0f;

float len = v.length();   // "v 的 length()" → 算出 5.0
v.normalize();             // "让 v 归一化自己" → 现在 v 长度变 1

对照着记:v.x 是数据成员(一个值,没括号);v.length() 是成员函数(会跑一段代码并返回一个值,有括号)。 点号左边永远是一个对象,右边是它的某个成员。

public / private:谁能碰这些成员

真实代码里,成员常常被分成两类:外人能碰的和只有自己人能碰的。控制这个的关键字是 public 和 private(它们叫访问说明符 access specifier): 

class Particle
{
public:                      // 这条线以下:外部代码可以访问
    Vector3 position;
    float getMass() const { return mass; }

private:                     // 这条线以下:只有类内部能访问
    float mass;             // 外面写 p.mass 会编译报错
};

读法:public: 后面列的成员,任何拿到这个对象的代码都能用 . 访问;private: 后面列的成员,只有这个类自己的成员函数能碰,外面写 p.mass 会直接编译失败。 所以你常看到的套路是:数据 mass 设成 private 藏起来,再开一个 public 的 getMass() 让外面"隔着窗口"读它。

VFX 类比 把 public 想成节点暴露在界面上的参数(谁都能调),private 是节点内部的中间计算(藏在 HDA 里,外面看不见也改不了)。封装的目的是一样的:只露出该露的,藏住内部细节。

struct 和 class 唯一的硬区别

你会同时见到 struct 和 class,它们几乎完全一样。语言层面唯一的硬性区别只有一条:默认访问权限——

关键字没写 public/private 时,成员默认是
struct默认 public(外部能访问)
class默认 private(只有内部能访问)

就这么多。习惯上,大家用 struct 表示"一捆公开的数据"(像前面的 Vector3),用 class 表示"有藏起来的内部状态的东西"(像 Particle)——但这只是约定,不是语言强制。读代码时:看到 struct 就默认成员公开,看到 class 就默认成员私有,然后再看里面有没有写 public: / private: 改了默认。

你会在真实代码里看到

渲染器、Houdini HDK、游戏引擎里满是这种结构。下面是一段贴近真实 HDK 风格的代码,综合了本课所有要点——试着自上而下读一遍: 

// 一个粒子:位置 + 速度 + 是否存活
struct Particle
{
    Vector3 P;        // 位置(注意:成员的类型可以是另一个 struct!)
    Vector3 v;        // 速度
    float   age;      // 已存活时间
    bool    alive;    // 是否还活着

    void step(float dt)   // 推进一个时间步
    {
        P.x += v.x * dt;  // 链式点号:P 的 x
        P.y += v.y * dt;
        P.z += v.z * dt;
        age += dt;
    }
};

// 别处的使用代码:
Particle pt;
pt.alive = true;
pt.step(0.04f);          // 调用方法,推进一帧
if (pt.P.y < 0.0f)        // pt 的 P 的 y:连着两个点号
    pt.alive = false;

注意 pt.P.y 这种链式点号:pt 是个 Particle,它的成员 P 又是个 Vector3,所以再点一层 .y 才到那个 float。从左往右一层层剥就行:pt 的 P 的 y。

读代码易错点

① 把方法当普通函数到处找。 看到调用 v.length() 却在文件顶层找不到 length 的定义?因为它是成员函数,定义藏在 struct Vector3 { ... } 的大括号里面。找方法要去它所属类型的定义里找,不是去全局找。

② 忘了对象名 + 点号。 在成员函数外面,不能裸写 length() 或 x——必须是 某个对象.length()、某个对象.x。裸写 x 只在成员函数内部才合法(那时它指当前对象自己的 x)。

③ 混淆类型和实例。 Vector3(大写、是类型/模具)不能直接 .x;只有具体对象 pos(是实例/零件)才能 pos.x。看到 类型名.成员 这种写法,八成读错了。

④ 漏看 private 就以为能访问。 一个成员能不能从外面 . 到,取决于它落在 public: 还是 private: 段里。读类定义时,养成习惯先扫一眼成员归在哪一段。
