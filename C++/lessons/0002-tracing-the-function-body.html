热身 · 回忆上一课(别翻笔记)

先凭记忆答一题,把上次的东西捞回来——这种"主动回忆"比重看一遍记得牢得多。

同一个 clamp,这次看里面

float clamp(float value, float minVal, float maxVal)
{
    if (value < minVal) return minVal;   // 太小 → 顶到下限
    if (value > maxVal) return maxVal;   // 太大 → 压到上限
    return value;                         // 不大不小 → 原样放行
}

三条你需要的规则

① 从上往下,一行一行执行。 代码像一份从头读到尾的步骤清单。

② if (条件) 是一道闸门。 括号里的条件算出来要么真要么假。真 → 执行它后面那句;假 → 跳过那句,继续往下。比较符号:

写法意思
a < ba 小于 b 吗
a > ba 大于 b 吗
a == ba 等于 b 吗(两个等号!)
a != ba 不等于 b 吗

③ return 是立刻退出。 一旦执行到 return,函数当场把那个值吐出去、结束,后面的行根本不会再跑。这点最关键——很多人读错就是忘了 return 会"提前离场"。

跟我手跑一遍:clamp(5.0, 0.0, 1.0)

把 value=5.0、minVal=0.0、maxVal=1.0 代进去,逐行执行:

// value=5.0, minVal=0.0, maxVal=1.0
if (5.0 < 0.0)  → 假   // 5 不小于 0,跳过这行
if (5.0 > 1.0)  → 真   // 5 大于 1,执行 → return 1.0,结束!
// 第三行 return value 永远到不了

结果:返回 1.0。5 被压到了上限。你刚刚就"读懂"了这段代码——不是猜,是跑出来的。
