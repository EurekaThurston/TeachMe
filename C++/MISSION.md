# Mission: C++ for VFX Technical Artists

## Why
我是特效向 TA,**主要在 Unreal Engine 里工作**(Niagara、材质、蓝图)。很多强大的功能锁在 Unreal C++ 背后(自定义 Actor/Component、引擎源码、Niagara 模块)。学会 C++ 能让我突破"只能用别人做好的工具/蓝图"的天花板,真正读懂并定制这些底层代码。

## Success looks like
- 能打开一个真实的 Unreal C++ 文件(Actor/Component、引擎模块),逐行看懂它在做什么
- 能对现有 C++ 代码做小修改并成功编译运行(改 UPROPERTY 默认值、加分支、调逻辑),在编辑器里看到效果
- 不再因为"这是 C++ 写的"就放弃定制某个功能

## Constraints
- 几乎零正式编程基础(主要靠节点图/蓝图、可视化工具工作)
- **不熟 Houdini**;主战场是 Unreal。教学示例与压轴实操以 Unreal 为准。
- **教学风格:少用类比、直接把代码讲清楚**(用户明确偏好直说,不要堆 DCC 比喻)
- 学习以**阅读+修改**为核心,不追求从零造大型工程
- 时间有限,每节课要能快速完成、有立刻能用上的收获(但内容要深、信息量大,不堆水)

## Out of scope (暂时)
- 大型 C++ 工程架构/设计模式
- 手写复杂模板元编程
- 从空白文件独立开发完整插件(先能读懂改动,再谈从零写)
