2D 旋转:一个角度怎么变成一个矩阵

旋转是 TA 最常碰、也最容易翻车的变换。我们从最干净的 2D 开始:一个角度 θ,如何变成一个 2×2 矩阵?答案把第 2 课的 sin/cos 和第 11 课的「矩阵=基向量按列」缝在一起。

本课目标(一个收获) 你能写出并看懂 2D 旋转矩阵:它的两列就是旋转后的 î 和 ĵ。î 转 θ 后落在 (cosθ, sinθ),ĵ 落在 (−sinθ, cosθ)。把这两列拼起来就是旋转矩阵。旋转不改变长度(正交基),所以它只转朝向、不缩放。

从「转一根基向量」开始

第 11 课说:矩阵的列 = 变换后的基向量。所以要造旋转矩阵,只需问两个问题:î 转 θ 后去哪?ĵ 转 θ 后去哪?

回忆第 2 课单位圆:一个单位向量,转到角度 θ,落点就是 (cosθ, sinθ)。î 一开始在 0°(指右),转 θ 后正好落在 (cosθ, sinθ)。ĵ 一开始在 90°(指上),转 θ 后落在 90°+θ,也就是 (−sinθ, cosθ)(用诱导公式:cos(90°+θ)=−sinθ,sin(90°+θ)=cosθ)。

î → (cosθ, sinθ)　　ĵ → (−sinθ, cosθ)

拼成旋转矩阵

把这两个去处按列摆好(第 11 课的读法),就是 2D 旋转矩阵:

R(θ) = [ cosθ　−sinθ ;　sinθ　cosθ ]　　第 1 列=转后的 î,第 2 列=转后的 ĵ

这就是全部。一个角度,经过 sin/cos,变成四个数,装进一个 2×2 盒子。下面的画布:拖滑块改变 θ,看红色 î 和绿色 ĵ 怎么一起转,橙点跟着被旋转,右边 readout 实时显示矩阵的四个数——你会看到对角线是 cosθ、反对角是 ∓sinθ。

拖角度滑块。注意:î、ĵ 始终保持单位长、互相垂直(正交基),只是整体转动——这就是「纯旋转不缩放」。

为什么旋转不改变长度

旋转后的 î=(cosθ, sinθ),它的长度是 √(cos²θ + sin²θ) = 1(第 2 课的勾股恒等式)。ĵ 同理。两列都是单位长、且互相垂直——这是一组正交基(第 10 课)。所以旋转矩阵是正交矩阵:它只转朝向,不拉伸、不剪切。它的行列式 det = cos²θ + sin²θ = 1(第 11 课:det=1 面积不变,纯旋转)。

正交矩阵的福利:逆=转置 上一课(13 课)用到的性质在这里兑现:旋转矩阵的逆就是它的转置(R⁻¹ = Rᵀ)。几何上,「转回去」就是转 −θ,而 R(−θ) 恰好等于 R(θ) 的转置(因为 cos(−θ)=cosθ、sin(−θ)=−sinθ,正好把反对角的负号对调)。所以求旋转的逆不用真去解逆矩阵,转置一下就行——又快又稳。

接回引擎:UV 旋转、Flipbook、贴花

2D 旋转天天用 材质里旋转 UV(转贴图、做风车/旋涡、Flipbook 朝向)用的就是这个 2×2 矩阵——UE 的 CustomRotator 节点内部正是 cos/sin 拼矩阵。Sprite 粒子的旋转、贴花朝向、屏幕空间 UI 转动,也都是 2D 旋转。记住:转 UV 前通常要先把 UV 平移到中心 (0.5,0.5) 再转、再移回去,否则会绕 UV 原点 (0,0) 转——这正是第 12 课「绕原点 vs 绕自身」的老问题在 UV 上重演。 
// HLSL:绕中心旋转 UV(先移到中心、转、再移回)
float  s = sin(theta), c = cos(theta);
float2x2 R = float2x2(c, -s, s, c);     // 旋转矩阵
float2 uv2 = mul(R, uv - 0.5) + 0.5;   // 绕 (0.5,0.5) 转

常见误区

  • 记错正负号的位置。 标准(逆时针)是 [c, −s; s, c]。符号反了就变成顺时针/转反方向。记不住就用画布对一下方向。
  • UV 不移到中心就转。 直接转 UV 会绕 (0,0) 角落转,贴图飞走。先 −0.5、转、再 +0.5。
  • 角度单位混了。 sin/cos 吃弧度,引擎面板常给角度。该转 radians 别忘(第 2 课)。
  • 以为旋转会改大小。 纯旋转 det=1、不缩放。如果转完物体变大变小,是混进了缩放或写错了矩阵。
