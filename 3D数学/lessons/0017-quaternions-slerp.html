四元数与 slerp:不会万向锁、能平滑插值的旋转

四元数听起来玄(四个数、虚部、复数推导),但 TA 不需要啃数学证明。我们只学三件事:它是什么、它解决了上一课的什么坑、怎么用 slerp 做平滑旋转插值。

本课目标(一个收获) 你能把四元数理解为「绕某根轴转某个角」的紧凑打包(轴 + 角)。它的好处:不会万向锁、叠加旋转只需相乘、能用 slerp 平滑插值(沿最短弧匀速转,不会突跳)。你不用手算四元数,但要知道何时该用它、以及 slerp 比 lerp 好在哪。

一句话直觉:轴 + 角

忘掉「四个数」「虚部」。抓住一个几何事实:任何 3D 旋转,都可以表达成「绕某一根轴 n̂,转某个角 θ」(这叫轴角表示)。四元数就是把这个「轴 + 角」紧凑编码进四个数:

q = ( cos(θ/2) , n̂ · sin(θ/2) )　　前 1 个数是「转多少」,后 3 个数是「绕哪根轴」

为什么是 θ/2(半角)?这正是四元数的数学魔法所在,也是它避免万向锁的关键——但你不需要推导它。需要记住的是:四元数 = 一根轴 + 一个角 的打包,没有「三个嵌套环」,所以根本不存在环重合的万向锁。

它解决了上一课的两个坑

欧拉角的坑四元数怎么绕开
万向锁(中轴 90° 丢自由度)没有嵌套轴,绕单一轴转,不会丢自由度
顺序敏感、表示不唯一叠加旋转=四元数相乘,干净;朝向表示基本唯一(q 和 −q 同朝向)
插值会突跳slerp 沿球面最短弧匀速转,平滑无突跳

代价是:四元数不直观,人没法盯着四个数想象朝向。所以实践里的分工是:欧拉角给人看/手调,四元数给机器算(叠加、插值、连续转动)。

slerp:为什么不用普通 lerp

你已经在第 3 课学过 lerp(线性插值)。要在两个朝向之间平滑过渡(相机转向、骨骼补间、粒子对齐),能不能直接对四元数做 lerp?不行——直接 lerp 会让旋转中间变快变慢(不匀速),还会因为没归一化而轻微缩放。

正确工具是 slerp(spherical linear interpolation,球面线性插值):它把两个朝向看成球面上的两个点,沿着最短的大圆弧、匀速滑过去。结果是恒定角速度的平滑旋转——这正是相机转向、动画过渡想要的手感。

lerp:走直线弦,转速忽快忽慢　|　slerp:走球面弧,转速恒定、走最短路
// 概念:在两个朝向间平滑转过去,t 从 0→1
Quat from = ...;          // 起始朝向
Quat to   = ...;          // 目标朝向
Quat now  = Slerp(from, to, t);   // 沿最短弧匀速插值
// 用于:相机平滑转向、骨骼补间、Niagara 粒子朝向渐变对齐
接回引擎 UE 里旋转的内部类型 FQuat 就是四元数;蓝图/C++ 的 RInterp/Slerp、动画混合、IK、相机平滑跟随,底层都是四元数 slerp。Niagara 里让粒子渐渐对齐到某方向(而不是瞬间 snap)、让朝向插值不抽搐,也是这个。记住信号:凡是「旋转要平滑过渡、要叠加、要连续转」就用四元数 + slerp;凡是「给人读/手填一个朝向」就用欧拉角。两者随时可互转。

一个常见小坑:最短路与 −q

四元数 q 和 −q 表示同一个朝向(转 θ 和转 θ+360° 一样)。但做 slerp 时,如果起点和终点的符号不一致,插值可能绕远路(走长弧、甚至快转一整圈)。所以 slerp 实现里通常会先检查两端点积、必要时把一端取反,保证走最短弧。「相机本该小转一下却猛甩一大圈」有时就是这个没处理好。

常见误区

  • 想用脑子直接读四元数的朝向。 别。四元数给机器算,读朝向看欧拉角或轴角。
  • 对旋转用 lerp 而不是 slerp。 lerp 转速不匀、会缩放。朝向插值用 slerp。
  • 忘了四元数要归一化。 单位四元数才表示纯旋转;累乘多次后要 normalize,否则漂移。
  • 没处理 q/−q 最短路。 slerp 前不对齐符号,可能绕远路猛甩。点积为负就取反一端。
