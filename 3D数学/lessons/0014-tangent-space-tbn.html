切线空间与法线贴图:TBN 三件套

法线贴图为什么整体偏蓝?「法线贴图发黑」到底是什么坏了?答案都在切线空间和它的三根基向量 T、B、N 里。这是模块 2 的收官,把空间、基、矩阵全部用在一个真实的 TA 场景上。

本课目标(一个收获) 你能说清:法线贴图存的是切线空间里的法线(所以偏蓝)。TBN 是表面上每个点的一组基(切线 T、副切线 B、法线 N),把它们排成列就是一个矩阵,能把切线空间的法线搬到世界空间去和光照计算。这正是第 10 课「基向量按列排成矩阵」的实战。

为什么法线贴图整体偏蓝

法线是单位向量,三个分量范围是 [-1, 1]。贴图像素只能存 [0, 1],所以编码时做了映射:颜色 = 法线 × 0.5 + 0.5。一个「没有凹凸、垂直于表面」的法线在切线空间里是 (0, 0, 1)(纯朝外),编码后变成颜色 (0.5, 0.5, 1.0)——这正是那种淡蓝紫色。

切线空间法线 (0, 0, 1) ── ×0.5+0.5 ──▶ 颜色 (0.5, 0.5, 1.0) = 法线贴图的「蓝底」

所以法线贴图偏蓝不是 bug,是因为大部分表面是平的、法线朝正外(Z=1),编码后 B 通道拉满。贴图里偏红偏绿的地方,才是有凹凸、法线被掰歪的细节。

切线空间是什么:每个点自带的小坐标系

关键问题:法线贴图存的法线,是相对谁的?如果存世界空间法线,模型一转、一动,贴图就废了。所以它存的是切线空间——一套贴在表面、随表面走的局部坐标系。无论模型怎么摆,(0,0,1) 永远表示「沿表面朝外」,贴图因此能复用。

这套局部坐标系由三根基向量定义,就是 TBN:

基向量含义方向
T 切线 Tangent沿 UV 的 U 方向(横向贴着表面)对应贴图 R 通道
B 副切线 Bitangent沿 UV 的 V 方向(纵向贴着表面)对应贴图 G 通道
N 法线 Normal垂直于表面朝外对应贴图 B 通道

T 和 B 贴着表面、由 UV 走向决定;N 是上一课那根垂直法线。三者通常互相垂直,构成一组正交基(第 10 课)——这就是为什么 T 和 B 叫「切」线:它们和表面相切。

TBN 矩阵:把切线空间搬到世界

第 10、11 课的核心:把三根基向量按列排好,就是一个矩阵。把世界空间下的 T、B、N 按列排成 3×3,就得到 TBN 矩阵。它能把切线空间里的向量变换到世界空间:

N_world = TBN · N_tangent　=　[ T | B | N ] · (贴图解出的法线)

直觉:N_tangent 是「T 方向几份 + B 方向几份 + N 方向几份」的配方(线性组合)。TBN 矩阵拿这个配方,用世界空间的 T、B、N 重新组装,得到的就是这根法线在世界里真正的指向。然后才能和世界空间的光方向做 N·L(第 7 课)算明暗。 

// 概念:解码法线贴图 → 用 TBN 搬到世界 → 才能和世界光照算
float3 n_tan = tex.Sample(s, uv).xyz * 2.0 - 1.0;  // 解码:颜色 → [-1,1] 法线
float3x3 TBN   = float3x3(T, B, N);                    // 三根基按列(注意引擎约定)
float3 n_world = normalize(mul(TBN, n_tan));         // 搬到世界,再归一化
float ndotl  = saturate(dot(n_world, L));           // 现在才能正确算光照

接回引擎:法线贴图发黑到底是什么

TA 的高频翻车 「法线贴图一接上去物体就发黑 / 出现死黑斑」,几乎都是空间没对齐(第 9 课的老问题):把切线空间法线没经过 TBN 就直接当世界法线去和光照算,N·L 大面积变负被裁成 0,于是死黑。其它常见诱因:贴图被当成 sRGB 解码(法线贴图必须标记为线性/非颜色数据,否则数值被 gamma 扭曲)、UV 镜像导致副切线方向翻转(B 的手性反了,凹凸内外颠倒)。UE 里法线贴图取样节点已经把 TBN 和解码包好了,但你自定义 HLSL 解法线贴图时,这三件套要自己接对。

常见误区

  • 以为法线贴图存的是世界法线。 它存切线空间法线(所以偏蓝),必须经 TBN 才能进世界光照。
  • 忘了解码 ×2-1。 贴图是 [0,1],法线是 [-1,1]。漏了这步,法线全挤在正方向,凹凸消失。
  • 法线贴图当 sRGB 采样。 它是数据不是颜色,要标线性/非颜色,否则发黑发花。
  • UV 镜像不翻副切线。 镜像 UV 会让 B 手性反转,凹凸内外颠倒,需要按 UV 朝向修正 B 的符号。
