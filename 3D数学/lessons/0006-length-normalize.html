长度与归一化:把箭变成纯方向

length 不过是勾股定理的推广,normalize 把任意箭压成「长度=1 的纯方向」。这一课讲透:为什么法线、光照方向、朝向,几乎都必须先归一化。

本课目标(一个收获) 你能算并理解 |a| = √(x² + y² + z²)(勾股定理),以及 normalize(a) = a / |a|(除以自己的长度,得到方向不变、长度=1 的单位向量)。并明白:凡是只关心「方向」的地方,都要归一化;不归一化会让光照、点积全部出错。

长度就是勾股定理

一支箭有多长?在 2D 里,(x, y) 这支箭和它的水平、竖直分量组成一个直角三角形,箭是斜边。勾股定理:斜边² = 两直角边² 之和,所以:

|a| = √(x² + y²)　　3D 直接加一项:|a| = √(x² + y² + z²)

还记得第 2 课的 sin² + cos² = 1 吗?那就是单位圆上(半径 1 的箭)的同一个勾股定理。长度公式只是把它推广到任意长度、任意维度。拖下面的红点,看 |a| 怎么由两个分量算出来。

HLSL 速记 length(a) 求模;distance(a, b) = length(b − a) 求两点距离。 开方较贵,只比大小时可用 dot(a, a)(= |a|²,下一课讲)省掉 sqrt——「谁离得更近」用平方距离比就够。

归一化:除以自己的长度

很多时候我们只要方向、不要长度:法线只说「面朝哪」、光照方向只说「光从哪来」。这时把向量除以它自己的长度,就得到一支方向不变、长度恰好为 1 的箭——单位向量 unit vector。这一步叫 normalize:

normalize(a) = a / |a| = ( x/|a| , y/|a| , z/|a| )　　结果长度 = 1

下面画布显示 a 归一化后的结果:方向和 a 一模一样,长度被压到 1。拖动 a(改方向、改长短),看归一化结果的方向跟着转,但长度永远是 1。

为什么必须归一化:点积与光照会骗你

下一课你会学到 N·L 算光照强度:dot(法线, 光向)。这个公式只有在两者都是单位向量时才等于 cos(夹角)——也就是才表示「正对光多少」。如果法线没归一化、长度是 2,算出来的光照就凭空亮一倍;长度是 0.3,就暗一截。非单位向量会把「方向信息」和「长度信息」混在一起,污染结果。 

float3 N = normalize(worldNormal);   // 插值后的法线长度会漂,必须重新归一化
float3 L = normalize(lightDir);
float  ndotl = saturate(dot(N, L)); // 现在它才真的=正对光的程度
高频坑:插值会破坏长度 顶点法线传到像素时被插值,两个单位向量的中间值长度会小于 1(想象两支箭取中点,会缩进去)。所以 shader 里几乎所有方向量在用之前都要重新 normalize。「法线贴图发暗 / 光照不对」十有八九是漏了这一步。

归一化的两个陷阱

零向量不能归一化:(0,0,0) 长度是 0,除以 0 会得到 NaN(非数),整个像素/粒子可能变黑或消失。当方向可能为零时(比如「指向目标」而粒子正好在目标上),要先判长度或加个极小量。

已经是单位向量就别重复归一化省点性能,但拿不准时宁可多归一化一次——错误的方向比一点点性能贵得多。Niagara 里给粒子「沿运动方向拉伸」「朝速度方向喷射」前,记得把 Velocity normalize 成纯方向再用。

接回引擎 常见配方:dir = normalize(Target − Position) 得到纯方向 → Velocity = dir * Speed 给定速度大小。方向(normalize)和大小(× Speed)分开管,是 Niagara 控制运动最清晰的套路。Sprite 朝向、Ribbon 切线、各向异性拉伸,也都要先归一化方向。

常见误区

  • 用方向向量前忘了 normalize。 插值/缩放会让长度漂移,直接拿去 dot/光照会算出错误强度。方向量用前先归一化。
  • 对可能为零的向量归一化。 零向量除以 0 = NaN,会让像素变黑或粒子消失。先判长度或加 epsilon。
  • 以为 normalize 改了方向。 它只改长度(压到 1),方向完全不变。要转方向得用旋转。
  • 比远近也开方。 只比大小时用 dot(a,a)=|a|² 即可,省掉 sqrt;真正要数值长度时才用 length。
