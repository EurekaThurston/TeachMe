坐标空间:同一个点,好几个地址

「东西显示在错的位置 / 朝向歪了 / 法线贴图发黑」——TA 八成的翻车都在这。根因只有一句话:同一个点,在不同坐标空间里有不同坐标。这一课先把这个概念彻底讲清,后面矩阵才有意义。

本课目标(一个收获) 你能说清:一个坐标空间 = 一套「原点 + 基」;同一个物理点在 Local / World / View / Tangent 等不同空间里,坐标值不同。并知道:把数从一个空间搬到另一个空间叫变换;只有同一空间的向量才能直接比较、相加。

回到第 1 课那句种子

第 1 课说过:坐标不是点本身,而是点相对于某套规则的地址。现在把它兑现。一个 3D 场景里,同一个物理点(比如角色右手的指尖)可以被好几套规则同时编号,每套给出一组不同的数。每套规则,就是一个坐标空间。

打个生活类比:你家这个位置,可以说「客厅东北角」(相对你家)、也可以说「某街 12 号」(相对城市)、还可以说「我前方 3 米」(相对你站的地方)。同一个地点,三个地址,都对——取决于你以谁为原点、用哪套轴。坐标空间就是这么回事。

TA 每天打交道的几个空间

空间原点在哪谁的坐标住在这
Local / Object 物体空间模型自己的轴心(pivot)网格顶点、建模时的法线/UV
World 世界空间关卡全局原点Actor 摆放位置、World 模式的粒子
View / Camera 视图空间相机「离相机多远」「在屏幕左右」
Tangent 切线空间表面上每个点,贴着表面法线贴图里存的法线(第 14 课)
Clip / Screen屏幕最终像素位置

顶点从「Local 出生」一路被变换到「屏幕上的像素」,中间经过 World → View → Clip,这条链叫变换管线。每一步都是「换一套原点和轴重新编号同一个点」。你在材质里见到的 TransformPosition、Transform Vector 节点,干的就是这件事。

为什么混空间 = 灾难

向量只有在同一个空间里才能直接比较或相加(回忆第 5 课最后那个坑)。把 Local 空间的速度直接加到 World 位置上,就像把「客厅东北角」和「某街 12 号」这两组数硬加起来——数能算,但毫无意义,结果乱飞。

三个经典翻车,全是空间错配
  • 位置错:把 Local 坐标当 World 用,物体出现在原点附近 / 跟着父物体乱跑。
  • 朝向歪:在错误空间里算方向,粒子朝向不对、风往错的方向吹。
  • 法线发黑:法线贴图存的是切线空间法线,不转到 World 就和世界光照对不上,出现死黑或诡异明暗(第 14 课专讲)。

接回引擎:Niagara 的 Local vs World

Niagara 发射器有个开关 Local Space:

  • 勾上(Local):粒子坐标相对发射器自身原点。发射器移动/旋转,整团粒子跟着走、跟着转——适合「跟随角色的光环」「武器拖尾」。
  • 不勾(World):粒子一出生就落在世界坐标里,不再被发射器后续移动影响——适合「拖在身后的烟」「掉在地上的火星」。

「我的烟雾跟着角色瞬移」或「拖尾不拖了、黏在原地」——多半就是这个开关选错,本质是粒子住在了错误的坐标空间。材质里同理:World Position、Object Position、Camera Vector 是不同空间的量,要转到同一空间才能正确运算。 

// 材质里把一个世界方向转进切线空间,才能和法线贴图比较
float3 L_tangent = TransformWorldToTangent(lightDirWorld);
// 或反过来:把法线贴图的切线空间法线转到世界,再做光照
float3 N_world   = TransformTangentToWorld(normalMap);

常见误区

  • 以为坐标是绝对的。 没有「这个点的坐标」,只有「这个点在某空间里的坐标」。问坐标先问:哪个空间?
  • 跨空间直接加 / 比较。 Local + World、Tangent 和 World 法线直接 dot——都是错配。先变换到同一空间。
  • Niagara Local/World 选反。 跟随类效果用 Local,遗留类(烟、火星)用 World。选错就「跟错了 / 不跟了」。
  • 把方向也当位置变换。 位置变换含平移,方向不该被平移(下一课基向量、第 11 课齐次坐标 w 会解释)。
