3D 旋转、欧拉角与万向锁

2D 只有一个角,3D 要绕三根轴转。这带来两个新麻烦:顺序又开始要紧,以及那个让无数 TA 抓狂的 万向锁(Gimbal Lock)。理解它们,就知道下一课的四元数到底在解决什么。

本课目标(一个收获) 你能说清:欧拉角用三个角(绕 X/Y/Z,即 Pitch/Yaw/Roll)描述 3D 朝向,直观但有两个坑:顺序不可交换(绕轴次序不同结果不同),以及万向锁——当中间那根轴转到 ±90°,两根旋转轴会重合,丢掉一个自由度,转起来「卡住/突跳」。这就是四元数存在的理由。

三个角:Pitch / Yaw / Roll

2D 平面里只有一种转法(绕「出屏幕」那根轴)。到了 3D,你可以绕三根轴各转一个角:

绕哪根轴名字直觉(以飞机/角色为例)
绕 X 轴Pitch 俯仰抬头 / 低头
绕 Y/Z 轴Yaw 偏航左转头 / 右转头
绕 forward 轴Roll 翻滚侧身翻 / 歪头

把三个 2D 旋转矩阵(上一课那种,扩到 3×3)连乘,就得到一个 3D 旋转。UE Transform 面板里的 Rotation (X, Y, Z) 三个数,就是欧拉角。它直观、好调、好读——这是它的优点。

注意:UE 的轴约定 不同软件 Pitch/Yaw/Roll 对应的轴不一样。UE 是左手系、X=前 Y=右 Z=上:Roll 绕 X、Pitch 绕 Y、Yaw 绕 Z。别背死「Pitch 一定绕 X」,记住几何意思(抬头/转头/侧翻),用时对一下当前软件的轴。

坑一:顺序不可交换

和第 12 课一样,旋转也是矩阵相乘,不可交换。「先 Pitch 再 Yaw」和「先 Yaw 再 Pitch」会把物体转到不同朝向。所以「欧拉角 (30, 45, 60)」这串数字,只有在说清楚旋转顺序(XYZ?ZYX?)时才有确定含义。不同 DCC 用不同默认顺序,这是跨软件导朝向时朝向乱掉的常见根因。

R_x · R_y · R_z　≠　R_z · R_y · R_x　　（绕轴顺序一换,最终朝向就变）

坑二:万向锁(Gimbal Lock)

这是欧拉角的致命伤。想象三个嵌套的旋转环(像陀螺仪那种):外环、中环、内环,各管一个角。正常时三环互相垂直,能转出任意朝向。但当中间那根轴转到 ±90°,内环和外环的旋转轴会对齐重合——这时转内环和转外环效果一样,你丢了一个独立的转动方向。

后果对 TA 很实在:

  • 转着转着「卡住」:某个方向怎么调都转不动,因为那个自由度没了。
  • 插值时突然翻转/抽搐:动画里相机或骨骼经过万向锁姿态,朝向会突跳(snap)。「相机俯仰到正上方时画面猛地一甩」就是典型。
  • 同一个朝向有多组欧拉角:数值表示不唯一,插值和叠加都容易出鬼。
中轴 → ±90° ⇒ 内外两轴重合 ⇒ 丢一个自由度 ⇒ 卡死 / 插值突跳

接回引擎:你为什么会撞上它

TA 现场 「Niagara 里粒子绕轴转着转着方向就乱了」「相机/骨骼 Pitch 接近 90° 时画面猛甩」「把两个旋转加起来朝向发疯」——这些常常就是欧拉角的顺序问题或万向锁。引擎内部其实用四元数(或矩阵)存旋转、做插值,只是在面板上显示成欧拉角方便你读。所以一旦涉及旋转的叠加、插值、连续转动,尽量在四元数层面做,只把欧拉角当「人类可读的输入/输出」。下一课就讲四元数怎么绕开这两个坑。 
// 欧拉角直观好读,但叠加/插值容易出万向锁与顺序问题
// 概念:三个轴旋转连乘(顺序写死才有确定含义)
float3x3 R = mul(Rz, mul(Ry, Rx)); // ZYX 顺序;换顺序结果就不同
// 实战:连续转动/插值请用四元数(下一课),欧拉角只作输入输出

常见误区

  • 以为欧拉角三个数就唯一确定朝向。 没说顺序就没定义;同一朝向还可能有多组欧拉角。
  • 跨软件直接搬欧拉角。 轴约定和默认顺序不同,导进来朝向就歪。要么转矩阵/四元数,要么对齐约定。
  • 在欧拉角上做插值。 经过万向锁会突跳。旋转插值要在四元数上做 slerp(下一课)。
  • 把万向锁当 bug 去 debug 数值。 它是欧拉角表示法的固有缺陷,不是哪行代码写错。换表示法(四元数)才是解法。
