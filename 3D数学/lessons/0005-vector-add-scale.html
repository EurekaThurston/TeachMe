3D 数学 for VFX · 第 5 课 · 模块 1 向量

加减与缩放:位移、速度 × dt

向量的三种基本运算——加、减、标量乘——每一种都有清晰的几何画面。学完你能亲手推出 Niagara 那句灵魂代码 Position += Velocity * dt,而不是死记。

本课目标(一个收获) 你能在脑里画出三种运算:a + b = 首尾相接(走完 a 再走 b)、b − a = 从 a 指向 b 的箭、k·a = 把 a 伸长/缩短 k 倍(负数则反向)。并用它们解释每一帧粒子是怎么动的。

加法:首尾相接

a + b 就是分量分别相加:(ax+bx, ay+by, az+bz)。几何上:先走完 a,再从 a 的终点接着走 b,最终落点就是 a+b。顺序无所谓(a+b = b+a),因为不管先走哪条,围出的是同一个平行四边形,对角线一样。

下面画布开了「sum」:黑色粗箭是 a+b,虚线是把 b 平移到 a 末端(首尾相接)。拖红/绿点,看 a+b 怎么跟着变。

接回引擎 Niagara 里所有「力的叠加」都是向量加法:重力 + 风 + 涡流 = 合力。Solve Forces and Velocity 模块本质就是把一堆力向量加起来,再加到速度上。你往粒子上挂的每个力模块,都是在做一次 a+b。

减法:从哪指向哪

b − a 是「从 a 指向 b」的箭。记忆法:终点减起点。想要「从粒子指向目标」的方向?就是 target − particlePos。这是 TA 最高频的操作之一——求朝向、求吸引方向、求两点连线,全靠它。

b − a = 从 a 出发、指到 b 的位移　　(箭头落在 b,尾巴在 a)

方向记反是头号 bug:a − b 和 b − a 长度相同、方向正好相反。粒子「冲向目标」要 target − pos,写成 pos − target 就变成「逃离目标」。下一课归一化之后,这个箭会变成纯方向。

标量乘:伸缩与反向

k · a 把向量每个分量乘以 k,几何上是沿原方向伸长或缩短:k=2 长度翻倍、k=0.5 减半、k=0 变零向量、k=−1 长度不变但掉头。方向只有「同向」或「反向」两种结果,标量乘不会改变向量指的那条直线(除非乘 0)。 

float3 back   = -1.0 * forward;     // 反向:朝后
float3 half   = 0.5 * velocity;     // 速度减半
float3 scaled = velocity * deltaTime;  // 速度×时间 = 这一帧的位移!

把三者接起来:推导 Position += Velocity × dt

这是欧拉积分,游戏里每个会动的东西每帧都在做。现在你有能力自己推它,而不是背:

  1. 速度 Velocity 是一支「方向+大小」的箭,单位是 cm/秒(往哪、每秒多快)。
  2. 这一帧只过了 dt 秒(deltaTime,比如 1/60 ≈ 0.0167 秒)。
  3. 标量乘:Velocity × dt = 这一帧实际走的位移(一支短箭)。速度是「每秒多远」,乘上「过了几秒」就得到「这帧走了多远」。
  4. 向量加:把这段位移接到当前位置上 —— Position + Velocity × dt = 新位置。
newPos = Position + Velocity × dt　　(位置 加 「速度 乘 时间」)
// Niagara 每帧对每个粒子做的事(Solve Forces and Velocity 内部)
Particles.Position += Particles.Velocity * Engine.DeltaTime;
为什么必须乘 dt 不乘 dt,粒子每帧走一个速度的距离——帧率越高跑得越快,游戏在不同机器上表现不一致。乘了 dt,粒子每秒走固定距离,和帧率脱钩。这叫帧率无关 framerate independence,是所有运动代码的铁律。

常见误区

  • 减法方向写反。 「从 A 指向 B」= B − A(终点减起点)。写成 A − B 就反了,吸引变排斥。
  • 忘记乘 dt。 Position += Velocity(漏 dt)会让运动随帧率乱跳。任何「每秒多少」的量加进状态前都要 ×dt。
  • 以为标量乘会改方向。 正的 k 只伸缩、不转向;只有负 k 会掉头 180°。要任意转向得用旋转(模块 3),不是乘标量。
  • 分量没对齐就加。 只有同维、同坐标系的向量才能相加。把 Local 空间的速度直接加到 World 位置上是空间错配(模块 2)。
