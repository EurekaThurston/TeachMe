坐标系与空间:给每个点编号

在算任何东西之前,先得能指着空间里一个点说出它叫什么。坐标系就是这套编号规则——也是后面所有数学的舞台。

本课目标(一个收获) 你能说清楚:一个坐标系 = 一个原点 origin + 几条带方向和刻度的轴 axis;空间里每个点都被一组数(坐标)唯一编号。并且记住 UE 的约定:左手系,X=前、Y=右、Z=上,单位 cm;UV 只是一个 2D 坐标系。

从一条数轴说起

最小的坐标系是一条数轴:选一个点叫 0(原点),选一个方向叫正、定一个刻度叫 1。现在这条线上每个点都有一个名字——它的坐标。「3」就是从 0 朝正方向走 3 格的那个点。

关键在于:坐标不是点本身,而是点相对于这套规则的「地址」。换一个原点或换一个刻度,同一个物理点的坐标就变了。这句话现在听着像废话,但它是模块 2「坐标空间」整章的种子——记住它。

加一根轴:2D 平面

再拉一条和第一条垂直的轴,共享同一个原点,就得到一个 2D 坐标系。现在描述一个点要两个数:(x, y)。横着走 x 格,竖着走 y 格,就到了。

你已经天天在用这个,只是没把它当数学:UV 就是一个 2D 坐标系。贴图左下角是 (0, 0),右上角是 (1, 1),uv = (0.5, 0.5) 是正中间。材质里你对 UV 做的一切——平铺、偏移、旋转——都是在这个 2D 坐标系里搬点。

再加一根轴:3D 空间

第三条轴垂直于前两条,描述一个点就要三个数 (x, y, z)。这就是「3D」的字面意思:three dimensions,三个独立的方向、三个数。粒子的位置、顶点的位置、相机的位置,全是 3D 坐标。

接回引擎 Niagara 里粒子的 Position 是 Vector(三个 float:X、Y、Z)。当你写 Particles.Position 时,你拿到的就是这个粒子在某个坐标系里的三数地址。它在哪个坐标系(Local 还是 World)——是模块 2 的核心问题,这里先埋个钩子。

手性:左手系还是右手系(UE 的坑)

三条轴互相垂直,但有两种搭法,像左右手镜像、无法旋转重合。区别在于:当 X 指向你、Y 指向右,Z 是朝上还是朝下。这叫手性 handedness。

虚线那根是「深度轴」(指向你/离开你)。判手性的关键就看深度轴朝哪:右手系里 X右、Y上,则 Z 朝你(出屏);UE 左手系里 Y右、Z上,则 X(前)朝屏幕里(进屏)。同样三根轴,深度轴反一下,手性就反。

为什么 TA 必须知道这个?因为手性决定了叉积的方向(第 8 课)和「正向旋转」是顺时针还是逆时针(模块 3)。你照搬一段网上的 shader 公式,发现法线朝里、旋转反了——十有八九是手性 / 轴向约定不一致。Unreal 用 左手系、Z 朝上、单位厘米,这和很多教程(右手、Y 朝上)不同,搬代码时要会换。

常见误区

  • 把坐标当成点的「本质」。 坐标永远是相对某个原点+轴的地址。换坐标系,数就变,点没变。这是后面不踩空间坑的根。
  • 默认 Y 朝上。 屏幕、UV、很多教程 Y 朝上;但 UE 世界空间 Z 朝上。混用会让你「往上」加错了轴。
  • 忽略单位。 UE 世界单位是厘米。一个粒子速度填 100,是 100 cm/s = 1 m/s,不是 100 米。
  • 以为 UV 和世界坐标是一回事。 UV 是贴图上的 2D 坐标 [0,1],和物体在世界里的 3D 位置是两套独立的坐标系。
