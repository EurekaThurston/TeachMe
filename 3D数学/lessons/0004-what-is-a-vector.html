向量是什么:位置 vs 方向

向量是 3D 数学的心脏。点、方向、速度、法线、甚至颜色,全是向量。这一课先建立最关键的双重身份:同一组数,什么时候是「一个点」,什么时候是「一支箭」。

本课目标(一个收获) 你能说清:向量就是一组有序的数 (x, y, z),它有两种读法——位置(从原点出发到达的一个点)或方向+大小(一支可以平移的箭)。并能在 HLSL 的 float2/3/4 与 Niagara 的 Vector 里认出它,会用 swizzle。

向量 = 一组有序的数

最朴素的定义:向量就是几个数排成一排。2 个数是 2D 向量 (x, y),3 个是 3D (x, y, z),4 个是 4D (x, y, z, w)。「有序」很重要:(3, 0) 和 (0, 3) 完全不同,位置/方向都不一样。每个数叫一个分量 component。

就这么简单。难的不是「是什么」,而是同一组数能代表两种截然不同的东西——这是新人最容易糊的地方,也是这节课的全部重点。

读法一:位置(一个点)

把 (4, 3) 读成位置:从原点出发,走到 x=4、y=3 的那个点。它钉在空间里,起点必须是原点。粒子的 Position、顶点坐标、相机位置,都是这种读法——它们回答「在哪」。

读法二:方向+大小(一支箭)

把同样的 (4, 3) 读成位移:朝某个方向、走某段长度的一支箭。这支箭不钉在原点——它只在乎「往哪、多长」,放在空间任何地方都是同一支箭。速度、法线、力、光照方向,都是这种读法——它们回答「往哪、多猛」。

一句话记牢 位置 = 钉在原点的点(在哪);方向 = 能自由平移的箭(往哪+多长)。 同一组数 (4,3),读法不同,用法天差地别。下一课的「加减」之所以有两种几何含义,根就在这。

在引擎里它长什么样

HLSL 里向量是 float2 / float3 / float4,Niagara 里是 Vector(3 float) 或 Position。同一个概念,两套包装:

float3 pos    = float3(4, 3, 0);  // 当位置:某个点在哪
float3 normal = float3(0, 0, 1);  // 当方向:面朝哪(下节课会归一化)
float4 color  = float4(1, 0, 0, 1);  // 颜色也是 4D 向量 RGBA
接回引擎 在 Niagara 里你早就在用向量了:Particles.Position(位置读法)、Particles.Velocity(方向+大小读法)。它们类型都是 Vector,但一个是「在哪」、一个是「往哪多快」。把这两种读法分清,后面调力场、朝向才不会乱。

Swizzle:重排和复制分量

shader 有个好用的语法糖:swizzle——用 .x .y .z .w 任意取、重排、复制分量。这在 TA 日常里极高频: 

float3 v = float3(1, 2, 3);
v.xy        // → float2(1, 2) 只取前两维
v.zyx       // → float3(3, 2, 1) 反序
v.xxx       // → float3(1, 1, 1) 把 x 复制三份(灰度转 RGB 常用)
v.rgb       // → 和 .xyz 等价,颜色语境的别名

颜色还可以用 .rgba 这套别名,和 .xyzw 完全等价,纯粹是读着顺。v.xxx 把一个灰度值铺成 RGB 三通道,是你以后会写无数次的小动作。

常见误区

  • 把方向向量当点用(或反过来)。 「法线 (0,0,1)」是方向,别把它当成「在 z=1 的位置」。混了会导致平移作用到方向上(模块 2 齐次坐标 w 就是来防这个的)。
  • 以为方向向量钉在原点。 方向箭可以平移到任何地方,只看「往哪、多长」。画图时从原点画只是为了看分量。
  • 分量顺序记错。 (x, y, z) 有序;UE 里还要记得 X=前、Y=右、Z=上。把 Y 当成「上」会让东西躺下。
  • 忘了颜色也是向量。 RGBA 就是 float4,能加减乘、能 lerp、能 dot。把颜色当向量算,很多材质技巧就通了。
