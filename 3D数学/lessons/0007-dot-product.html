点积 dot:一个乘加,读出「多同向」

点积是整个 shader 世界最常用的运算,没有之一。兰伯特光照、菲涅尔、阴影、遮罩——核心都是它。这一课让你看到:dot 就是在量「两个方向有多同向」。

本课目标(一个收获) 你能同时握住点积的两副面孔:代数 a·b = ax·bx + ay·by + az·bz(逐项乘再相加,结果是一个标量);几何 a·b = |a||b|cosθ。并会用符号读关系:>0 同侧、=0 垂直、<0 反向;单位向量时 a·b 直接就是 cos 夹角。

两个公式,同一件事

点积有两种算法,结果一模一样,但各有用处:

代数:a·b = ax·bx + ay·by + az·bz　（好算,GPU 一条指令）
几何:a·b = |a| |b| cos θ　（好懂,θ 是两向量夹角）

代数式告诉你怎么算(几个乘加),几何式告诉你它意味着什么(和夹角的 cos 成正比)。把两式连起来,就能反求夹角:cosθ = a·b / (|a||b|)。如果 a、b 都已归一化(长度 1),那干脆 a·b = cosθ,一步到位。

符号就是关系:同向 / 垂直 / 反向

点积最实用的地方是看符号。因为 |a||b| 总是正的,a·b 的正负完全由 cosθ 决定:

a·bcosθ夹角几何含义
> 0正< 90°大致同向(同一侧)
= 00= 90°恰好垂直
< 0负> 90°大致反向(对侧)

拖动下面的 a、b,盯住 a·b 的数值和夹角:让它们越靠拢,dot 越大;转到垂直,dot 归零;转到对着干,dot 变负。橙色虚线是 a 在 b 上的投影——点积的另一种几何解读:「a 在 b 方向上有多长的影子」×|b|。

杀手级应用:N·L 兰伯特光照

一个表面被光照得多亮,取决于它多正对光源。正对(法线指着光)最亮,侧着变暗,背着光全黑。这正是「两个方向多同向」——点积量身定做的活: 

float3 N = normalize(worldNormal); // 表面朝向(单位向量)
float3 L = normalize(toLight);     // 指向光源的方向(单位向量)
float diffuse = saturate(dot(N, L)); // =cosθ,正对=1 侧面=0 背光被 saturate 夹成 0

dot(N,L) 因为两者是单位向量,直接等于 cos(夹角):正对时 1(全亮)、90° 时 0、背光时负数。最后 saturate 把负的背光夹成 0(不可能负亮度)。这一行就是兰伯特漫反射,实时渲染最基础的光照模型——你现在能从头解释它了。

点积的 TA 工具箱
  • 菲涅尔边缘光:1 - saturate(dot(N, V))(V=看向相机的方向)。正对相机→dot≈1→菲涅尔≈0;边缘→dot≈0→菲涅尔≈1,于是边缘发亮。
  • 朝向遮罩:只想让「朝上的面」长草/积雪?saturate(dot(N, float3(0,0,1))) 当遮罩。
  • |a|² = dot(a, a):平方长度,比远近省掉开方(上一课提过)。
  • 投影:a 在单位向量 b 上的投影长度 = dot(a, b)。分解速度到某个轴常用。

常见误区

  • 以为点积结果是向量。 点积出来是一个标量(数),不是向量。要垂直向量请用叉积(下一课)。
  • 没归一化就把 dot 当 cosθ。 只有单位向量时 a·b 才等于 cosθ。法线/光向先 normalize,否则光照亮度被长度污染。
  • 忘了 saturate 背光。 背光时 dot 是负数,直接当亮度会出现「负光」诡异结果。漫反射要 saturate(dot) 夹到 0。
  • 菲涅尔方向搞反。 菲涅尔要的是「边缘亮」,用 1 - dot(N,V);直接 dot(N,V) 是「正面亮」,正好相反。
