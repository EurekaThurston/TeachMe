数值管道:把一个数变成另一个数

TA 每天的工作,一大半是「把一个数(时间、距离、UV)映射成另一个数(强度、透明度、混合)」。这一课把五个最常用的映射函数一次讲透,做软边渐变从此不靠瞎调。

本课目标(一个收获) 掌握五个基本映射工具的几何含义:lerp(按比例混合)、clamp / saturate(限制范围)、smoothstep(平滑过渡 / 软边)、frac(取小数 / 重复)。看到它们能立刻在脑里画出输入→输出的曲线。

lerp:一切混合的母函数

lerp(a, b, t)(linear interpolation,线性插值)是这一切的核心。它在 a 和 b 之间按比例 t 取一个值:

lerp(a, b, t) = a + (b − a) · t　　t=0 → a　,　t=1 → b　,　t=0.5 → 正中间

拖滑块感受:t 就是「从 a 走到 b 走了百分之几」。这条规则对数、对颜色、对向量、对位置全部通用——混合、淡入淡出、A 到 B 的过渡,底层都是它。

接回引擎 Niagara 的颜色/大小曲线、材质的 Lerp 节点,本质都是这一个函数。粒子「从生到死颜色由白变红」= lerp(白, 红, Age/Lifetime)。你拖的那条曲线,只是把直线 t 换成了弯的 t。

clamp 与 saturate:别越界

映射出来的数常常会冲出你想要的范围(比如算出 1.4 的透明度、−0.2 的强度)。clamp(x, lo, hi) 把 x 摁回 [lo, hi]:小于 lo 给 lo,大于 hi 给 hi。shader 里 99% 的情况是夹到 [0,1],所以有个专用简写 saturate(x) = clamp(x, 0, 1)。 

float a = saturate(rawAlpha);   // 把任意值锁进 [0,1],当 alpha 安全
float d = clamp(dist, 0.0, 5.0); // 距离最多算到 5

saturate 在 GPU 上几乎免费(常和别的运算合并),所以放心用它兜底。凡是要当颜色/透明度/遮罩用的中间结果,先 saturate 一下,能挡掉一大票「超亮溢出、负值变怪」的 bug。

smoothstep:软边的灵魂

lerp 是硬的直线过渡。但特效要的常是软的:边缘羽化、渐隐、能量场的柔和外沿。这就是 smoothstep(edge0, edge1, x):当 x 从 edge0 升到 edge1,输出从 0 平滑爬到 1,而且两端是平的(导数为 0),所以没有生硬的折角。

区别一眼可见:直线匀速到顶、两端有折角;S 形起步慢、中间快、收尾慢,两端平滑接住。眼睛觉得 smoothstep「更自然、更高级」,正是因为它没有突变。做遮罩软边、光晕外沿、消融过渡,默认用它。 

// 用距离场做一个软边圆(配合后面 SDF 课)
float mask = smoothstep(0.5, 0.48, dist); // 注意 edge0>edge1 → 内部=1 外部=0
HLSL 速记 smoothstep(a, b, x):x≤a→0,x≥b→1,中间走平滑 S。a 和 b 的间距 = 软边的宽度,差得越小边越硬、越接近 step。把 a、b 反过来写(a>b),输出就反向(内部 1、外部 0),做遮罩常这么用。

frac:把任何数折回 [0, 1)

frac(x) 取小数部分:frac(3.7) = 0.7、frac(12.25) = 0.25。几何上它把整条数轴切成一段段长度为 1 的格子,再叠在一起——任何输入都被折回 [0, 1)。这是 UV 平铺(贴图无缝重复)、做周期图案、以及第 20 课随机/噪声的基石。 

uv = frac(uv * 3.0);  // 把一张贴图在 0..1 里平铺成 3×3

把 frac 和 lerp 配在一起,你就能造「锯齿波」:frac(Time) 是一条从 0 爬到 1、到顶瞬间归零再爬的信号——和上一课的 sin(平滑往返)互补,sin 给「呼吸」,frac 给「扫描/重置」。两者是程序化动画的两块基石。

把管道接起来:一个真实例子

一个「从中心扩散、边缘柔和」的能量环,就是几个映射串联:

float d    = length(uv - 0.5);          // 到中心的距离(第 6 课)
float ring = smoothstep(0.30, 0.28, d)   // 外沿软边
            - smoothstep(0.20, 0.18, d); // 内沿软边,相减得环
ring = saturate(ring);                  // 兜底锁进 [0,1]
color = lerp(bgColor, ringColor, ring);  // 用遮罩混合两色

读懂这五行,你就读懂了一大半程序化材质——它们全是「数值管道」的拼接。后面模块四会把这套玩法推到 SDF、噪声、动画。

常见误区

  • 忘了 clamp/saturate 兜底。 算出来的遮罩/强度越界(>1 或 <0)会导致溢出发白、反相发黑。当颜色/alpha 用之前先 saturate。
  • 该软边时用了 lerp / step。 生硬的边缘多半是用了线性或硬阶跃。要羽化就上 smoothstep,用 edge 间距控制软硬。
  • smoothstep 的两个 edge 写反还不自知。 a<b 是「升」,a>b 是「降」。做遮罩想要「内部亮」却写成 a<b,结果整个反掉。
  • 以为 lerp 的 t 必须在 [0,1]。 公式对任何 t 都算(t=2 会外推到 b 之外)。要安全就先 saturate(t)。
