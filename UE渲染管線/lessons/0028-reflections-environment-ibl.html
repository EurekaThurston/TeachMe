反射与环境光：Reflection Environment、天空、IBL

SSR 只能给出屏幕内的反射；屏幕外和低频的环境光照依靠预烘焙立方体图与天光，三者按优先级回退合成最终反射与间接镜面光。

时长 ~20 分钟 前置 0027（SSR）、0021（GBuffer 粗糙度/金属度） 产出 能描述反射捕获立方体图、天光 IBL、SSR 如何按层级回退合成最终反射

0027 的 SSR 精度最高，但只能给出屏幕内已渲染的内容。SSR 没有覆盖的区域（屏幕外的物体、被遮挡的几何、低频漫反射环境光）要靠 Reflection Environment（预卷积立方体图）+ Sky Light（天空光/天光）来填充。这节课讲这两个来源的数据从哪里来，如何预处理，以及三个来源（SSR、反射捕获、天光）如何在合成 Pass 里按优先级混合出最终结果。

合成规则：SSR 置信度高的区域优先用 SSR；SSR 没覆盖（置信度低/为零）的区域回退到最近的 Reflection Capture；天光作为最低频兜底，提供全方向均匀的漫反射环境光（Indirect Diffuse）和粗糙表面的镜面回退。

2Reflection Capture：预卷积立方体图

Reflection Capture（反射捕获）由场景中放置的 AReflectionCaptureActor（球形/盒形）表示。它在编辑器或运行时捕获周围场景的立方体图，并做 IBL 预卷积（Image Based Lighting）：

  • 对立方体图按 GGX 分布预积分镜面反射，结果存入 FReflectionEnvironmentCubemapArray 的不同 mip 级别；
  • mip 0 = 完全镜面（粗糙度 ≈ 0）；mip N = 完全漫反射（粗糙度 ≈ 1）；
  • 着色时按 GBuffer 粗糙度选取对应 mip，单次纹理采样即得到对应粗糙度的环境镜面值（Split Sum 近似）。
// ReflectionEnvironment.usf（IBL 采样，简化）
float Roughness = GBufferB.a;
float MipLevel  = ComputeReflectionCaptureMipFromRoughness(Roughness);
float3 ReflDir  = reflect(-ViewDir, WorldNormal);
float4 EnvSample = ReflCaptureCubemapArray.SampleLevel(
    CubemapSampler, float4(ReflDir, CaptureIndex), MipLevel);
// Split Sum 第二项：BRDF LUT 按 (NoV, Roughness) 查表
float2 EnvBRDF  = PreIntegratedGF.SampleLevel(..., 0).rg;
float3 SpecIBL  = EnvSample.rgb * (SpecularColor * EnvBRDF.x + EnvBRDF.y);

Split Sum 近似：把 GGX 积分拆成"预卷积环境"和"BRDF LUT"两项分别预计算，运行时两次采样相乘——这是 UE 自 4.x 起的标准 IBL 实现。

FReflectionEnvironmentCubemapArray

场景里所有反射捕获的立方体图打包进一个 2D 数组纹理（FReflectionEnvironmentCubemapArray）。着色时以 Capture Index 为数组层索引，避免频繁纹理切换。每个像素根据世界坐标找最近的 Capture（并在多个 Capture 间做空间混合），取出对应 index 后做上面的 Split Sum 采样。

3Sky Light：天光与 IBL 兜底

ASkyLight 捕获周围天空（可以是 HDR 天空球、SkyAtmosphere 的大气，或固定颜色），生成：

  • Irradiance（辐照度）球谐函数（SH）：低频漫反射间接光的来源，在每像素根据法线方向查 SH 系数得到漫反射间接光；
  • 预卷积镜面立方体图：同 Reflection Capture，但代表无限远处的天空信息，用于粗糙表面的镜面回退。

天光的意义：Reflection Capture 只覆盖有限区域，角色移动到没有 Capture 的区域时仍需要天光兜底。天光提供全局均匀的漫反射 ambient，是保证场景不"纯黑"的最后防线。

具体：先用 IBL（反射捕获 + 天光 Split Sum）算出基础镜面间接光；再把 SSR 结果按其置信度（alpha 通道）做 lerp 叠加：lerp(IBL, SSR, SSR.alpha)。SSR.alpha 为 0 的区域完全用 IBL，为 1 的区域完全用 SSR，中间过渡。

为什么不直接用 SSR 代替一切

SSR 的置信度在以下情况下为零：反射方向射向屏幕之外（视角外）、被几何遮挡的表面、光滑度低于阈值的表面。若强制用 SSR 这些区域反射为黑——视觉上更差。分层回退策略确保了最坏情况（SSR 完全失效）时仍有合理的 IBL 兜底。

5平面反射（Planar Reflection）

对于需要高精度反射的平面（水面、镜子），UE 4.26 提供 APlanarReflectionActor：在反射平面的镜像位置再渲染一帧场景，直接作为反射纹理——精度最高，但代价是多一次完整的场景渲染（Draw Call 翻倍）。只在视觉关键的地标平面上使用，并通过 r.Reflections.AllowScreenSpaceReflections 控制其与 SSR 的混合策略。

6漫反射间接光的完整路径

容易混淆：IBL 同时提供镜面反射（上面）和漫反射间接光（ambient diffuse）。漫反射路径：天光 SH → 像素法线 → 漫反射 IBL 辐照度，结合 GBuffer 的 BaseColor（漫反射颜色），得到间接漫反射分量，再乘以 SSAO 系数（0027）后叠加到 SceneColor。

动态 Reflection Capture 的性能代价

Reflection Capture 默认在编辑器"Build Reflection Captures"时离线烘焙，运行时只读不写——极廉价。若设置为 Realtime Capture（实时捕获），每帧重新渲染立方体图的 6 个面，相当于 6 次额外场景渲染，仅用于绝对必要的动态反射场景（如随时间变化的天空）。

检索练习

Reflection Capture 立方体图为什么要存多个 mip 级别，且 mip 越高对应粗糙度越大？

当 SSR 置信度为零（反射方向射出屏幕外）时，这个像素的镜面间接光从哪里来？经过哪些数据和计算步骤？
  1. 根据该像素的世界坐标找到最近的 Reflection Capture（或多个 Capture 空间混合）；
  2. 从 GBuffer B 读取粗糙度，调用 ComputeReflectionCaptureMipFromRoughness 换算为 mip 级别；
  3. 用世界法线和视方向算出反射方向，以 (反射方向, CaptureIndex, MipLevel) 采样 FReflectionEnvironmentCubemapArray；
  4. 乘以 Split Sum 第二项（BRDF LUT，按 NoV 和粗糙度查 PreIntegratedGF），得到 SpecIBL；
  5. SSR.alpha = 0 → lerp 100% 用此 IBL 结果写入 SceneColor 的镜面分量。
  6. 天光的预卷积立方体图作为无穷远处兜底（当 Capture 覆盖不足时）。

带走这几条

  • 反射能量来源有三层：SSR（屏幕内最准）→ Reflection Capture（预卷积立方体图）→ Sky Light（全局兜底），按 SSR 置信度 lerp 回退。
  • Reflection Capture 的 IBL 用 Split Sum 近似：粗糙度 → mip 级别选择预卷积环境图，乘以 BRDF LUT，单次采样近似完整 GGX 积分。
  • FReflectionEnvironmentCubemapArray 是所有捕获立方体图的数组，按 Capture Index 索引。
  • 天光 SH 提供低频漫反射 IBL；天光预卷积立方体图提供粗糙镜面兜底。
  • 合成入口：RenderDeferredReflectionsAndSkyLighting()（ReflectionEnvironment.cpp）。
  • 平面反射（Planar Reflection）代价高（重新渲染场景），仅用于视觉关键平面。
